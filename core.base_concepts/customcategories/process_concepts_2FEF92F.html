Основное описание

Процесс разработки содержит структурированное определение действий, которые необходимо выполнить для разработки системы. Таким действием может быть реализация проекта в соответствии с формальным процессом.   Структурированное определение процесса позволяет распределить операции по времени или этапам жизненного цикла и сформировать так называемую структуру процесса и (или) диаграммы операций. Таким образом, процесс представляет собой упорядоченную иерархическую структуру базовых элементов метода, примерами которых могут служить задачи и продукты работы. Стандартные элементы адаптируются и уточняются для применения в конкретных проектах.

Для определения процессов в UMA применяются следующие фундаментальные концепции:

  • Операция
  • Шаблон возможностей
  • Процесс поставки
  • Дополнение к процессу
  • Дескриптор
  • Составная роль
  • Профайл группы
