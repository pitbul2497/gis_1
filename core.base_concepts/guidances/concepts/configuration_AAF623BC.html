Концепция: Конфигурация метода
Конфигурация метода представляет собой логическое подмножество библиотеки методов.
Основное описание

Конфигурация метода - это совокупность модулей метода, пакетов и процессов метода (см. разделы Концепция: шаблон возможностей и Концепция: процесс поставки). Конфигурацию можно экспортировать в автономную библиотеку, если она содержит полный в себе список элементов, в котором нет внешних зависимостей. Конфигурация метода представляет собой логическое подмножество библиотеки методов.

Приложения

Конфигурация обычно создается для одного или нескольких процессов поставки. Процесс поставки может применяться в нескольких конфигурациях методов, однако его содержание может быть скорректировано в каждой из этих конфигураций. Например, в процесс поставки можно включить наполнение для разработки схем различных реляционных и объектно-ориентированных систем управления базами данных. При работе с таким процессом поставки пользователь может выбрать нужный тип СУБД и исключить из конфигурации наполнение для всех остальных типов СУБД. Для этого нужно выбрать конфигурацию, в которой исключено лишнее наполнение, или создать такую конфигурацию.
