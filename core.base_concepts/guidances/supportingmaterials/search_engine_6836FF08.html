Функция поиска
Функция поиска предназначена для поиска страниц на опубликованном сайте.
Основное описание

Примечание: функция поиска реализована в виде аплета, и поэтому для ее работы нужна среда JRE 1.4.2 или одной из последующих версий (JRE можно загрузить с сайта http://java.sun.com/j2se).

Советы по работе с функцией поиска

Функция поиска позволяет находить страницы на опубликованном сайте несколькими способами. Примеры:

  • Можно найти страницы, на которых есть все указанные слова.
  • Можно найти страницы, на которых есть хотя бы одно из указанных слов.
  • Можно найти страницы, на которых есть точные фразы.
  • Можно найти страницы, на которых нет ни одного из указанных слов.

Введите искомые слова в одном из полей Все слова, Любое слово, Точная фраза и Без слов и нажмите ENTER или кнопку Найти. По завершении поиска названия и краткие описания всех найденных страниц будут показаны в поле Результаты. Для просмотра текста страницы в окне опубликованного сайта щелкните на ее названии в окне результатов поиска.

Например, для поиска страниц, на которых одновременно есть слова "Rational", "Unified" и "Process", а также хотя бы одно из слов "артефакт" и "видение", нужно ввести Rational Unified Process в поле Все слова и артефакт видение в поле Любое слово.

Можно изменить количество результатов, которое будет показано на одной странице в списке Показать. Если все результаты не поместятся на одну страницу, можно перемещаться по страницам с помощью кнопок вперед и назад.

Кроме того, можно указать область поиска страниц: опубликованный сайт или developerWorks. Для переключения между опубликованным сайтом и developerWorks нужно выбрать соответствующее значение в списке В разделе.

Поиск слова на странице

После того как текст найденной страницы будет показан в браузере, можно найти на ней нужное слово с помощью средств поиска браузера. Для поиска средствами браузера нажмите CTRL+F.

Ввод запроса для поиска

Запрос для поиска состоит из одного или нескольких слов. Булевские операторы не поддерживаются. Вместо булевских операторов можно пользоваться полями Все слова, Любое слово, Точная фраза и Без слова. Поиск выполняется без учета регистра, то есть между Hello, HELLO и hElLo разницы нет. Символ подстановки не поддерживается: *.

Если значения указаны в нескольких полях, они обрабатываются сверху вниз в порядке убывания приоритета. Например, можно указать следующий запрос:

Все слова: project management
Любое слово: adopt vision
Точная фраза: Rational Unified Process
Без слов: implementation


Этот запрос будет интерпретирован следующим образом: (project AND management) AND (adopt OR vision) AND (Rational Unified Process) NOT (implementation). То есть, страница должна содержать слова "project" и "management", хотя бы одно из слов "adopt" и "vision", а также фразу "Rational Unified Process". При этом на странице не должно быть слова "implementation".

Особые рекомендации

  • Функция поиска игнорирует служебные слова, например "where", "when" и "the", поскольку эти слова игнорируются при индексировании страниц для функции поиска. Исключение этих слов из индекса позволяет повысить скорость поиска без снижения качества.
  • Чтобы значение в поле Без слов имело смысл, нужно указать какое-либо значение хотя бы в одном из остальных полей. Иными словами, если не будет указано какие слова должны быть на искомых страницах, функция поиска не сможет найти страницы, на которых нет определенных слов.
  • Функция поиска не поддерживает символы подстановки.
  • Булевские операторы не поддерживаются. Информация о том, как формировать запросы, равносильные запросам с булевскими операторами, приведена в разделе Ввод запроса для поиска.
  • Не рекомендуется указывать отдельные цифры от 0 до 9 для поиска. Вместо поиска числовых значений рекомендуется указывать соответствующие слова, например "ноль", "шесть", "девять", "десять" и так далее.
