Определите индикаторы проекта

"Индикаторы" проекта - это информационные элементы проекта, сообщающие о ходе создания проекта согласно плану разработки программного обеспечения. Обычно руководителя проекта интересуют индикаторы, относящиеся к объему работы, бюджету, качеству и рискам проекта. По мере создания проекта руководитель проекта отслеживает эти индикаторы и в случае нарушения заранее заданных пороговых условий инициализирует выполнение исправляющих действий (см. раздел Определить процедуру и пороги для исправляющих действий). Ниже перечислены возможные индикаторы проекта:

  • Общие расходы по сравнению с бюджетом
  • Проверенный объем работы (выполненная работа + оценка того, что предстоит выполнить) по сравнению с запланированным объемом работы
  • Плотность дефектов по сравнению с требуемым качеством
  • Индикаторы рисков (ситуации, сообщающие о наличии риска)

Определение этих индикаторов зависит от бюджета проекта, требуемого качества и расписания (подробное описание - в Плане разработки программного обеспечения) и захватывается в Плане измерения проекта и Плане управления рисками.

Определите источники индикаторов проекта

В большинстве случаев, индикаторы проекта будут объединенными величинами проекта, вычисляемыми по нескольким более конкретным простым показателям. Эти показатели регулярно сообщаются коллективом проекта. Способ захвата этих простых показателей и процесс их использования для вычисления индикаторов проекта определяются в Плане управления проектом.

Другие индикаторы (особенно индикаторы рисков) могут просто указывать, возникла ли та или иная конкретная ситуация или нет. В этих случаях достаточно определить источник информации для состояния, отображаемого индикатором.

Раздел 4.4 Отслеживание и управление проектом Плана разработки программного обеспечения (SDP) должен включать краткое описание индикаторов проекта, которые будут применяться в вашем проекте. Учтите, что в этом разделе SDP темы, связанные с управлением расписанием, бюджетом и качеством, выделены в отдельные подразделы. Управление требованиями проекта рассматривается отдельно - в Плане управления требованиями.

Определите процедуру отчета коллектива о текущем состоянии

После определения простых показателей и индикаторов проекта вы должны определить процедуру и частоту, с которой сотрудники коллектива проекта будут отчитываться о текущем положении дел. В этой процедуре должны быть предусмотрены выделение времени на предстоящие операции, отчет о выполнении задач, прохождение контрольных точек проекта и отчет о проблемах. Для обеспечения согласованности потока информации обычно определяют стандартные шаблоны для расписаний и отчетов коллектива сотрудников о текущем состоянии.

Эта процедура документирована в Разделе 4.4.5 План отчетности Плана разработки программного обеспечения.

Определите пороги процедуры для исправляющих действий

Для того чтобы постоянно следить за ходом проекта, руководитель проекта определяет пороговые значения или условия (триггеры) для каждого из определенных индикаторов проекта. Эти пороговые условия записаны в соответствующих параграфах Раздела 4.4 Отслеживание и управление проектом Плана разработки программного обеспечения.

При превышении этих порогов руководитель проекта должен выполнить исправляющее действие, чтобы вернуть проект в нормальное состояние. В зависимости от серьезности условия, руководитель проекта может исправить ситуацию в рамках своих полномочий (выдав соответствующие задания). Если ситуация выходит за рамки полномочий руководителя проекта, то он должен будет выдать Запрос на изменение и запустить процесс управления изменением проекта.

Определите процедуру отчета о состоянии проекта

В Разделе 4.4.5 План отчетности Плана разработки программного обеспечения должны быть также описаны частота и процедура, согласно которым руководитель проекта должен отчитываться о ходе разработки проекта Контрольному совету проекта (PRA) (путем выдачи Отчета о текущем состоянии). Эта процедура определяет, когда и где произойдут запланированные и внеплановые проверки PRA и какую информацию следует включить в Отчет о текущем состоянии. В периоды между составлением Отчетов о текущем состоянии руководитель проекта будет записывать информацию о проблемах в Список вопросов (за исключением проблем, относящихся к другим управляющим инструментам, например Запросу на изменение или Списку рисков).
