Определите процедуры устранения неполадок

Первый шаг при разработке плана устранения неполадок - это определение процедур устранения неполадок каждой категории. В Rational Unified Process эти процедуры основаны на следующих факторах:

  • в задаче Задача: Управление исключительными ситуациями, - на неполадках, указанных в Оценке состояния;
  • на запросах изменений, требующих устранения неполадок
  • на фактах отклонения от нормы, обнаруженных в процессе проверок
  • на случаях несоответствия, обнаруженных при проведении проверок и процедур контроля

Оценки состояния проводятся в процессе подготовки к запланированным проверкам состояния проекта. Однако если обнаружены неполадки, требующие немедленного устранения, Список дефектов (если он существует) можно обновлять в любое время в процессе выполнения Задачи: Мониторинг состояния проекта.

Необходимо учитывать следующие моменты:

  • Методы, используемые специалистами для оглашения неполадки (например, определение неполадки, подача запроса изменения).
  • Кто будет оценивать неполадку и утверждать методы ее устранения?
  • Каким образом будет реализовано решение об устранении неполадки (например, будет подан запрос изменения или установлен порядок обработки)?
  • Каковы критерии того, что процесс устранения неполадки завершен?
Выберите инструменты и методы отслеживания неполадок

Необходимо составить и постоянно обновлять список или протокол неполадок и вести записи об их статусе. Для разных категорий неполадок можно использовать разные инструменты. Например, с помощью системы отслеживания неполадок можно управлять неполадками, относящимися к продукту, а с помощью простых таблиц можно следить за неполадками в проекте.

На этом этапе выберите инструменты, базы данных и файлы для отслеживания неполадок в проекте. Выберите методы работы. Эти методы могут касаться следующих аспектов:

  • Определение проблемы
  • Анализ проблемы
  • Определение важности проблемы
  • Подтверждение действий по устранению неполадок
Распределите задачи по устранению неполадок между группами специалистов

В большинстве проектов возникающие неполадки проверяются на постоянной основе "сборным" комитетом, в который входят представители каждого отдела проекта (например, управление проектом, разработка, тестирование, контроль качества). Комитет рассматривает каждую проблему и разрабатывает план ее устранения.

В плане укажите специалистов, которые войдут в этот сборный комитет. Если разные категории неполадок будут рассматриваться разными комитетами, укажите состав каждой группы отдельно.

Также укажите специалистов (одного или нескольких), которые будут утверждать окончание процесса устранения каждой неполадки.

Составьте расписание работы над неполадками

Разработайте и занесите в план расписание регулярных совещаний сборного комитета.

Расписание работы над неполадками необходимо для правильного хода проекта. Регулярные совещания дают возможность обсуждать и устранять неполадки. Во многих организациях практикуются ежедневные утренние совещания, где любой сотрудник может представить на рассмотрение новую неполадку.
