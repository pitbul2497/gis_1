Определите инструменты управления рисками

При составлении плана управления рисками сначала нужно разработать процедуры для следующих действий:

  • Обнаружение рисков
  • Анализ рисков
  • Классификация рисков по важности

Также нужно выбрать специализированные инструменты и методы для сбора и хранения информации о рисках. Например, можно создать список рисков и открыть доступ к нему для других систем. Дополнительная информация о процедурах управления рисками в Rational Unified Process приведена в разделах Рекомендация по рабочему продукту: Список рисков и Задача: Определение и оценка рисков.

Создайте начальный список рисков

Перед разработкой стратегий управления рисками нужно определить виды рисков. На ранних стадиях начального этапа составьте первоначальный список рисков и используйте его при разработке стратегий. При составлении этого списка следуйте инструкциям, приведенным в разделе Задача: Определение и оценка рисков.

Сформируйте группу управления рисками

Выберите специалистов из группы проекта, которые будут управлять рисками. Важно, чтобы в группу управления рисками входили представители как технических, так и руководящих отделов проекта. Рекомендуется такой состав группы: руководитель проекта, представитель заказчика (или менеджер по продукции), архитектор программного обеспечения и руководители групп тестирования, разработки, документации и развертывания. Рекомендуется выбрать руководителя группы управления рисками. Его задача - собирать и классифицировать риски сразу после их обнаружения, разрабатывать отчеты о состоянии рисков и составлять расписание собраний по работе над рисками.

Разработайте стратегии по устранению 10 самых серьезных рисков

Группа управления рисками разрабатывает принципы отслеживания каждого риска из списка, а также план действий, если какой-то риск материализуется (план действий в нештатной ситуации). Существуют следующие методы управления рисками: избежание, перенос, принятие и снижение. Более подробно эти стратегии описаны в разделах Рекомендация по рабочему продукту: Список рисков и Задача: Определение и оценка рисков.

Укажите индикаторы для 10 самых серьезных рисков

Для каждого риска из списка определите условие, выполнение которого будет сигнализировать, что риск скоро оправдается. Такие условия называются индикаторами рисков. Руководитель проекта должен следить за индикаторами на протяжении всего проекта и при необходимости реализовывать план действий в нештатной ситуации, указанный в плане управления рисками.

Создайте расписание составления отчетов и проведения собраний

Управление рисками наиболее эффективно, если оно рассматривается как постоянный процесс. В план управления рисками необходимо добавить расписание составления отчетов о состоянии рисков и проведения собраний по работе над рисками. Расписание также должно содержать информацию об условиях проведения незапланированных собраний.

Дополнительная информация об управлении рисками приведена в разделе Рекомендация по рабочему продукту: Список рисков.
