Назначение
Задача процедур поддержки разработки заключается в поддержке среды разработки (процессов и инструментов), в которой работает коллектив разработчиков программного обеспечения.

Описание
В ходе первых итераций проекта выполняется подготовка среды. Затем среда корректируется по мере необходимости.

Примечания по использованию

Выберите способ выполнения потока операций

Необходимо принять несколько решений в отношении потока операций дисциплины управления средой:

  • Проанализируйте поток операций и выберите способ его выполнения. Изучите диаграмму и указанные на ней сторожевые условия с учетом приведенных рекомендаций. Определите, какие операции и в каком порядке нужно выполнить.  
  • Решите, какие операции управления средой нужно выполнить. В целом артефакты дисциплины управления средой вводятся по мере необходимости. Например,  рекомендации по стилю руководств разрабатываются только в случае, если в состав проекта входят справочные материалы.
  • Решите, в какие моменты должны вводиться те или иные компоненты потока операций.  Процесс разработки всегда вводится в начале проекта.   Кроме того, можно создать  прецедент разработки и задокументировать в нем подробные инструкции по настройке процесса для данного конкретного проекта. Пользовательский процесс поставки можно импортировать в Rational Portfolio Manager для упрощения распределения работы и контроля хода выполнения проекта. Другие артефакты вводятся по мере необходимости.
