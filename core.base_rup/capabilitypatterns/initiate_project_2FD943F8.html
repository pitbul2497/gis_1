Назначение Совета проекта (PRA)

Совет проекта (PRA) является организационным элементом, который отвечает за наблюдение за проектом. Настоятельно рекомендуется чтобы сотрудникам в PRA помощь в наблюдении за проектом оказывала определенная группа старшего технического и управленческого персонала из организации проекта, а также персонал заказчика на уровне исполнителей. В состав типичной группы, помогающей PRA, для договорного проекта создания программного обеспечения среднего размера может входить следующий персонал:

  • VP, разработка программного обеспечения
  • VP, маркетинг
  • Руководитель контроля качества
  • Ответственный представитель процесса разработки программного обеспечения
  • Руководитель бизнес-подразделения клиента, заказывающего программное обеспечение
Назначение руководителя проекта

Руководитель проекта с соответствующими навыками и опытом назначается и утверждается советом проекта.

Назначение группы планирования проекта

Группа планирования проекта является начальной группой в коллективе проекта, которая выполняет работу на начальном этапе. Группа планирования определяется, утверждается и назначается руководителем проекта совместно с PRA. В состав группы планирования проекта могут выходить следующие сотрудники:

  • Руководитель проекта
  • Разработчик архитектуры
  • Аналитики системы
  • Руководитель разработки
  • Руководитель тестирования
  • Руководительн управления конфигурацией
  • Представитель заказчика
Утверждение критериев приемки продукта

Последним шагом на начальном этапе проекта является определение некоторых объективных критериев, которые будут применяться заказчиком для определения приемлемости конечных продуктов. Эти критерии следует разрабатывать совместно организацией заказчика и коллективом проекта. В их число могут входить следующие критерии:

  • Доставка заказчику всех рабочих продуктов, определенных как конечные продукты
  • Список необходимых участников тестирования приемлемости
  • Необходимое расположение(я) тестов
  • Успешное выполнение оценок рабочего продукта, определенных в плане приемки продукта
  • Успешное завершение обучения заказчика
  • Успешное выполнение установки в системе заказчика
  • Показатели, которые определят степень соответствия исходным спецификациям проекта
  • Показатели, которые определят степень соответствия задачам бизнес-прецедента
