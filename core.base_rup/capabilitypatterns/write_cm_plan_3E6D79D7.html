Написание плана управления конфигурацией
Цель План управления конфигурацией описывает все связанные задачи, подлежащие выполнению в течение жизненного цикла продукта/проекта. Он содержит подробную информацию о расписании задач, ответственных сотрудниках и требуемых ресурсах (включая персонал, инструменты и аппаратное обеспечение).  

План управления конфигурацией представляет собой составной документ, в котором приведена информация, необходимая для эффективного выполнения задач управления конфигурацией. В разделе Рабочий продукт: План управления конфигурацией описаны исходные данные, необходимые для эффективного планирования управления конфигурацией. Шаблон выполняет роль рекомендаций. Смысл каждого раздела следует рассматривать с учетом контекста конкретного проекта/продукта. Создание плана не должно вызвать затруднений; план не должен быть чрезмерно формальным и создавать бюрократических помех.

Обратите внимание, что план управления конфигурацией является важным документом, гарантирующим доступность организационных ресурсов.

Проверка и утверждение плана управления конфигурацией
Цель Эффективное представление плана управления конфигурацией основным заинтересованным лицам.  

План управления конфигурацией разрабатывается после утверждения видения и экономического обоснования проекта. Он составляется руководителем проекта параллельно с другими планами проекта и проверяется всеми заинтересованными сторонами.

Несмотря на то, что окончательное решение об утверждении плана принимает руководитель проекта, он должен быть проверен всеми заинтересованными сторонами. В частности, менеджером по конфигурации, ключевыми разработчиками и интеграторами.

Обслуживание плана управления конфигурацией
Цель Проверка актуальности плана.  

План управления конфигурацией проекта следует уточнять в начале каждого этапа (построение и внедрение), для того чтобы обеспечить соответствие между общей стратегией управления конфигурацией и текущим состоянием разработки проекта.

План управления конфигурацией необходимо передать системе управления конфигурацией и разместить в подкаталоге 'plans' структуры проекта.
