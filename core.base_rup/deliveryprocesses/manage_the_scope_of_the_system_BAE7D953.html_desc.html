Описание

Данная операция предполагает выполнение следующих действий:

  • Определение приоритетов и уточнение входных данных для выбора компонентов и требований, которые должны быть включены в текущую итерацию (см. раздел Задайте приоритет вариантов).
  • Определение набора сценариев поведения вариантов, существенных для архитектуры (см. раздел Задайте приоритеты вариантов).
  • Определение механизма прослеживаемости, включая выбор атрибутов требований и взаимосвязей прослеживаемости (см. раздел Управление зависимостями).
  • Уточнение видения для описания содержания решения (см. раздел Разработка видения).

Содержание проекта задается набором установленных для него требований. Ключ к успешной реализации проектов заключается в приведение содержания проекта в соответствие с объемом ресурсов (времени, людей и средств) для его реализации.

Системные аналитики отвечают за получение информации о приоритете, трудоемкости, стоимости, риске и других характеристиках требований у заинтересованных лиц. Данная информация заносится в хранилище атрибутов требований. Руководитель проекта пользуется этими атрибутами при планировании итераций, а разработчик архитектуры - при поиске существенных для архитектуры или готовых вариантов для создания представления вариантов  для архитектуры (см. раздел Задайте приоритет вариантов).

Персонал

В эту операцию должны быть вовлечены все разработчики архитектуры.   Коллектив разработчиков должен провести общее собрание и расставить на нем приоритеты требований.  

Кроме того, не помешает обучить руководителей коллективов ведению переговоров и назначить единого лидера проекта на стороне исполнителя и такого же лидера на стороне заказчика. У лидеров должны быть достаточно полномочий на отказ от реализации требований, которые невозможно реализовать с имеющимися в распоряжении ресурсами и на выделение дополнительных ресурсов, необходимых для удовлетворения дополнительных требований.

Указания по использованию

Эту процедуру следует непрерывно выполнять на всем протяжении проекта.

Ключевые условия

Применение атрибутов требований, включая приоритет, трудоемкость и риск, в качестве параметров при согласовании набора требований к системе очень удобно в ходе определения содержания проекта. Смещение внимания с требований как таковых на их атрибуты позволяет исключить эмоции из переговоров о согласовании требований к системе.
