Дисциплина: Реализация
Данная дисциплина охватывает разработку, систематизацию, тестирование и интеграцию компонентов в соответствии со спецификациями плана.

Основное описание

Цели дисциплины реализации:

  • структурирование кода путем создания нескольких уровней подсистем
  • реализация элементов плана в форме элементов реализации (исходных файлов, исполняемых файлов, исполняемых программ и т.п.)
  • тестирование блоков из разработанных компонентов
  • интеграция результатов работы отдельных сотрудников (или коллективов) в рабочую систему

В дисциплине реализации тестирование рассматривается только в рамках тестирования блоков классов. Тестирование системы и интеграции входит в сферу ответственности дисциплины тестирования.

Реализация связана с другими дисциплинами:

  • Дисциплина управления требованиями посвящена сбору информации о требованиях и описанию требований в форме модели прецедентов.
  • Дисциплина анализа и проектирования посвящена разработке модели проекта. Модель проекта - это объект, подлежащий реализации и основной источник данных для дисциплины реализации.
  • Дисциплина тестирования охватывает вопросы тестирования интеграции каждой компиляции в ходе интеграции системы. Кроме того, эта дисциплина занимается проверкой выполнения требований, обнаружением и регистрацией дефектов.
  • Дисциплина управления средой посвящена разработке и обслуживанию артефактов, используемых в ходе реализации, включая описания процессов, рекомендации по проектированию и рекомендации по программированию.  
  • В дисциплине развертывания рассматриваются вопросы применения модели реализации для создания и поставки кода конечному пользователю.
  • Дисциплина управления проектом охватывает вопросы планирования проекта. При планировании проекта важную роль играют план итераций и системы управления изменениями и мониторинга дефектов.
