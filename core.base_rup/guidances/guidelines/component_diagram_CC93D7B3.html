Основное описание

Объяснение

На диаграмме компонентов показана структура компонентов, включая обозначающие их классификаторы и реализующие их артефакты.

Их также можно использовать для изображения высокоуровневой структуры Модели реализации в терминах Подсистем реализации и взаимосвязей между Элементами реализации.

Применение

Диаграммы компонентов RUP применяются, главным образом, для изображения высокоуровневой структуры Модели реализации. Конкретно:

  • Подсистемы реализации и их импортные зависимости. См. раздел Рекомендация по рабочему продукту: импортная зависимость в реализации.
  • Подсистемы реализации с многоуровневой структурой.

Диаграммы компонентов также могут применяться для изображения других объектов. Это:

  • Файлы исходного кода и их компиляционные зависимости. См. раздел Рекомендация по рабочему продукту: компиляционная зависимость в реализации.
  • Файлы приложений и их зависимости среды выполнения.
  • Производные взаимосвязи между файлами исходного кода и файлами, создаваемыми в результате компиляции или связывания.
  • Реализационные зависимости между элементами реализации и реализуемыми ими элементами проектирования. См. раздел Рекомендация по рабочему продукту: манифестная зависимость.

Заметьте, что в RUP придается особое значение применению диаграммы компонентов для моделирования Подсистем реализации, важнейших Элементов реализации и их взаимосвязей. Однако диаграмму компонентов, изображающую компоненты UML и обозначающие их классификаторы, можно использовать и в Модели проектирования, используя компонент UML для представления артефакта Подсистема проектирования. Дополнительная информация по этой теме приведена в разделе Рекомендация по рабочему продукту: подсистема проектирования.
