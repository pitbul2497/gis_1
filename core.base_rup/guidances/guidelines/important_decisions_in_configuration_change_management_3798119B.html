Рекомендация: Важные решения в сфере управления конфигурацией и изменениями
В этой рекомендации приведено описание важных аспектов, которые нужно учитывать при адаптации элементов управлении конфигурацией и изменениями для конкретного проекта.

Основное описание

Выбор способа применения рабочих продуктов

Выберите рабочие продукты и способ их применения.  Помимо этого, обязательно адаптируйте каждый продукт к особенностям проекта.  

В следующей таблице перечислены рабочие продукты управления конфигурацией и изменениями. Эти продукты разделены на рекомендуемые и необязательные (которые можно использовать только в определенных случаях). Дополнительные замечания по адаптации рабочих продуктов приведены в разделе об адаптации на странице описания конкретного продукта.

Рабочий продукт Назначение Адаптация (Необязательный продукт, рекомендуется)
Запрос изменения Используется для отслеживания запрошенных изменений рабочих продуктов, включая неисправности. Рекомендуется для большинства проектов.
Результаты контроля конфигурации Содержит результаты контроля конфигурации. Необязательный продукт.

Обычно составляется в более официальных процессах.
План управления конфигурацией Содержит описание всех задач, связанных с управлением конфигурацией и изменениями (CCM), и запланированных на весь жизненный цикл проекта или продукта. Необязательный продукт.
Хранилище проекта Содержит все версии файлов и каталогов проекта. Рекомендуется.

Система управления конфигурацией, которая отслеживает версии файлов и компоновки, рекомендуется для всех проектов.
Рабочая область Предоставляет индивидуальную область разработки, где отдельный специалист может редактировать рабочие продукты, при этом изменения не будут сразу видны другим участникам группы. Рекомендуется для большинства проектов.
