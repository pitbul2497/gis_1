Основное описание

Выбор способа применения рабочих продуктов

Выберите рабочие продукты и способ их применения.  Помимо этого, обязательно адаптируйте каждый продукт к особенностям проекта.  

В следующей таблице перечислены рабочие продукты среды. Эти продукты разделены на рекомендуемые и необязательные (которые можно использовать только в определенных случаях). Дополнительные замечания по адаптации рабочих продуктов приведены в разделе об адаптации на странице описания конкретного продукта.

Рабочий продукт Назначение

Адаптация (Необязательный продукт, рекомендуется)

Процесс разработки Содержит данные о процессе разработки проекта. Обеспечивает доступ специалистов ко всем указаниям, относящимся к процессу. Рекомендуется для всех проектов.
Прецедент разработки С помощью этого рабочего продукта производится доводка процесса разработки и его адаптация к требованиям проекта. В частности, определяются рабочие продукты, которые будут созданы, и время их создания.

Необязателен в небольших проектах, потому что в них обычно достаточно рабочего продукта Процесс разработки. Рекомендуется для средних и крупных проектов.

Рекомендации к проекту Рекомендации к проекту применяются при наличии стандартов или устоявшихся процедур, на которые нужно ориентироваться при работе над проектом.

Рекомендуются для соответствующих задач проекта.

Во многих проектах не создаются свои рекомендации, а используются уже готовые. В небольших сплоченных группах с опытными специалистами, разделяющими общую точку зрения, можно официально не документировать некоторые рекомендации. Однако в этом случае существует риск, что со временем может меняться представление о стандартах и качестве.

Шаблоны проекта Служат основой для создания документальных и модельных рабочих продуктов и обеспечивают их взаимосвязь.

Рекомендуются для соответствующих рабочих продуктов.

Инфраструктура разработки (включая Инструменты) Инструменты аппаратного и программного обеспечения, применяемые при разработке. В любых проектах существует инфраструктура разработки. Во многих проектах не создается своя инфраструктура разработки, а используется уже готовая.
Оценка организации разработки Инженер процесса опирается на этот рабочий продукт при адаптации процесса для отдельной организации.

Необязательный продукт.

В крупных организациях такая оценка часто необходима для вынесения связанных с проектом решений.

Руководство по стилю публикаций Определяет стиль и качество справочных материалов.

Рекомендуется для большинства проектов со справочными материалами.

Во многих проектах не создаются свои руководства, а используются уже готовые.
