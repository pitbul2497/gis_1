В данной памятке по инструменту описывается процедура добавления элементов к управлению исходным кодом с помощью Rational ClearCase.
Инструмент: Rational ClearCase

Основное описание

Обзор

В центре ClearCase находятся мощные функции по управлению и контролированию версий элементов, таких как файлов с исходным кодом для программного обеспечения и документации, каталогов, отчетов и т.д. Перед изъятием одного или нескольких элементов необходимо вначале добавить исходную версию этих элементов в управление исходным кодом.

Этапы работы с инструментом

С помощью следующих шагов можно добавить один или несколько элементов в управление исходным кодом с помощью Rational ClearCase:

  • Запустите ClearCase Explorer
  • Перейдите к элементам, которые требуется добавить в управление исходным кодом
  • Выберите элементы
  • Добавьте элементы к управлению исходным кодом
  • Выполните операцию управления исходным кодом

    Дополнительная информация

Запустите ClearCase Explorer

  1. В меню Пуск выберите Программы > Rational Software > Rational ClearCase > ClearCase Explorer
  2. Выберите панель, указывающую на VOB, в котором требуется добавить элемент(ы) к управлению исходным кодом.


  3. Если панель отсутствует, создайте ее. Если панель существует, но VOB не виден, необходимо смонтировать VOB, который будет содержать элемент(ы).

Перейдите к элементам, добавляемым к управлению исходным кодом

Рекомендуется выбирать наиболее удаленные от структуры корневого каталога элементы. Для каждого элемента ClearCase также добавляет родительские каталоги (до корневого каталога VOB), еще отсутствующие в управлении исходным кодом.

Примечание: Перед добавлением элемента в VOB следует изъять его родительский каталог из панели. Если вы еще не сделали этого, ClearCase изымет его после создания элемента.

Выберите элемент(ы)

  1. Для добавления в управление версиями можно выбрать один или несколько элементов с помощью стандартных методов выбора.

Добавление элементов к управлению версиями

  1. Щелкните правой кнопкой мыши на выбранных файлах и выберите во всплывающем меню опцию Добавить к управлению исходным кодом.
  2. Для добавления элементов в проект Унифицированное управление изменениями (UCM) выберите или создайте операцию ClearCase в окне Добавить к управлению исходным кодом.
  3. В поле Комментарии опишите элементы, добавляемые к управлению исходным кодом.
  4. Для продолжения работы с новыми элементами после их добавления к управлению исходным кодом выберите опцию Изъятьпосле добавления к управлению исходным кодом. Чтобы оставить их добавленными, отключите эту опцию.
  5. При работе в скопированном VOB (MultiSite) без применения панели UCM можно сделать текущим главным узлом копии любую ветвь, созданную при выполнении операции Добавить к управлению исходным кодом, выбрав опцию Сделать текущую копию главной для всех создаваемых ветвей. После создания элемента вы сможете создать новые версии элемента в любой новой ветви.

  6. Если эта опция не выбрана, главной ветвью элемента становится копия, являющаяся главным типом ветви. Если главная копия не является текущей копией, вы не сможете создать новые версии ветви. Также если Спецификация конфигураций содержит правила автоматического создания ветвей, создать ветвь если типы ветвей не являются главными для текущей копии не удастся.

Выполните операцию управления исходным кодом

Выполните действие следующим образом:

  1. Для добавления только одного элемента к управлению исходным кодом нажмите кнопку OK.
  2. Для добавления нескольких элементов к управлению исходным кодом и применения одинаковых комментариев и изъятия ко всем элементам, нажмите кнопку Применить ко всем.
  3. Для отмены операции нажмите кнопку Отмена. Элементы не будут добавлены к управлению исходным кодом и останутся файлами для частного просмотра.
