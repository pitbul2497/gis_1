Руководство по инструменту: Добавление шаблонов в проект Rational RequisitePro
В данной памятке по инструменту описан процесс применения документов Microsoft® Word в качестве шаблонов для документов в проектах Rational RequisitePro. В RequisitePro шаблоны называются "схемами" документов.
Инструмент: Rational RequisitePro
Взаимосвязи
Связанные элементы
  • Настройка инструментов
  • Подготовка шаблонов для проекта
  • Разработка плана управления требованиями
  • Разработка справочных материалов
Основное описание

Обзор

Каждый новый документ в RequisitePro создается исходя из типа документа. Тип документа включает в себя расширение файла по умолчанию, тип требований по умолчанию, текст и форматирование оп умолчанию (касающиеся шрифта, табуляций и т.д.) и контролируется связанным с ним шаблоном. RequisitePro предоставляет следующие схемы для разработки требований ипрецедентов.

  • Документ с требованиями к продукту
  • Спецификации требований программного обеспечения
  • Современные спецификации требований программного обеспечения
  • Спецификация нескольких прецедентов
  • Документ с тестовыми требованиями
  • Прецеденты функциональных тестов
  • План управления требованиями
  • Запросы заинтересованных лиц
  • Видение
  • Глоссарий
  • Спецификации прецедентов
  • Спецификации требований программного обеспечения (с вариантами)
  • Спецификации требований программного обеспечения (без вариантов)
  • Дополнительная спецификация
  • План тестирования

Доступ к схемам

Обратиться к схемам документов можно несколькими способами:

  • В RequisitePro схемы связаны с типами документов. Для создания документа выберите пакет в проводнике и нажмите Файл > Создать > Документ. Затем выберите тип документа для создания нового документа. Для добавления нового типа документа и схемы в проект, выберите проект в проводнике, выберите в меню Файл > Свойства, перейдите на вкладке Типы документов и нажмите кнопку Добавить. В диалоговом окне Тип документа выберите схему.
  • Схемы, созданные на базе файлов Word .dot, находятся в следующем каталоге установки RequisitePro: Program Files/RationalRequisitePro/outline.
  • Шаблоны Word доступны через браузер RUP (выберите Шаблоны > Microsoft Word).

Создание пользовательских схем

С помощью существующего документа Word можно создать пользовательские схемы. Схема RequisitePro является справочным документом и используется для управления форматированием документов Word в RequisitePro. Это удобно использовать для поддержки согласованности документов одного типа.

Схемы RequisitePro состоят из двух файлов, находящихся в каталоге outlines в установочной папке RequisitePro:

  • Файл схемы, имеющий расширение .def, является простым неформатированным текстовым файлом, содержащим имя схемы, описание и ссылку на шаблон Word.
  • Шаблон Word, имеющий расширение файла .dot, содержит стили абзацев и прочие сведения, такую как форматы, сведения о разметке страницы, атрибуты, определения атрибутов и системные значения по умолчанию, контролирующие вид документа RequisitePro.

Этапы работы с инструментами

Для добавления новой схемы к проекту RequisitePro выполните следующие действия:

  1. Создайте шаблон Microsoft Word
  2. Создайте файл схемы
  3. Добавьте схему в RequisitePro
  4. Создайте тип документа на базе схемы

1. Создайте шаблон Microsoft Word

  1. Запустите Microsoft Word вне RequisitePro.
  2. Откройте файл, который будет использован для создания шаблона Word.
  3. Измените при необходимости файл, чтобы подготовить содержимое для применения в качестве схемы.
  4. Сохраните файл как шаблон Word с расширением .dot. Дополнительные сведения по созданию шаблонов приведены в документации Microsoft Word.

2. Создайте файл схемы

Файл схемы создается в текстовом редакторе и сохраняется с расширением .def. Файл схемы является простым текстовым файлом без форматирования, содержащим имя схемы, описание схемы и ссылку на шаблон Word. Файлы .def и .dot, созданные в описанной выше процедуры, должны иметь одинаковое имя с разными расширениями. Пример: usecase.def , usecase.dot. Оба файла следует сохранить в каталоге outlines в папке установки RequisitePro или во вспомогательном каталоге outlines , указанном в RequisitePro, выбрав Инструменты > Схемы Опций/Каталогов/Документов.

  1. Откройте текстовый редактор, например, Блокнот.
  2. Создайте новый текстовый файл, содержащий следующие три строки:
    • Полное (логическое) имя схемы, не превышающее 64 символа. Убедитесь, что это логическое имя является уникальным в каталоге outlines .
    • Описание схемы, не превышающее в длину 256 символов.
    • Имя файла шаблона Word (с расширением .dot).
  3. Сохраните текстовый файл под тем же именем, что и шаблон, но с расширением .def.

Например, файл usecase.def содержит следующие строки:

Спецификация варианта RUP
Шаблон Rational Unified Process для указания варианта. rup_ucspec.dot

3. Добавьте схему в RequisitePro

Скопируйте файлы .dot / .def в каталог outlines в папке установки RequisitePro или во вспомогательный каталог outlines . Путем к каталогу outlines обычно является:C:\Program Files\Rational\RequisitePro\outlines. В вашем случае путь установки может отличаться в зависимости от установленных продуктов Rational.

При использовании вспомогательного каталога убедитесь, что путь указан в RequisitePro. Выберите Инструменты > Опции и введите в поле Схемы каталогов/документов путь и имя каталога для пользовательских схем.

4. Создайте тип документа на базе схемы

  1. В проводнике выберите проект и затем щелкните на опции Файл > Свойства. Откроется диалоговое окно Свойства проекта.
  2. Перейдите на вкладку Типы документов и нажмите кнопку Добавить. Откроется диалоговое окно Тип документа.
  3. В текстовом поле Имя введите имя типа документа (до 64 символов).
  4. В текстовом поле Расширение файла введите расширение из трех символов или, при поддержке длинных имен файлов, задайте более длинное расширение. (В RequisitePro поддерживаются расширения до 20 символов в длину). Расширение файлов применяется ко всем документам, связанным с данным типом документа.
  5. Выберите тип требований по умолчанию из списка или нажмите кнопку Создать для создания нового типа требований.
  6. Выберите схему из списка Имя схемы.
  7. Нажмите кнопку OK чтобы закрыть диалоговое окно Тип документа.
  8. Нажмите кнопку OK чтобы закрыть диалоговое окно Свойства проекта.

Дополнительная информация

Обратитесь к следующим разделам электронной справки RequisitePro:
  • Создание и изменение типов документов (Индекс: типы документов > создание)
  • Создание схем документов (Индекс: схемы > создание)
