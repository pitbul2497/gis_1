Руководство по инструменту: Создание контрольной версии проекта Rational RequisitePro
В этой памятке по инструменту описывается процесс создания контрольной версии требований с помощью Rational Administrator и Rational ClearCase.
Инструмент: Rational RequisitePro
Взаимосвязи
Связанные элементы
  • Настройка среды управления конфигурацией (CM)
  • Создание контрольных версий
Основное описание

Обзор

Рекомендуется создавать контрольную версию проекта требований для всех вех проекта. Контрольная версия создает объект ClearCase, представляющий стабильную конфигурацию проекта Rational RequisitePro® в определенный момент времени. Этот объект можно использовать для создания нового проекта.

Проект RequisitePro состоит из базы данных требований, панелей проектов и документов, содержащих требования. Требования можно создать непосредственно на панели проекта или в документе Microsoft® Word в RequisitePro. Все требования хранятся в базе данных, вне зависимости от их расположения в проекте. Можно создать контрольную версию любого проекта RequisitePro вне зависимости от того, использует ли он базу данных Oracle, Microsoft Access или SQL Server.

Примечание: Сведения об архивации проекта RequisitePro project приведены в памятке по инструменту Архивация требований с помощью RequisitePro.

Создание контрольной версии проекта с помощью Unified Change Management

С помощью модели Unified Change Management (UCM) администраторы проектов RequisitePro могут создавать контрольные версии проектов RequisitePro, выполняя задачи в Rational Administrator и ClearCase.

Перед созданием контрольной версии проекта RequisitePro вначале следует выполнить следующие предварительные условия:

  • Проект RequisitePro должен быть связан с поддерживающим UCM проектом Rational в Rational Administrator.
  • Проект RequisitePro должен быть закрыт, активных пользователей быть не должно.

Для создания контрольной версии проекта RequisitePro:

  1. Откройте Rational Administrator.
  2. Щелкните правой кнопкой мыши на проекте Administrator и выберите Настроить.
  3. В диалоговом окне Настроить проект в области Требуемые ресурсы выберите Пометить все.
  4. В появившемся сообщении подтвердите выбор и укажите действие для описания контрольной версии.
  5. После завершения операции Пометить все закройте окно Настроить проект.
  6. В меню Rational Administrator Инструменты выберите Обзор проектов Rational ClearCase. ОТкроется проводник ClearCase.
  7. Щелкните правой кнопкой мыши на потоке интеграции Требуемые ресурсы RequisitePro и выберите во всплывающем меню пункт Создать контрольную версию.
  8. Укажите название контрольной версии, выберите Полная в качестве типа контрольной версии, выберите Показать контекст и нажмите кнопку OK.
