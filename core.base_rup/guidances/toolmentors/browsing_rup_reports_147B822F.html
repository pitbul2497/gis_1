Руководство по инструменту: Добавление шаблонов Rational Unified Process к структуре навигации ProjectConsole
В этой памятке по инструменту описано добавление шаблонов Rational Unified Process в структуру навигации ProjectConsole.
Инструмент: Rational ProjectConsole
Взаимосвязи
Связанные элементы
  • Подготовка шаблонов для проекта
Основное описание

Обзор

ProjectConsole использует технологию аплетов Java для предоставления вам знакомых множественных форм навигации.  При щелчке на тексте в узле управления структурой отображается страница продукта работы, связанной с этим узлом.  Если развернуть узел управления структурой, щелкнув на значке плюса, будут показаны гиперссылки, отображаемые при щелчке на тексте узла, в виде дочерних узлов, которые можно использовать для дальнейшей навигации.

Эта памятка по инструменту применима при работе с Microsoft Internet Explorer (начиная с версии 5.5)  или Netscape (начиная с версии 7.0).

Этапы работы с инструментами

Для добавления шаблона Rational Unified Process к структуре навигации ProjectConsole:

  1. Запустите браузер.
  2. Введите URL установочного сайта ProjectConsole. Получить URL можно у администратора ProjectConsole. Появится окно входа в систему ProjectConsole.
  3. Введите допустимое ИД пользователя и пароль. Откроется обозреватель продуктов работы ProjectConsole.
  4. Выберите родительский узел, то есть узел, расположенный над добавляемым шаблоном Rational Unified Process. Например, для добавления дочернего узла к узлу Анализ и разработка разверните следующие узлы: ProjectConsole, Проекты ClassicsCD.com, Webshop, Версия 1.0, Дисциплины RUP и Анализ и разработка.
  5. Щелкните правой кнопкой на родительском узле и выберите Добавить во всплывающем меню. Например, щелкните правой кнопкой на узле Анализ и разработка и выберите Добавить.
  6. В правом кадре под пунктом Тип узла выберите Отчет.
  7. В поле Метка укажите имя отчета. Перейдите к столбцу Метки приведенной ниже таблицы. Нажмите кнопку Обзор рядом с полем Шаблон и перейдите к папке Rose, выберите нужный шаблон. Перейдите к столбцу Шаблоны в приведенной ниже таблиц и нажмите OK. Например, введите "Отчет класса процесса Rational Unified" в поле Метка и перейдите к шаблону "Rose/rup_class_list_for_class_report" для отображения отчета класса процесса Rational Unified.

    Метки Шаблоны
    Отчет субъекта процесса Rational Unified rup_actor_list
    Отчет бизнес-сущности процесса Rational Unified rup_class_list_for_business_entities
    Отчет модели бизнес-объекта процесса Rational Unified rup_business_object_model_survey
    Отчет модели бизнес-прецедента процесса Rational Unified rup_business_use_case_model_survey
    Отчет реализации бизнес-прецедента процесса Rational Unified rup_use_case_list_for_business_uc_realization
    Отчет бизнес-обработчика запросов процесса Rational Unified rup_class_list_for_business_workers
    Отчет класса процесса Rational Unified rup_class_list_for_class_report
    Отчет модели эскиза процесса Rational Unified rup_design_model_survey
    Отчет пакета процесса Rational Unified rup_package_list_for_package_report
    Документ архитектуры ПО процесса Rational Unified rup_software_architecture_document
    Отчет модели прецедента процесса Rational Unified rup_use_case_model_survey
    Отчет реализации прецедента процесса Rational Unified rup_use_case_list_for_uc_realization
    Отчет прецедента процесса Rational Unified rup_use_case_list_for_uc_report
    Отчет раскадровки пользовательского интерфейса процесса Rational Unified rup_use_case_list_for_user-experience_storyboard

  8. Нажмите кнопку Обзор рядом с полем Значок и перейдите к "/projectconsole/pjc/Icons/Rose/Model.gif", затем нажмите кнопку OK.
  9. В поле ИД продукта работы укажите путь к модели Rose. Для того чтобы отчеты отображали осмысленные данные, модель Rose должна быть создана согласно описанию процесса Rational Unified. Для применения простой модели Rose введите "Rose:Model:Path=${ARTIFACT_REPOSITORY}\Rose\ClassicsCDWorld.mdl".

    Примечание: Модель ClassicsCDWorld Rose не соответствует требованиям процесса Rational Unified.

  10. Нажмите кнопку Изменить рядом со списком Защита для добавления групп и/или пользователей, которые должны иметь доступ к отчету. Нажмите OK.
  11. Нажмите кнопку Сохранить.
  12. Структура навигации ProjectConsole теперь будет содержать узел нового отчета. Щелкните на узле для отображения отчета.
  13. Для продолжения добавления шаблонов Rational Unified Process к структуре навигации повторите шаги 4-12.

Примечание: Отчеты, показываемые ProjectConsole являются простыми шаблонами, созданными компоновщиком шаблонов Rational ProjectConsole, в сводной панели ProjectConsole создаются опубликованные диаграммы.
