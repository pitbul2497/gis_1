Руководство по инструменту: Захват общего словаря с помощью Rational RequisitePro
В этой памятке по инструменту описан процесс захвата общего словаря с помощью Rational RequisitePro.
Инструмент: Rational RequisitePro

Связанные элементы
  • Создание общего словаря
Основное описание

Обзор

Общий словарь создается с целью снижения разногласий в общении между членами коллектива и определить единый язык для использования при обсуждении создаваемой системы.

Глоссарий содержит определения терминов, используемых в описании создаваемой системы. Для каждого проекта должен быть создан глоссарий.

Этапы работы с инструментами

Для документирования глоссария проекта с помощью Rational RequisitePro® выполните следующие действия:

  1. Добавьте тип документа Глоссарий к проекту (при необходимости)
  2. Создайте документ Глоссарий
  3. Создайте в документе Глоссарий требования

1.  Добавьте тип документ Глоссарий в проект (при необходимости) 

Если документ Глоссарий недоступен для вашего проект, можно использовать электронный онлайн-глоссарий, предоставляемый в RequisitePro. При создании проекта RequisitePro с помощью одного из шаблонов проекта по умолчанию (Прецедент, Традиционный или Составной) в проекте уже будет содержаться документ Глоссарий в пакете Компоненты и видение, переходите к процедуре 3.

Для добавления типа документа Глоссарий в открытый проект RequisitePro выполните следующие действия:

  1. Щелкните на проекте на панели Проводник. Затем выберите Файл > Свойства. Откроется диалоговое окно Свойства проекта.
  2. Перейдите на вкладку Типы документов и выберите Добавить. Откроется диалоговое окно Тип документа.
  3. Укажите имя, описание и расширение файла для добавляемого типа документа. Расширение файлов применяется ко всем документам, связанным с данным типом документа. В качестве расширения файлов для документов глассария обычно используется GLS.
  4. В списке Типы требований по умолчанию выберите тип требований глоссария. Если он недоступен, нажмите кнопку Создать. Откроется диалоговое окно Тип требований.
    • Введите в поле Имя "Тип требований глоссария".
    • Введите "TERM" в поле Префикс тега требований.
    • Введите или измените значения прочих полей (необязательно).
    • Нажмите кнопку OK для возврата к диалоговому окну Тип документ.
  5. В поле Имя схемы выберите Глоссарий RUP (для разработки ПО) или Бизнес-глоссарий RUP (для бизнес-моделирования).
  6. Нажмите кнопку OK чтобы закрыть диалоговое окно Тип документа.
  7. Нажмите кнопку OK чтобы закрыть диалоговое окно Свойства проекта.

Дополнительная информация

Обратитесь к разделу Создание и изменение типов документов (Индекс: типы документов > создание) электронной справки RequisitePro.

2. Создайте документ Глоссарий

Документ Глоссарий содержит термины и определения, определяемые при выполнении действий проект и особенно при определении потребностей заинтересованных лиц. (См. Задача: Определение запросов заинтересованных лиц.)

Для создания документа Глоссарий выполните следующие действия:

  1. Щелкните на проекте в проводнике моделей и выберите Файл > Создать > Документ. Откроется диалоговое окно Свойства документа.
  2. В поле Имя введите "Глоссарий". (Таким образом вы будете ссылаться на документ Глоссарий в RequisitePro).
  3. В поле Описание задайте краткое описание.
  4. В поле Пакет примите значение по умолчанию или нажмите кнопку Обзор для перехода к пакету, в котором следует сохранить документ Глоссарий.
  5. В поле Имя файла укажите имя файла, используемое RequisitePro при сохранении документа Глоссарий.
  6. В поле Тип документа выберите тип глоссария, созданного в процедуре 1.
  7. Нажмите кнопку OK чтобы закрыть диалоговое окно Свойства документа. RequisitePro откроет новый созданный документ Глоссарий в Microsoft® Word.
  8. В документе Глоссарий добавьте термины и описания. Для часто используемых терминов с одним определением можно применять псевдонимы (например, Оператор, см. Машинист).
  9. Выберите RequisitePro > Документ > Сохранить для сохранения документа Глоссарий.

Дополнительная информация

Обратитесь к разделу Создание документов требований (Индекс: документ>создание) электронной справки RequisitePro.

3. Создайте требования в документе Глоссарий

Глоссарий не содержит требований по умолчанию, однако, можно пометить некоторые термины глоссария как требования, чтобы отслеживать их изменения и иметь возможность обновлять документы при необходимости для отражения этих изменений. Пометив эти термины как требования вы сможете задавать ссылки трассироуемости между терминами глоссария и компонентами продукт а или системными требованиями, которые требуется обновить при изменении определения термина глоссария.

Для создания требований в документе глоссария выполните следующие действия:

  1. Выберите (выделите) текст термина глоссария.
  2. Выполните одно из следующих действий:
    • Щелкните правой кнопкой и выберите Создать требование.
    • Выберите RequisitePro > Требование > Создать.
    Откроется диалоговое окно Свойства требования.
  3. Выберите Тип требования глоссария (TERM) в качестве требования и нажмите OK чтобы закрыть диалоговое окно Свойства требования.
  4. Выберите RequisitePro > Документ > Сохранить.

RequisitePro сохранит документ, обновит базу данных и присвоит номер требования к требованию (вместо тега Приостановить).

Дополнительная информация

Обратитесь к разделу Создание требований в документе (Индекс: требования>создание) электронной справки RequisitePro.
