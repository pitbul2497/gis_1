Руководство по инструменту: Настройка проектов с помощью Rational Administrator
В этой памятке по инструменту описывается применение Rational Administrator для настройки проектов Administrator.
Инструмент: Rational Administrator
Взаимосвязи
Связанные элементы
  • Разработка справочных материалов
Основное описание

Обзор

В Administrator можно создавать проекты Administrator, в которых хранятся сведения о тестировании ПО и разработке. Все компоненты Rational на вашем компьютере обновляют и извлекают данные из одного проекта. Типы данных в проекте Administrator зависят от установленного ПО Rational.  

Сведения о проектах Administrator приведены в следующих разделах электронной справки Administrator:

  • Что такое проект Rational
  • Компоненты проекта Rational
  • Преимущества проекта Rational

С помощью Administrator проект Administrator можно также включить для UCM (Унифицированное управление изменениями), являющегося подходом Rational Software для управления изменения в требованиях, моделях разработки, документации, компонентах, тестах, исходном коде и т.п.  

Дополнительные сведения о UCM приведены в следующих разделах электронной справки Administrator:

  • Что такое UCM?
  • Что такое UCM относительно Rational Administrator
  • Определение необходимости сделать проект доступным для UCM

Перед настройкой проекта следует создать его. Перед созданием проекта Administrator следует ответить на ключевые вопросы. Если вы планируете сделать проект доступным для UCM, вам также потребуется настроить Rational ClearCase.  

Дополнительные сведения о действиях по подготовке и созданию проекта Administrator приведены в следующих разделах электроннй справки Administrator: 

  • Перед тем, как создать проект
  • Действия по настройке ClearCase
  • Создание проекта Rational Administrator

При необходимости использовать существующие хранилища, можно преобразовать их в проекты Administrator, чтобы о ни стали доступны для текущей версии Administrator.  

Дополнительная информация л преобразовании приведена в следующих разделах электронной справки Administrator: 

  • Обновление хранилища SQA 6.x
  • Обновление хранилища Rational Test 7.x

Эта памятка по инстурменту применима для систем с Windows 98/2000/NT 4.0.

Для настройки проекта Administrator выполните следующие действия:

  1. Определите, какие структуры данных Rational вы хотите добавить в проект
  2. Определите, следует ли связать проект Administrator с проектом UCM (если вы еще не сделали этого)
  3. Определите, следует ли вызывать проект Rational RequisitePro или хранилище данных Rational Test при данном управлении конфигурацией (если вы еще не сделали этого)

1.   Определите какие структуры данных Rational следует добавить в проект

Можно связать различные структуры данных с проектом Administrator. Структурами данных являются:

  • Хранилища данных Rational Test
  • Проекты Rational RequisitePro
  • Базы данных Rational ClearQuest
  • Файлы моделей Rational Rose

Создать структуры данных можно в Administrator, также можно выбрать существующие структуры данных, за исключением файлов моделей Rose. В Administrator файлы моделей Rose можно только добавлять, но не создавать.

  Обратитесь к разделу Настройка проекта Rational Administrator электронной справки Administrator.

2.   Определите, следует ли связать проект Administrator с проектом UCM

После создания проекта Administrator можно сделать его доступным для UCM, но только если в системе установлен Rational ClearCase. Также можно сделать доступными для UCM ресурсы требований и тестов в проекте, в качестве выполнения изменения свойств проекта.

Обратите внимание, что отключить UCM для проекта, содержащего хранилища данных под управлением UCM, нельзя. Это означает, что нельзя изменить состояние проекта с UCM на проект без UCM, если только хранилища данных тестов и требований не используют UCM.

   Обратитесь к разделу Взаимосвязь проекта Rational Administrator с проектом UCM электронной справки Administrator.

3.   Определите, следует ли вызывать проект Rational RequisitePro или хранилище данных Rational Test при данном управлении конфигурацией

Если вы уже создали взаимосвязь между проектом RequisitePro или хранилищем данных тестов и проектом с UCM, но не вызвали их в текущем управлении конфигурацией, вы можете сделать это.

Также, для любого проекта UCM, вязанного с проектом Administrator, будет включена поддержка управления запросов на изменение если вы выберите базу данных ClearQuest в диалоговом окне Настройка проектов.

Однако, если база данных ClearQuest уже связана с проектом UCM, вы не сможете выбрать или создать другую базу данных ClearQuest для связи с проектом Administrator. Для создания или изменения базы данных ClearQuest, связанной с проектом Administrator, следует создать новый проект. База данных ClearQuest предопределяется существующими взаимосвязями с проектом UCM.

В проекте Administrator с UCM можно отменить взаимосвязь проекта RequisitePro или хранилища данных тестов, находящихся под управлением конфигурацией. Обратите внимание, что при удалении проекта RequisitePro или хранилища данных тестов из управления конфигурацией происходит следующее:

  • Копируются все файлы проекта RequisitePro и хранилища данных тестов в неуправляемую версиями область
  • Создается взаимосвязь проекта с неуправляемой версиями областью

   Обратитесь к разделу Настройка проекта Rational Administrator электронной справки Administrator.
