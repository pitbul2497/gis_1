Руководство по инструменту: Создание реализаций прецедентов с помощью Rational Rose
В этой памятке по инструменту описывается процесс представления реализации прецедентов в Rational Rose.
Инструмент: Rational Rose

Связанные элементы

  • Архитектурный анализ

Основное описание

Обзор

Ниже приведен обзор действий, выполняемых для создания Продукта работы: реализации прецедентов:

  1. Создайте пакет Реализация прецедентов в соответствующей модели

  2. Создайте реализацию прецедентов
  3. Создайте трассируемость между вариантом и его реализацией

1. Создайте пакет Реализация прецедента в соответствующей модели

Реализации прецедентов можно создать в Продукте работы: модель анализа или в Продукте работы: модель разработки, либо в них обоих. Эти продукты работы представлены в виде пакетов в Логической панели модели. Обратитесь к Памятке по инструменту: захват результатов анализа прецедентов и к Памятке по инструменту: управление моделью разработки для получения сведений о создании этих пакетов. При создании пакета переименуйте его в "Реализацию прецедентов". При создании новой диаграммы классов для пакета переименуйте ее в "Трассируемости".

2. Создайте реализацию прецедента

Реализация вариантов представляет проекцию разработки варианта. Это элемент организационной модели, используемый для группировки определенного числа продуктов работы, связанных с разработкой варианта.Варианты отделены от реализаций вариантов, поэтому вы можете управлять ими по отдельности и изменять эскиз вариантов не влияя на вариант контрольной версии.Для каждого варианта в модели вариантов имеется реализация варианта в модели разработки с зависимостью (стереотипно <<реализованной>>) от варианта.

Для создания реализации варианта нужно создать и присвоить имя новому варианту в пакете Панель вариантов и затем переместить его в пакет "Реализация варианта".С помощью спецификации вариантов присвойте стереотип <<реализация варианта>> варианту.В открывшемся диалоговом окне, сообщающем о наличии варианта в двух пространствах имен, нажмите кнопку OK.В созданном пакете реализаций прецедентов создайте пакет для управления реализацией прецедентов, присвоив ему тое же имя, что и реализации прецедентов. См. Памятка по инструменту: управление моделью разработки для получения сведений о создании и именовании пакетов. В браузере переместите реализацию прецедента в этот новый созданный пакет. Реализация прецедентов теперь будет присутствовать в модели разработки или анализа, в собственном пакете, вместе с другими реализациями прецедентов. Наличие отдельных пакетов для реализаций прецедентов делает возможным независимое управление и поддержку версий продуктов работы.

3. Создайте трассируемость между вариантом и его реализацией варианта

Для создания трассируемости между вариантом и его реализацией варианта можно переместить вариант из "Панели вариантов" и поместить его на диаграмме трассируемостей в пакете реализаций вариантов.Переместите реализацию варианта из пакета реализаций варианта на диаграмму трассируемостей, поместив его рядом с вариантом.Затем с помощью инструмента ассоциаций из панели инструментов диаграммы начертите ассоциацию между реализацией варианта и вариантом.Откройте спецификацию ассоциации и назначьте ей стереотип <<реализовать>>.
