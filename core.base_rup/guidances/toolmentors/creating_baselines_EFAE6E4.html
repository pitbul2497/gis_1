Руководство по инструменту: Создание контрольных версий с помощью Rational ClearCase
В этой памятке описывается процедура обеспечения безопасности ресурсов проекта и возможности их воссоздания при необходимости с помощью соответствующих приемов контролирования версий.
Инструмент: Rational ClearCase
Взаимосвязи
Связанные элементы
  • Реализация стратегий управления конфигурацией (CM)
  • Создание контрольных версий
Основное описание

Обзор

На следующей диаграмме показан поток операций для управления проектами UCM. Затемненные области описаны в этой памятке.

В Rational ClearCase UCM контрольная версия является объектом, обычно представляющим стабильную конфигурацию компонента. Контрольная версия идентифицирует действия и одну версию каждого элемента компонента, выполняя роль версии компонента.

По мере того, как разработчики доставляют работу в поток интеграции, руководители проекта создают новые контрольные версии для интеграции проекта или общие рабочие области, учитывающие внесенные изменения. Разработчики затем могут перейти к новым контрольным версиям и учитывать при этом новые изменения проекта.

Эта памятка по инструменту применима к Microsoft Windows.

Терминология

Типы контрольных версий

Дополняющая контрольная версия является контрольной версией, создаваемой ClearCase с помощью записи последней полной контрольной версии и тех версий, которые были изменены после создания последней полной контрольной версии.

Полная контрольная версия является контрольной версией, создаваемой ClearCase с помощью записи всех версий в корневом каталоге компонента. Обычно для создания дополняющей контрольной версии требуется меньше времени. Однако поиск информации в полной контрольной версии ClearCase выполняет быстрее.

Этапы работы с инструментом

Выполните следующие действия для создания контрольных версий:

  • Заблокируйте поток интеграции проекта
  • Запустите диалоговое окно создания контрольной версии
  • Укажите описание контрольной версии
  • Разблокируйте поток интеграции

1. Заблокируйте поток интеграции проекта

В заблокированный поток интеграции добавить новую работу нельзя, что гарантирует стабильную конфигурацию для создания контрольной версии.

  1. На панели инструментов Windows выберите Пуск > Программы > Rational Software > Rational ClearCase > Project Explorer.
  2. В Project Explorer найдите и выберите поток интеграции проекта.
  3. Выберите Файл > Свойства для отображения страницы свойств потока интеграции.
  4. Перейдите на вкладку Блокировка.
  5. Выберите Заблокировано и нажмите кнопку OK.   

2. Откройте окно Создать контрольную версию

  1. В Project Explorer найдите и выберите поток интеграции проекта.
  2. Выберите Инструменты > Создать контрольную версию.  Появится окно Создать контрольную версию.

3. Укажите описание контрольной версии

Это описание содержит имя контрольной версии, тип создаваемой контрольной версии, компоненты для которых создается контрольная версия, сведения о панели и потоке.

  1. Введите имя в поле Имя контрольной версии. По умолчанию ClearCase называет контрольную версию, добавляя дату к имени проекта.
  2. Выберите тип создаваемой контрольной версии: дополняющая или полная.
  3. Выберите контекст панели для контрольной версии, указав одну из панелей интеграции проекта, панель, подключенную к потоку интеграции проекта.
  4. Укажите компоненты, для которых создается контрольная версия. ClearCase автоматически добавит уникальный идентификатор ко всем контрольным версиям, чтобы облегчить различие версий, связанных с отдельными компонентами.

  Дополнительные сведения приведены в разделе Обозреватель структуры компонентов ClearCase электронной справки ClearCase.

4. Разблокировать поток интеграции

  1. В Project Explorer найдите и выберите поток интеграции проекта.
  2. Выберите Файл > Свойства для отображения страницы свойств потока интеграции.
  3. Перейдите на вкладку Блокировка.
  4. Нажмите Разблокировать, затем нажмите кнопку OK.
