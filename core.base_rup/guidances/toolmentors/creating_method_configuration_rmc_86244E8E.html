Руководство по инструменту: Создание конфигурации метода с помощью Rational Method Composer
В этой памятке по инструменту описывается создание конфигурации метода с помощью Rational Method Composer.
Инструмент: Rational Method Composer
Взаимосвязи
Связанные элементы
  • Настройка процесса разработки для проекта
  • Подготовка рекомендаций по проекту
  • Подготовка шаблонов для проекта
Основное описание

Конфигурация метода является пользовательским подмножеством Библиотеки методов , хранящимся в Rational Method Composer (RMC).

Создание конфигурации с помощью RMC состоит из принятия следующих решений:

  • Выбор подходящих пакетов метода в структуре RUP и необходимых встраиваемых модулей.
  • Отмена выбора/удаление ненужных пакетов метода.
  • Определение панелей в конфигурации для поддержки различных проекций заинтересованных лиц для конфигурации. Эти панели будут управлять навигацией по опубликованным конфигурациям.

RMC предоставляет набор предустановленных конфигураций RUP для определенных контекстов проекта. Перед созданием новой конфигурации рекомендуется использовать одну из предустановленных конфигураций в RMC в качестве исходной точки. Выберите и скопируйте предустановленную конфигурацию, наиболее схожую с характеристиками проекта, и настройте ее дополнительно, выбрав нужные пакеты метода. Элементы метода, представленные в RMC, имеют описания, содержащие сведения о содержимом, а также рекомендации по поводу того, почему вы должны включить их в свою конфигурацию. Существующие конфигурации хранятся в пакете "Конфигурации" в библиотеке метода.  Сведения о перемещении по библиотеке методов приведены в Памятке по инструменту: навигация по содержимому метода с помощью Rational Method Composer.

Помимо настройки содержимого, добавляемого в конфигурацию, можно также задать панели в содержимом, используемом для подавления нежелательных элементов методов для определенных команд проекта. Например, панели, основанные на ролях (разработчики, к примеру, необязательно хотят просматривать ту же информацию, что и руководитель проекта).

Конфигурации методов используются не только для создания процессов (см. Разработка процессов с помощью Rational Method Composer), но также используются для публикации (то есть конфигурация определяет, какие элементы будут опубликованы в HTML, а какие нет). При публикации конфигурации получившийся Web-сайт RUP содержит только выбранные элементы, а панели представлены в виде отдельных экземпляров (или вкладок) обозревателя структуры.  Дополнительные сведения приведены в Памятке по инструменту: публикация конфигурации метода с помощью Rational Method Composer.

Дополнительная информация о создании конфигурации метода с помощью  RMC приведена в памятке "Создание собственной конфигурации методов".  Все памятки RMC доступны через меню Справка>Памятка.
