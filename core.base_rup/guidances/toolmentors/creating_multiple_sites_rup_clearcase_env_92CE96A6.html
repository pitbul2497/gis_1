Основное описание

Чем может быть полезен ClearCase MultiSite для вашей компании?

Продукт ClearCase MultiSite является дополнением к ClearCase, предназначенным для распределение процесса разработки ПО между разработчиками, находящимися в различных географических регионах. Например, если сотрудники вашей компании находятся в Бостоне, Сан-Франциско и Бангалоре, вы сможете предоставить каждому разработчику локальный доступ в реальном времени к хранилищу исходных кодов (база объектов с управлением версий, VOB). С помощью MultiSite каждый сайт может иметь собственную копию (реплику) VOB и все разработчики могут работать параллельно с исходными файлами без возникновения конфликтов.  

Конфликтующие изменения отслеживаются и устраняются схемой участников ClearCase MultiSite. Для большинства объектов ClearCase достаточно одной копии, изменять объект могут лишь разработчики, использующие эту копию. MultiSite позволяет использовать общее управление элементами исходного кода, чтобы над одним файлом с исходным кодом могли работать сразу несколько разработчиков по очереди.  

Вместо "перебрасывания изменений через стену" с помощью инструментов синхронизации, поставляемых с ClearCase MultiSite можно отправлять изменения между копиями так часто, как это позволяет соединение между сайтами. При наличии IP-соединений можно использовать механизмы отправки и настраиваемые сценарии MultiSite для синхронизации копий автоматически с заданной высокой частотой, предоставляя разработчикам других сайтов доступ ко все последним изменениям, внесенным на вашем сайте. При отсутствии IP-соединений можно выбрать другой способ синхронизации (например, электронную почту, ftp или накопители на магнитной ленте), подходящий для вашей конфигурации.  

Нужен ли вам ClearCase MultiSite?

Применение ClearCase и ClearCase MultiSite целесообразно при распределении работы между разработчиками, находящимися в различных географических местах, так как эти продукты облегчают удаленный доступ к VOB. Кроме того, дополнительным преимуществом MultiSite является возможность применения продукта в качестве механизма резервного копирования для создания копии в удаленном расположении и ее синхронизации с основной копией. Применяя резервное копирование вспомогательной копии вместо основной вы обеспечиваете продолжение разработки в основной копии без каких-либо помех. Также в случае внезапного сбоя жесткого диска хоста основной копии вы сможете воспользоваться резервной копией и избежать простоя.
