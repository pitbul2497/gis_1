Руководство по инструменту: Создание тестового плана с помощью Rational TestManager
В этой памятке по инструменту описано применение Rational TestManager для создания тестового плана.
Инструмент: Rational TestManager
Взаимосвязи
Связанные элементы
  • Определение конфигурации среды тестирования
  • Определение мотивационных факторов тестирования
  • Определение подхода к тестированию
  • Определение цели теста
Основное описание

Обзор

Эта памятка по инструменту применима к Windows 98/ME/XP/2000 и NT 4.0.

Тестовый план является дополняемым набором сведений , организованным TestManager. Он представляет соглашение и обзор объектов тестирования с информацией о времени их тестирования. Проект может иметь несколько планов тестирования, представляющих различные этапы или аспекты теста. Рекомендуется организовывать тестовые планы по наборам функций, функциональности и типам тестирования.

Каждый план тестирования может содержать несколько папок тестовых сценариев и самих тестовых сценариев. Тестовый план задает все ключевые элементы 
тестирования, представленные иерархией папок с тестовыми сценариями и тестовыми сценариями.  Дополнительная информация по созданию тестовых сценариев приведена в Памятке по инструменту: создание тестового сценария с помощью Rational TestManager.

Этапы работы с инструментами

Для создания тестового плана выполните следующие действия:

  1. Создайте план тестирования
  2. Задайте свойства плана тестирования

1. Создайте план тестирования

  1. На вкладке Планирование рабочей области тестовых ресурсов щелкните правой кнопкой мыши на опции План тестирования и выберите Создать план тестирования.
    Единственным обязательным полем является имя тестового плана. Задайте имя плана, исходя из типа тестирования, например, функциональный тест.  
  2. В поле Имя введите соответствующее имя тестового плана. Нажмите OK.

   Обратитесь к следующим разделами электронной справки TestManager:

  • Планирование теста (Индекс: планирование теста)
  • Планы тестирования (Индекс: план тестирования > создание)

2. Задайте свойства плана тестирования

Помимо имени плана тестирования можно также присвоить и другие свойства по мере разработки плана тестирования. Например, можно назначить владельца плана тестирования, задать указатели на внешние документы и указать конфигурации и итерации, связанные с планом тестирования. Планирование тестирования дорабатывается с течением времени. План тестирования является дополняемым ресурсом, определяемым итерационно. Тестовый план обладает массой свойств:
  • Описание плана тестирования - в этом поле можно описать область плана тестирования, например, "Касается всех сценариев основного использования".  
  • Владелец плана тестирования - присвойте плану тестирования владельца для структуризации команды и гарантии обновления тестовых планов при обновлении элементов.
  • Конфигурации, связанные с планом тестирования - укажите конфигурации программного и аппаратного обеспечения в которых необходимо выполнять планы тестирования для проверки правильного поведения целевого объекта тестирования.
  • Итерации, связанные с планом тестирования - присвойте вехи жизненного цикла проекта , задающие время выполнения определенных фрагментов (наборов тестовых сценариев) плана тестирования. Так вы сможете изучить расписание в планах тестирования.
  • Внешние документы, связанные с планами тестирования - внешние документы могут содержать перечисления следующих элементов:
    • ограничения ресурсов по времени, бюджету, телам и наборам навыков
    • стратегии тестирования, определяющие выполняемые типы тестирования - функция, производительность, надежность
    • реализация стратегии тестирования, включая автоматическое и ручное тестирование, а также текущий набор инструментов
    • область проекта
    • продолжительность проекта
    • ожидаемое расписание
    • определение качества проекта
    • результаты
    • определение показателей проекта

   Обратитесь к следующим разделами электронной справки TestManager:

  • Сопоставление конфигурации с планом тестирования (Содержание: планирование тестов)
  • Сопоставление итерации с планом тестирования (Содержимое:планирование тестов)
  • Разработка тестов (Содержимое: планирование тестов)
