Обзор 

Шаги, описанные в этой памятке по инструменту, соответствуют этапам выполнения задачи. Ссылки на разделы электронной справки Rational XDE (TM) помечены значком .

В этой памятке по инструменту подразумевается, что модель прецедента была создана согласно Рекомендациям по структуре модели XDE. Кроме того предполагается, что вариант был создан следующими действиями, описанными в Памятке по инструменту: поиск субъектов и вариантов с помощью Rational XDE.

В этой памятке по инструменту выполняются следующие шаги:

  • Описание потока событий в варианте
  • Структура потока событий варианта
  • Демонстрация взаимосвязей с помощью субъектов и прочих вариантов <необязательно>
  • Опишите особые требования варианта
  • Опишите протоколы связи
  • Опишите предварительные условия варианта <необязательно>
  • Опишите выходные условия варианта <необязательно>
  • Опишите точки расширения <необязательно>
  • Вычисление результатов

Перед описанием варианта необходимо решить, как эти сведения будут записаны. Доступные следующие варианты:

  • С помощью окна документирования модели можно записать любой выбранный элемент модели. См. Документирование элементов модели
  • Можно создать независимый документ спецификации варианта с помощью предоставляемого в RUP шаблона (см. Продукт работы: Вариант для шаблонов). Затем документ можно связать с элементом модели прецедента. См. Привязка внешних файлов к элементам модели ;
  • Также можно связать варианты с требованиями и документам с помощью интеграции Rational RequisitePro-XDE Integration. Для получения сведений о связывании варианта с требованием или документом RequisitePro обратитесь к справке интеграции RequisitePro-XDE. Для ее вызова выберите в строке меню пункт Инструменты > Rational RequisitePro > Справка Integration. Более подробные сведения об интеграции RequisitePro-XDE приведены ниже.

Интеграция Rational RequisitePro-XDE

С помощью интеграции Rational RequisitePro-XDE можно изменять элементы моделей прецедентов из связанных с ними требований в базах данных и документах RequisitePro. Можно связать варианты с документами требований в RequisitePro, после чего можно будет разрабатывать текстовые определения варианта в документе RequisitePro Microsoft Word.В документах вариантов RequisitePro можно разработать варианты с описаниями, потоками событий, особыми требованиями и условиями.

Также интеграция позволяет присвоить атрибуты требований, такие как трассируемость, элементам модели. Связанные требования можно использовать для отслеживания взаимосвязей между элементами моделей и прочими требованиями с помощью функций трассируемости требований RequisitePro. Связывание элементов модели с требованиями позволяет просматривать и оценивать воздействие изменений требований на элементы модели.

Для применения интеграции необходимо установить RequisitePro и применить профайл RequisitePro к модели XDE. Профайл RequisitePro применяется автоматически к модели, создаваемой с помощью шаблона модели варианта. Для существующих моделей необходимо применить профайл вручную. См. Применение профайлов к модели .

Описание потока событий в варианте

  1. Откройте модель прецедентов.
  2. Перейдите к описываемому варианту.
  3. Введите подробные поток операций в документации, связанной с элементом модели.

Структура потока событий варианта

Можно продемонстрировать структуру потока событий с помощью диаграммы операций. Для этого выполните следующие действия:

  1. Щелкните правой кнопкой мыши на обозревателе моделей и добавьте диаграмму операций. См. Добавление диаграмм .
  2. (необязательно) Добавьте дорожки. См. Добавление форм к диаграммам .
  3. Добавьте исходное состояние, окончательно состояние и операции. См. Добавление форм к диаграммам .
  4. Добавьте для каждой операции краткое описание. См. Документирование элементов модели .
  5. Подключите состояния к переходам. См. Добавление взаимосвязей переходов .
  6. Создайте линии синхронизации (при необходимости). См. Добавление форм к диаграммам .
  7. Укажите сторожевые условия для переходов (при необходимости). См. Определение сторожевых условий во взаимосвязях переходов .
