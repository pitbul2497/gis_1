Руководство по инструменту: Разработка видения с помощью Rational RequisitePro
В этой памятке по инструменту описывается применение Rational RequisitePro® для документирования видения проекта. Документ видение является общим оператором основных требований проекта и предоставляет договорную основу для технических требований.
Инструмент: Rational RequisitePro
Взаимосвязи
Связанные элементы
  • Разработка видения
Основное описание

Обзор

Документ видение предоставляет основу высокого уровня, и иногда контрактную, для технических требований. Также может существовать и формальная спецификация требований. В видении содержатся требования и ограничения разработки высокого уровня для предоставления читателю возможности понять разрабатываемую систему. Он сообщает относящиеся к проекту "что и почему" и является мерой всех будущих решений.

Этапы работы с инструментами

Для документирования видения с помощью RequisitePro выполните следующие действия:

  1. Создайте документ Видение
  2. Создайте в документе Видение требования

1. Создайте документ Видение

Документ Видение содержит компоненты продукта, определенные в Задаче: оценка запросов заинтересованных лиц. При создании проекта RequisitePro с помощью одного из шаблонов по умолчанию (Прецедент, Обычный, Составной) в проектах уже содержится документ Видение в пакете Компоненты и Видение. Если в проекте отсутствует документ Видение, выполните следующие действия для его создания.

Для создания документа требований Видение выполните следующие действия:

  1. Откройте проект RequisitePro.
  2. В обозревателе выберите пакет, в который следует поместить документ Видение и затем выберите Файл > Создать > Документ. Откроется диалоговое окно Свойства документа.
  3. Введите имя, описание и имя файла для документа.
  4. В списке Тип документа выберите "Тип документа Видение". Нажмите OK. В Microsoft® Word откроется схема документа Видение.
  5. Замените информационный текст по умолчанию на относящиеся к проекту сведения.
  6. Выберите RequisitePro > Документ > Сохранить.

Дополнительная информация

Обратитесь к следующим разделам электронной справки RequisitePro:

  • Создание проекта RequisitePro (Индекс: проекты>создание)
  • Создание документов требований (Индекс: документы>создание)

2. Создайте требования в документе Видение

Создайте требования из компонентов продукта, обозначенных в разделе Компоненты продукта документа Видение. Это позволяет управлять атрибутами требований в матрице атрибутов и помогает отслеживать ссылки трассируемости между компонентами продукта и требованиями к ПО.

Для создания требований в документе Видение выполните следующие действия:

  1. В разделе Компоненты продукта документа Видение выберите (выделите) текст, определяющий требование.
  2. Выполните одно из следующих действий:
    • Щелкните правой кнопкой и выберите Создать требование.
    • Выберите RequisitePro > Требование > Создать.
      Откроется диалоговое окно Свойства требования.
  3. На вкладке Общие примите тип требований компонента (FEAT) и введите имя для него.
  4. Перейдите на вкладку Атрибуты и задайте атрибут Приоритет равным одному из предоставленных значений (Обязательно, Рекомендуется, Возможно, Ненужное). Рекомендуется записывать приоритеты заинтересованных лиц при оценке их запросов требований. (См. Задача: Определение запросов заинтересованных лиц.)
  5. Выберите атрибут Сведения о создании и выберите запись, представляющую отправителя определенного компонента. Это поможет отследить отправителей запросов к определенным компонентам продукта, чтобы вы знали к кому обратиться при возникновении необходимости прояснить или обсудить запрос.
  6. Нажмите OK.
  7. Можно повторить эти действия для каждого компонента продукта, обозначенного в разделе Компоненты продукта документа Видение. Завершив выберите в меню RequisitePro > Документ > Сохранить.

Для просмотра атрибутов и требований выполните следующие действия:

  1. В окне RequisitePro выберите пакет, в котором требуется создать панель.
  2. Выберите Файл > Создать > Панель. Откроется диалоговое окно Свойства панели.
  3. Введите имя панели, выберите тип панели Матрица атрибутов и выберите тип требований FEAT, использованный в документе Видение.
  4. Нажмите OK. Просмотрите требования к компоненту и значения их атрибутов.

Дополнительная информация

Обратитесь к следующим разделам электронной справки RequisitePro:

  • Создание требований в документе (Индекс: требования > создание)
  • Присвоение значений атрибутам требований (Индекс: атрибут требований > значения, присвоение)
  • Создание панели (Индекс: панели > создание панелей)
