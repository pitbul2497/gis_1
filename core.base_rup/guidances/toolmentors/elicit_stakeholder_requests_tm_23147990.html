Руководство по инструменту: Оценка запросов заинтересованных лиц в Rational RequisitePro
В этой памятке описывается применение Rational RequisitePro® для оценки запросов заинтересованных лиц.
Инструмент: Rational RequisitePro
Взаимосвязи
Связанные элементы

Основное описание

Обзор

Целью оценки запросов заинтересованных лиц является сбор ввода от лиц или представителей организаций, заинтересованных в результатах проекта. Заинтересованными лицами могут быть пользователи, обслуживающий персонал предлагаемых систем, представители технической поддержки, акционеры и т.п.

Запрос заинтересованных лиц для предлагаемой системы можно собрать с помощью различных приемов оценки, включая интервью, вопросники, совещания по требованиям, ролевые игры и т.п. (См. описание этапа Сбор информации в описании Задачи: оценка запросов заинтересованных лиц для получения полного списка).  

Вне зависимости от выбранного способа оценки, необходимо записывать результаты в одном месте, например в базе данных Rational ClearQuest® или в документе требований RequisitePro для их использования при определении системных и бизнес-требований. (Сведения об интеграции ClearQuest и RequisitePro приведены в разделе Управление запросами заинтересованных лиц с помощью Rational ClearQuest и Rational RequisitePro). Вы можете добавить документы оценки в проект RequisitePro, содержащий системные требования, или создать отдельный проект RequisitePro только для документов оценки и затем использовать трассируемость между проектами для связывания результатов оценки с проектом RequisitePro системных требований.

Методы сбора запросов заинтересованных лиц могут отличаться для разных проектов, в зависимости от знакомства разработчика и заказчиков с системным доменом и предлагаемой функциональностью. В качестве исходной точки RequisitePro предоставляет пример схемы запросов заинтересованных лиц и предустановленный тип требований (STRQ: запрос заинтересованных лиц) для облегчения сбора запросов заинтересованных лиц. Вам остается лишь просто изменить предоставленную схему запросов заинтересованных лиц и создать дополнительные схемы RequisitePro при необходимости для сбора дополнительного ввода. Вам может потребоваться создать схему RequisitePro для каждого приема сбора информации: одну для результатов работы с вопросником, одну для раскадровки, и т.п., либо вы можете записывать эти результаты в одном документе.

Этапы работы с инструментами

Так как сбор запросов заинтересованных лиц может различаться между проектами, мы предлагаем инструкцию по изменению существующей схемы запросов заинтересованных лиц RequisitePro или по созданию абсолютно новой схемы, отвечающей потребностям проекта. Начните с работы с существующей схемой запросов заинтересованных лиц.

  1. Добавьте тип документа Запросы заинтересованных лиц в проект
  2. Измените схему Запросы заинтересованных лиц
  3. Создайте документ запросов заинтересованных лиц
  4. Создайте требования в документе запросов заинтересованных лиц (необязательно)

1.  Добавьте тип документа Запросы заинтересованных лиц в проект 

Для применения шаблона Запросы заинтересованных лиц, предоставляемого в RequisitePro, необходимо добавить тип документа Запросы заинтересованных лиц в проект.

Примечание: чтобы проверить, содержит ли ваш проект тип документа Запросы заинтересованных лиц, выберите проект в обозревателе и выберите Файл > Свойства, перейдите на вкладку Типы документов. Если этот тип документа указан, можете переходить к шагу 3. Создайте документ запросов заинтересованных лиц.

Для добавления типа документа Запросы заинтересованных лиц в открытый проект RequisitePro выполните следующие действия:

  1. Выберите Файл > Свойства. Откроется диалоговое окно Свойства проекта.
  2. Перейдите на вкладку Типы документов и выберите Добавить. Откроется диалоговое окно Тип документа.
  3. Введите имя документа, описание и расширение файла. Расширение файлов применяется ко всем документам, связанным с данным типом документа.
  4. В списке Типы требований по умолчанию выберите "Тип требований запросов заинтересованных лиц". Если тип отсутствует, щелкните на кнопке Создать, в диалоговом окне Тип требований выберите "Тип требований запроса" в поле Имя, заполните остальные поля и нажмите кнопку OK для возврата к диалоговому окну Тип документа.
  5. В списке Имя схемы выберите схему "Запросы заинтересованных лиц RUP".
  6. Нажмите кнопку OK чтобы закрыть диалоговое окно Тип документа.
  7. Нажмите кнопку OK чтобы закрыть диалоговое окно Свойства проекта.

Дополнительная информация

Обратитесь к разделу Создание и изменение типов документа (Индекс: типы документов > создание) в электронной справке RequisitePro.

2.  Измените схему запросов заинтересованных лиц

Схему запросов заинтересованных лиц RequisitePro можно изменить для определенных аспектов проекта. Измените схему RUP_stkreq.dot в Microsoft® Word (независимо от RequisitePro). Затем можно использовать настроенную схему для создания нового документа запросов заинтересованных лиц.

Для изменения предоставленной схемы запросов заинтересованных лиц выполните следующие действия:

  1. Создайте копию файла шаблона Word "RUP_stkreq.dot", расположенного в каталоге Схемы в установочной папке RequisitePro. (При этом исходная схема будет сохранена для использования в будущем). Задайте копии имя для ее идентификации как исходного файла (например, "Stakeholder_orig.dot").
  2. Откройте файл шаблона "RUP_stkreq.dot".
  3. Измените этот файл чтобы он относился к проекту.
  4. Выберите Файл > Сохранить и Файл > Закрыть.
    Примечание: При внесении этих изменений RequisitePro должен быть закрыт.

3.  Создайте документ запросов заинтересованных лиц

Создайте документ запросов заинтересованных лиц в проекте RequisitePro, исходя из измененной схемы запросов заинтересованных лиц.

Для создания документа запросов заинтересованных лиц:

  1. В обозревателе выберите пакет, в котором будет находиться новый документ. Затем выберите Файл > Создать > Документ. Откроется диалоговое окно Свойства документа.
  2. В поле Имя введите "Запросы заинтересованных лиц" или другое имя, чтобы идентифицировать документ. (По этому имени вы будете ссылаться на документ в RequisitePro.)
  3. В поле Описание введите краткое описание документа.
  4. В поле Имя файла введите имя файла, используемое RequisitePro при сохранении документа.
  5. Выберите тип документа, добавленного в шаге 1, (Добавление типа документа запросов заинтересованных лиц в проект),  основанного на схеме RUP Запросы заинтересованных лиц. Нажмите OK.
  6. В новом созданном документе запросов заинтересованных лиц запишите результаты интервью. Вы можете создать несколько документов интервью или сочетать результаты в одном документе. Мы рекомендуем документировать отдельно ответы всех типов, полученные от заинтересованных лиц, чтобы убедиться, что в опросе участвовал широкий диапазон заинтересованных лиц.
  7. Выберите RequisitePro > Документ > Сохранить.

Дополнительная информация

Обратитесь к разделу Создание документов требований (Индекс: документы > создание) электронной справке RequisitePro.

4. Создайте требования в документе запросов заинтересованных лиц (необязательно)

Документ запросов заинтересованных лиц не содержит Требования по умолчанию. Однако чтобы задать ссылки Трассируемости между запросами заинтересованных лиц (отмеченных в документе запросов заинтересованных лиц) и компонентами продуктов (отмеченных в документе Видение). Изменение запросов заинтересованных лиц может повлиять на определение одного или нескольких компонентов продукта. Записав различия между ними вы сможете принимать осторожные и взвешенные решения при расстановке приоритетов вариантов.Пометив запросы заинтересованных лиц как требования вы сможете использовать их в ссылках трассируемости. (См. Задача: управление зависимостями).

Для создания требований из запросов заинтересованных лиц выполните следующие действия:

  1. В документе запросов заинтересованных лиц выберите текст требования.
  2. Выполните одно из следующих действий:
    • Щелкните правой кнопкой и выберите Создать требование.
    • Выберите RequisitePro > Требование > Создать.
      Откроется диалоговое окно Свойства требования.
  3. Выберите STRQ в качестве типа требований. Перейдите на вкладку Атрибуты для изменения значений атрибутов.
  4. Нажмите OK.

Дополнительная информация

Обратитесь к разделу Создание требований в документе (Индекс: требования>создание) электронной справки RequisitePro.
