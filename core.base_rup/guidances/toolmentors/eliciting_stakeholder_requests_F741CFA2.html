Руководство по инструменту: Оценка запросов заинтересованных лиц в Rational ClearQuest
В этой памятке по инструменту описывается захват и управление запросами заинтересованных лиц с помощью различных функций Rational ClearQuest.
Инструмент: Rational Suite AnalystStudio
Взаимосвязи
Связанные элементы
  • Определение требований заинтересованных лиц
Основное описание

Обзор

Rational ClearQuest помогает аналитикам эффективно собирать запросы заинтересованных лиц, предоставляя полную и согласованную форму запроса расширения. Эту форму можно при необходимости настроить для вашей рабочей области разработки. Все запросы записываются в протокол в единой центральной базе данных запросов изменения. Внутренние заинтересованные лица (участники команды разработки) применяют приложения ClearQuest Windows или Web для отправки запросов расширений% внешние заинтересованные лица могут использовать Web-приложения ClearQuest. После отправки запросов расширения аналитик может оценить эти запросы, добавив сведения к каждому из них.

Этапы работы с инструментом

Этот документ описывает следующие действия:

  1. Запись в протокол запросов расширений с помощью ClearQuest Web
  2. организация запросов расширений в ClearQuest Windows
  3. Выполнение запросов

Дополнительная информация о настройке форм ClearQuest приведена в электронной справке ClearQuest и в книге Rational ClearQuest - руководство администратора.

1. Запись в протокол запросов расширений с помощью ClearQuest Web

ClearQuest предоставляет Web-приложение, которое можно настроить для предоставления ограниченной функциональности сторонам, у которых отсутствует лицензия ClearQuest. При ограничении доступа пользователь ClearQuest Web сможет отправлять запросы изменений и обладать доступом к одному определенному администратором запросу ClearQuest. Эта уникальная функция позволяет заинтересованным лицам, не входящим в команду разработчиков ПО, использовать Web для обратной связи. Кроме того, эти заинтересованные лица остаются информированными об обновленном состоянии запросов и извлечении данных, с помощью выполнения запроса ограниченного доступа.

Для отправки запроса расширения с помощью ClearQuest Web выполните следующие действия:

  1. Запустите браузер и укажите URL ClearQuest Web. Появится страница Вход в ClearQuest.
  2. Введите имя пользователя и пароль, нажмите кнопку Войти. Вы подключились к базе данных ClearQuest в Web.
  3. Из списка Оглавление в левой части экрана выберите заголовок Операции и щелкните на подзаголовке Отправить запись. Появится страница Выбрать тип записи.
  4. В списке типов записей выберите Расширить запрос и нажмите кнопку Отправить. Появится форма Отправить запрос расширения, состоящая из трех разделов.
  5. В Основном разделе укажите сведения в полях Заголовок, Источник, Приоритет заказчика и Описание.
  6. В разделе Вложение прикрепите доступную дополнительную информацию для запроса расширения, щелкнув на кнопке Добавить и привязав внешний файл.
  7. В третьем разделе Контактная информация введите допустимые сведения о заказчике и отправителе (имя, название кампании, адрес электронной почты).
  8. Нажмите OK. Запрос расширения будет создан и добавлен в центральную базу данных.

Подробные сведения о настройке ClearQuest Web для предоставления ограниченной функциональности лицам, без лицензии ClearQuest, приведена в разделе Изменение параметров Web электронной справки ClearQuest Web.

2. Организуйте запросы расширения в ClearQuest Windows

После отправки запроса расширения ClearQuest отобразит дополнительные поля, организованные по вкладкам, для дополнения отправленной информации в форме. Вкладки Анализ, Решение и Примечание позволяют аналитику сбалансировать сведения отправителя с относящейся к команде информацией, такой как приоритеты маркетинга и целевой выпуск, для структурирования. Аналитик использует эти сведения для отправки запросов к базе данных запросов расширения для определения запросов, реализуемых в каждом выпуске.

Для организации запросов расширения с помощью ClearQuest Windows:

  1. Выберите запросы расширения и щелкните на опции Действие > Изменить в правой части формы Запрос расширения. Это активирует запрос и позволяет изменить запись.
  2. Щелкните на вкладке Анализ и активируйте ее.
  3. Задайте значения для всех связанных полей, щелкнув на стрелке в каждом поле (Приоритет маркетинга, Тип запроса, Продукт, Область продукта, Целевой выпуск, Владелец) и выберите требуемые значения. Примечание: При необходимости эти поля можно настроить.
  4. Завершив правку нажмите кнопку Применить для сохранения изменений.

Теперь запросы расширений можно запрашивать исходя из заданных значений.

Для просмотра дополнительной информации об организации запросов расширения обратитесь к руководству Применение Rational ClearQuest и Rational RequisitePro для анализа, доступного в Let's Go AnalystStudio.

Информация о запросах и настраиваемых полях приведена в электронной справке ClearQuest.

3. Выполнение запросов

ClearQuest предоставляет мощный интерфейс запросов, позволяющих аналитикам запрашивать все запросы неполадок и расширений в подключенных проектах. Ниже приведены запросы, которые могут пригодиться аналитикам при управлении запросами расширений. Аналитики могут выполнить следующие запросы:

  1. Изолируйте эти запросы расширения, нацеленные на определенные выпуски. Так как не все полученные запросы расширения можно реализовать в данном выпуске, рекомендуется запрашивать все запросы расширений в поле "Целевой выпуск". Этот запрос возвращает список всех запросов расширений, разработанных для указанного в запросе выпуска.
  2. Просмотрите все запросы расширения, не связанные с требованием. Этот запрос помогает аналитику просматривать запросы расширения, которые все еще рассматриваются или находятся в очереди, но еще не были присвоены определенной итерации ПО.
  3. Найдите число экземпляров или вхождений одного запроса расширения. Этот запрос помогает аналитику определить число отправки запроса расширения заинтересованным лицом. Это добавляет процесс распределения весовых коэффициентов для определения важности запроса заинтересованных лиц и уровня приоритета.
  4. Разделение истинных запросов расширения и запросов новых функций. Этот запрос показывает состояние выполнения разработки ПО. Аналитик может понять, предназначено для большинство запросов расширения для исправления существующих функций или они требуют добавления новых функций в текущий продукт.

Аналитик может легко привязать утвержденные текущие запросы расширения версии к допустимым требованиям из базы данных требований с помощью интеграции ClearQuest и RequisitePro.
