Руководство по инструменту: Выполнение набора тестирования с помощью Rational TestManager
В этой памятке по инструменту описывается применение Rational TestManager для выполнения тестового комплекта.
Инструмент: Rational TestManager
Взаимосвязи
Связанные элементы
  • Выполнение комплекта тестов
Основное описание

Обзор

Как часть возможностей по управлению тестированием TestManager позволяет скоординировать тестовые сценарии всех типов - GUI, ручной тест, виртуальный пользователь, командная строка, Visual Basic и Java - в один исполняемый пользователем шаг. Этот этап, называемый комплектом, позволяет распространять, выполнять и вычислять семейства тестовых сценариев в тестовой команде в виде единого целого, а не фрагментов.

При выполнении комплекта вы предоставляете касающиеся выполнения сведения. Каждый виртуальный тестер, выполняющий присвоенные ему элементы комплекта, следует данным рекомендациям. Результаты выполнения комплекта хранятся в протоколах. После выполнения теста можно создать отчеты для анализа данных, хранимых в протоколах, и для отображения результатов выполнения в виде графиков и диаграмм.

См. Памятку по инструменту: проектирование автоматизированного тестового комплекта с помощью Rational TestManger для получения информации о создании тестового комплекта.

Эта памятка по инструменту применима к Microsoft Windows 98/2000/NT 4.0.

Этапы работы с инструментами

Для выполнения тестового комплекта с помощью TestManager необходимо выполнить следующие действия:

  1. Укажите комплект
  2. Задайте пользовательскую нагрузку
  3. Укажите файл протокола
  4. Включите или отключите мониторинг ресурсов
  5. Выберите, следует ли игнорировать конфигурацию
  6. Укажите опции создания отчетов выполнения

1.   Укажите комплект

  • Чтобы указать комплект выберите в меню Файл пункт Выполнить комплект. Откроется диалоговое окно Выполнить комплект.
  • Для изменения комплекта выберите Изменить... в разделе Сведения о комплекте.

2.   Задайте пользовательскую нагрузку

  • Если выбранный для выполнения комплект имеет заданную фиксированную группу компьютеров, поля Имя отсутствуют, переходите к этапу 3, Укажите файл протокола.
  • В ином случае введите нужное число виртуальных тестеров в поле Число пользователей.
  • Если комплект содержит фиксированные и масштабируемые группы, фиксированные группы присваиваются первыми. Число доступных виртуальных тестеров зависит от типа имеющейся у вас лицензии. Если ваша лицензия не поддерживает указанное число виртуальных тестеров, появится сообщение об ошибке.

Примечание: При определении групп компьютеров, если выбрать Запрашивать компьютеры перед выполнением комплекта появится список Компьютеры с кнопками Изменить компьютеры, Изменить списки компьютеров и Свойства.

3.   Укажите файл протокола

В поле Компиляция появится номер текущей компиляции. Имя папки протоколов основано на комплекте, имя протокола основано на числе виртуальных тестеров и числе повторов выполнения комплекта.

Для изменения папки протоколов или имени протокола выберите Изменить... в разделе Сведения о протоколе.

4.   Включите ил отключите мониторинг ресурсов

Если планируется выполнять мониторинг ресурсов, отметьте переключатель Мониторинг ресурсов. Измените интервал в поле Частота обновления чтобы изменить частоту обновления панелей. Чем ниже интервал, тем быстрее обновление.

Мониторинг выполняет наблюдение за использованием компьютерных ресурсов при воспроизведении комплекта и затем на графике эти данные отображаются относительно времени ответа соответствующего виртуального тестера при анализе результатов.

5.   Укажите, следует ли игнорировать конфигурацию

Выберите переключатель Игнорировать конфигурацию для тестовых сценариев чтобы TestManager игнорировал конфигурации для тестовых сценариев и выполнял их на доступных компьютерах.

TestManager предлагает три способа выполнения комплектов при выбранной этой опции:

  • Если в комплекте имеется тестовый сценарий с настроенными тестовыми сценариями и в родительском тестовом сценарии имеется, к примеру, реализация тестового сценария или комплекта, TestManager выполняет родительский тестовый сценарий на любом доступном компьютере, но не выполняет настроенные тестовые сценарии.
  • Если тестовый комплект содержит тестовый сценарий с настроенными тестовыми сценариями и родительский тестовый сценарий не имеет реализаций, TestManager не выполняет родительский или любой другой настроенный тестовый сценарий.
  • Если один настроенный тестовый сценарий имеет реализацию, TestManager выполняет тестовый сценарий на указанном компьютере.

6.   Укажите опции создания отчетов выполнения

  • Для изменения отчетов TestManager о состоянии выполнения комплекта, выберите пункт Опции... . Вы сможете задать следующие типы опций:
    • Монитор - эта вкладка контролирует отображение информации в TestManager при выполнении комплекта.
    • Отчеты - эта вкладка задает отчеты, создаваемые TestManager после завершения выполнения комплекта. Эта опция применима только к тестированию производительности.
    • Выполнить - эта вкладка отвечает за ведение протоколов выполнения комплекта для последующей оценки.
    • Компиляция VU - эта вкладка задает определения по умолчанию для условной компиляции VU и добавления внешних библиотек C в комплект.

   Дополнительные сведения приведены в разделе Диалоговое окно Опции электронной справки Rational TestManager.

  • Настроив выполнение нажмите OK для выполнения теста. TestManager проверит комплект и скомпилирует нескомпилированные ли устаревшие сценарии. Для остановки выполнения комплекта при проверке комплекта в TestManager нажмите кнопку Отмена.
  • После того, как TestManager проверит комплект и скомпилирует необходимые тестовые сценарии, окно Подготовка к запуску будет свернуто. В этот момент отменить или закрыть окно нельзя. Чтобы прервать выполнение выберите пункт Остановить в меню Монитор.
  • Выполнив тесты можно оценить их результаты.
