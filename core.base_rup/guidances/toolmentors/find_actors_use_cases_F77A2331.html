Руководство по инструменту: Поиск субъектов и вариантов с помощью Rational Rose
В этой памятке по инструменту описывается применение Rational Rose для записи результатов поиска субъектов и вариантов.
Инструмент: Rational Rose
Взаимосвязи
Связанные элементы
  • Поиск субъектов и вариантов
Основное описание

Обзор

Для записи результатов поиска субъектов и вариантов с помощью Rational Rose выполните следующие действия:

  1. Создайте пакет модели прецедентов

  2. Создайте диаграмму прецедентов
  3. Создайте субъекты
  4. Создайте варианты
  5. Задокументируйте взаимосвязь между субъектами и вариантами
  6. Управление вариантами с помощью Rational Rose и Rational RequisitePro

1. Создайте пакет модели прецедентов

В Rational Rose можно представить отдельную модель варианта с помощью пакета панели вариантов с именем "Модель варианта".Для создания пакета с именем "Модель варианта" на панели Вариант выполните следующие действия:

  • Щелкните правой кнопкой мыши для выбора панели Вариант в браузере.
  • Выберите Пакет в пункте Создать всплывающего меню. В браузер будет добавлен значок "NewPackage".
  • Выберите значок и введите имя "Модель прецедента".

Отдельный пакет модели прецедента необходим только при хранении модели бизнес-прецедента и модели системного прецедента в одной модели Rational Rose. В ином случае варианты и субъекты можно создавать непосредственно в панели Вариант в браузере.

2. Создайте пользовательскую диаграмму

субъекты и варианты можно создать на диаграмме вариантов.

Для создания диаграммы прецедентов из модели прецедентов выполните следующие действия:

  • Щелкните правой кнопкой для выбора пакета с именем "Модель прецедента" в браузере и откройте всплывающее меню.
  • Выберите Диаграмма вариантов в пункте Создать всплывающего меню. В браузере появится значок диаграммы прецедентов "NewDiagram".
  • Выбрав новую диаграмму вариантов введите ее имя.
  • Дважды щелкните на новой диаграмме прецедентов чтобы открыть ее в окне диаграмм.

3. Создайте субъекты

Для создания субъекта на диаграмме прецедентов выполните следующие действия:

  • Дважды щелкните на диаграмме вариантов в панели Вариант в браузере для отображения диаграммы в окне диаграмм.
  • Выберите на панели инструментов пункт Субъект. Форма курсора изменится на символ плюса.
  • Щелкните левой кнопкой мыши на диаграмме прецедентов в том месте, куда требуется вставить символ субъекта. Введите имя нового субъекта.

Для краткого описания субъекта выполните следующие действия:

  • Откройте диалоговое окно Спецификация субъекта, дважды щелкнув на символе субъекта в диаграмме прецедентов в браузере. Откроется спецификация класса, в которой "Субъект" будет указан как параметр стереотипа.
  • Перейдите на вкладку Общие.
  • Впишите краткое описание субъекта в поле Документация.
  • Нажмите OK чтобы принять краткое описание и закрыть окно спецификации субъекта.

4. Создаете варианты

Для создания нового варианта на диаграмме вариантов выполните следующие действия:

  • Дважды щелкните на диаграмме вариантов в панели Вариант в браузере для отображения диаграммы в окне диаграмм.
  • Выберите на панели инструментов пункт Вариант. Форма курсора изменится на символ плюса.
  • Щелкните левой кнопкой мыши на диаграмме вариантов в том месте, куда требуется вставить символ варианта. Введите имя нового варианта.

Для краткого описания варианта выполните следующие действия:

  • Откройте диалоговое окно Спецификация варианта, дважды щелкнув на символе варианта в диаграмме вариантов в браузере. Перейдите на вкладку Общие.
  • Впишите краткое описание варианта в поле Документация.
  • Нажмите OK чтобы принять краткое описание и закрыть окно спецификации варианта.

5. Задокументируйте взаимосвязь между субъектами и вариантами

Для вставки ассоциации связи между субъектом и вариантом в диаграмму вариантов выполните следующие действия:

  • Выберите стрелку Ассоциация из панели инструментов в редакторе диаграмм прецедентов.
  • Наведите курсор на субъект в диаграмме прецедентов. Щелкните левой кнопкой мыши на символе прецедента.
  • Дважды щелкните на созданной ассоциации и выберите стереотип <<связывается>> в диалоговом окне Спецификация ассоциации.
  • Нажмите OK.
  • Щелкните правой кнопкой на созданной ассоциации и убедитесь, что во всплывающем меню выбран пункт Показать стереотип.
  • Метку стереотипа можно переместить на диаграмме.

Для добавления краткого описания ассоциации связи выполните следующие действия:

  • Откройте диалоговое окно Спецификация ассоциации, дважды щелкнув на символе ассоциации на диаграмме прецедентов.
  • Теперь по умолчанию вкладка Общие окна Спецификация ассоциации отображается.
  • Впишите краткое описание в поле Документация.
  • Нажмите OK для сохранения краткого описания и закрытия окна Спецификация ассоциации.

Для описания множественности роли в ассоциации связи выполните следующие действия:

  • Щелкните правой кнопкой мыши на линии ассоциации рядом с субъектом или вариантом, в котором требуется выбрать множественность.
  • Выберите множественность из раздела множественностей всплывающего меню. Отсутствующие во всплывающем меню значения можно указать в диалоговом окне Спецификация ассоциации, открыть которое можно с помощью двойного щелчка по ассоциации.

Чтобы указать инициативу роли в ассоциации связи выполните следующие действия:

  • Щелкните правой кнопкой мыши на линии ассоциации рядом с субъектом или вариантом, в котором требуется выбрать инициативу.
  • Выберите или отмените выбор свойства Инициатива во всплывающем меню.

6. Управление вариантами в Rational Rose и 
Rational RequisitePro

Интегрированное управление вариантами Rational позволяет управлять вариантами в Rational Rose с помощью атрибутов, таких как Приоритет, Риск, Состояние и Итерация, связывая варианты в Rose с документами и требованиями Rational RequisitePro.Вы можете легко перемещаться между моделями прецедентов в Rose и документами прецедентов и требованиями RequisitePro. Управление вариантами в RequisitePro добавляет глубину и дополнительную информацию к вашим вариантам Rose.
