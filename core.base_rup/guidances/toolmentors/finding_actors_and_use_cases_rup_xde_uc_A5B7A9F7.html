Основное описание

Обзор 

Шаги, описанные в этой памятке по инструменту, соответствуют этапам выполнения задачи. Ссылки на разделы электронной справки Rational XDE (TM) помечены значком .

Эта памятка по инструменту подразумевает, что модель прецедента уже создана соответственно рекомендациям по структуре модели XDE.

В этой памятке по инструменту выполняются следующие шаги:

  • Поиск субъектов
  • Поиск вариантов
  • Опишите как взаимодействуют субъекты и варианты
  • Упакуйте варианты и субъекты
  • Представьте модель прецедента и диаграммы прецедента
  • Разработайте обзор модели прецедентов
  • Вычисление результатов

Поиск субъектов

Найдя субъекты запишите результаты следующим образом:

  1. Откройте модель прецедентов.
  2. Перейдите к пакету, в котором будут захвачены субъекты. См. Рекомендации по структуре модели Rational XDE.
  3. Создайте диаграмму для захвата субъектов, выбрав диаграмму прецедентов в качестве типа диаграммы. Справочник: Добавление диаграмм.
  4. Добавление субъектов к диаграмме. См. Добавление форм к диаграммам .
  5. Добавьте для каждого субъекта краткое описание. См. Документирование элементов модели .

Справочник: Работа с диаграммами прецедентов.

Поиск вариантов

Найдя варианты запишите результаты следующим образом:

  1. Перейдите к пакету, в котором будет выполняться захват вариантов. См. Рекомендации по структуре модели Rational XDE.
  2. Создайте диаграмму вариантов для захвата вариантов. Справочник: Добавление диаграмм
  3. Добавьте варианты к диаграмме. См. Добавление форм к диаграммам.
  4. Задайте краткое описание и схематичный поток событий. Для него можно добавить примечание в элементе модели. (См. Документирование элементов модели .) Либо можно использовать предоставленный RUP шаблон для Продукта работы: вариант и прикрепить его к варианту в модели.См. Привязка внешних файлов к элементам модели.

Опишите как взаимодействуют субъекты и варианты

Опишите взаимодействие между субъектами и вариантами, выполнив следующие действия:

  1. Добавьте ранее созданные субъекты в диаграмму вариантов, содержащую варианты. См. Добавление форм к диаграммам .
  2. Добавьте взаимодействия между субъектами и вариантами. (См. Добавление взаимосвязей ассоциаций .) Если ассоциация доступна для навигации только в одном направлении, выберите направленную ассоциацию, щелкните на форме получателя и затем на форме поставщика.
  3. При необходимости опишите ассоциацию. См. Документирование элементов модели .
  4. При необходимости добавьте множественность роли. См. Настройка множественности на концах ассоциации .
