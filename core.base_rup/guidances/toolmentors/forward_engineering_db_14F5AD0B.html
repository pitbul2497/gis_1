Задайте свойства модели

Необходимо задать два типа свойств модели для включения прямого инжиниринга модели данных для создания базы данных или файла языка определения данных (DDL).  Этими свойствами являются:

  • Целевая система управления базой данных (DBMS) базы данных по умолчанию (см. Настройка цели базы данных по умолчанию )
  • Присваивание базы данных по умолчанию (см. Настройка присваивания базы данных по умолчанию )

Rational XDE поддерживает прямой инжиниринг в стандарт ANSI SQL 92 и в системы управления базами данных (DBMS), перечисленными в разделе электронной справки Rational XDE Базы данных, поддерживаемые в Rational XDE .

Справочник: Настройка опций моделирования данных .

Прямой инжиниринг модели данных в базу данных 

Rational XDE можно использовать для прямого инжиниринга модели данных в новую схему базы данных или в новый файл DDL. МОжно выполнить прямой инжиниринг отдельных таблиц или всей модели данных. Справочник: Создание базы данных и DDL Процесс прямого инжиниринга вызывается с помощью моделирования данных Мастера прямого инжиниринга, поставляемого с Rational XDE. (См. Создание баз данных или DDL .)  

Если вы решите только создать DDL или выполнить DDL для создания базы данных, Rational XDE создаст файл DDL за вас в каталоге, указанном в мастере. Элементы, добавляемые Rational XDE в процесс прямого инжиниринга, зависят от опций, выбранных в Мастере прямого инжиниринга. Все элементы или сценарии DDL создаются исходя из указанной DBMS. Обратитесь к следующим разделам электронной справки XDE для получения информации о различных типах поддерживаемых DBMS:

  • Создание баз данных DB2 и DDL
  • Создание баз данных Oracle и DDL
  • Создание базы данных SQL Server и DDL
  • Создание баз данных Sybase и DDL

При выполнении прямого инжиниринга над моделью данных ANSI SQL 92, Rational XDE создаст независимый от DBMS DDL. Для выполнения этого DDL можно использовать инструмент базы данных на ваш выбор.

Для выполнения прямого инжиниринга базы данных Rational XDE должен установить соединение с соответствующей DBMS. См. Подключение к системам управления базами данных  для получения дополнительных сведений об установке подключений к базе данных. Обратитесь к следующим разделам электронной справки Rational XDE для получения сведений об установке соединений с определенным типом DBMS:

  • Соединения с базой данных Oracle
  • Соединения с базой данных SQL Server
  • Соединения с базой данных Sybase
  • Соединения с базой данных DB2

Управление изменениями в модели и базе данных

После создания базы данных DDL изменения модели данных и создаваемой базы данных следует контролировать и отслеживать.  Rational XDE содержит функцию Сравнить и синхронизировать для помощи разработчику базы данных в управлении изменениями.
