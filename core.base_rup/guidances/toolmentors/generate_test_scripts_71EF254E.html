Основное описание

Обзор

"Пилот" является инструментом Rational TestFactory, используемым для создания тестовых сценариев. Пилот создает сценарии, проверяющие функционирование управляющих элементов в тестируемом приложении (AUT), представленных объектами UI в области схемы приложения. Сведения о разработке схемы приложения приведены в Памятке по инструменту: настройка тестовой среды в Rational TestFactory.

Эта памятка по инструменту применима для систем с Windows 98/2000/NT 4.0.

Для применения Rational TestFactory для автоматического создания сценариев:

  1. Добавьте Пилот к схеме приложения
  2. Настройте и запустите Пилот

1. Добавьте Пилот к схеме приложений

Можно добавить объект Пилот в любое место на схеме приложения. Оттуда Пилот автоматически создаст тестовые сценарии, сфокусированные на определенной функциональной области AUT, соответствующей данной области схемы приложения.

Чтобы определить часть приложения для тестирования, обратитесь к описанию первой задачи в Памятке по инструмент: захват результатов проектирования теста для автоматического тестирования.

   Обратитесь к следующим разделами электронной справки Rational TestFactory:

  • Пилоты: что это такое и как они работают
  • Эффективное размещение пилота
  • Добавление пилота

2. Настройте и запустите Пилот

Настроив Пилот укажите значения и опции, контролирующие выполнение Пилота. Вы можете указать следующую установочную информацию:

  • Глубина схемы приложения для тестирования Пилотом.
  • Критерий, используемый Rational TestFactory для остановки выполнения Пилота.
  • Дополнительные объекты UI на схеме приложения, которые следует добавить в тест.
  • Объекты UI под управлением Pilot, которые следует исключить из теста.

При выполнении Пилота он создает оптимизированный "лучший сценарий", предоставляющий расширенный охват теста на базе кода и содержащий избыточный код сценария.

Пилот также создает "сценарий UI", оптимизированный для охвата кода на базе UI. Он создан из AUT, вы можете выполнить сценарии UI в виде простого теста герметичности для проверки управляющих элементов в пользовательском интерфейсе.

При каждом запуске Пилота, Rational TestFactory добавляет папку "run" под Пилотом на схеме приложения. Папка run содержит тестовые сценарии, создаваемые Пилотом. При обнаружении пилотом неполадок при выполнении, TestFactory помещает дефектные тестовые сценарии в подкаталог "defects" папки run.
