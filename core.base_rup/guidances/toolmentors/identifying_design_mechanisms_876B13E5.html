Основное описание

Обзор 

Эта памятка по инструменту поддерживает механизмы идентификации разработки с Rational XDE Developer - Java Platform Edition.

  • Шаги, описанные в этой памятке по инструменту, соответствуют этапам выполнения определенных задач.
  • Ссылки на разделы электронной справки XDE помечены с помощью значка .
  • В памятке описаны следующие действия:
    • Категоризация клиентов механизмов анализа
    • Идентификация механизмов разработки для механизмов пользовательского интерфейса
    • Реестр механизмов реализации
    • Преобразование механизмов разработки в механизмы реализации
    • Механизмы документации архитектуры

Категоризация клиентов механизмов анализа

Для данного шага определенные руководства Rational XDE отсутствуют.

Идентификация механизмов разработки для механизмов пользовательского интерфейса

Для данного шага определенные руководства Rational XDE отсутствуют.

Реестр механизмов реализации

Для перечисления всех доступных механизмов реализации определенные указания Rational XDE отсутствуют. Однако некоторые механизмы реализации J2EE описаны в следующих разделах электронной справки Rational XDE:

  • Знакомство с разработкой на основе шаблонов J2EE
  • Разработка на основе шаблонов J2EE
  • Поддержка шаблонов J2EE

Преобразование механизмов разработки в механизмы реализации

Для данного шага определенные руководства Rational XDE отсутствуют.

Механизмы документации архитектуры

Сами по себе механизмы являются элементами модели разработки (такими как пакет разработки, класс разработки, подсистема разработки), которые могут быть представлены в продукте работы: модель разработки в виде части соответствующих операций разработки. См. Памятка по инструменту: идентификация элементов разработки для ознакомления с рекомендациями по созданию элементов модели разработки. Обратите внимание, что шаблон Rational XDE особенно хорошо подходит для документирования механизмов разработки и реализации, так как позволяет клиентам механизма развернуть шаблон и создать большую часть эскиза и кода.
