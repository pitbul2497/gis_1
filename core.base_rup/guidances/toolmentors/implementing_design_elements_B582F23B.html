Основное описание

В этой памятке предполагается, что набор структурированных моделей уже создан согласно Рекомендациями по структуре модели XDE, в частности, модели реализации.

Создать реализацию можно изменив исходный код и выполнив обратный инжиниринг модели кода либо смоделировав код в Rational XDE (обычно с помощью развития ранее созданной модели разработки) и создав исходный код.

Для создания элементов модели обратитесь к памяткам по инструменту для операций разработки. Обратите внимание, что элементы модели реализации зачастую создаются с помощью перемещения элемента модель разработки в модель реализации. Элемент по-прежнему может указываться из модели разработки - см. Концепция: преобразование из разработки в код.
