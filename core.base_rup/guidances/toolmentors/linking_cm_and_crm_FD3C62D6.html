Руководство по инструменту: Привязка управления конфигурацией и запросами изменений с помощью Rational ClearQuest и Rational ClearCase
В этой памятке по инструменту описывается применение Rational ClearQuest с проектами унифицированного управления изменениями (UCM) Rational ClearCase.
Инструмент: Rational ClearCase
Взаимосвязи
Связанные элементы
  • Настройка среды управления конфигурацией (CM)
Основное описание

Обзор

Проекты ClearCase UCM можно интегрировать с ClearQuest, инструментом управления запросами изменения. С помощью ClearQuest можно отправить запросы на изменения, просмотреть и изменить существующие запросы на изменение, создать и выполнить определенные для пользователя или сайта запросы и отчеты. Интеграция UCM-ClearQuest добавляет мощные функции по управлению проектом и операциями в проекты UCM. Например, записи ClearQuest можно привязать к операциям ClearCase, которые можно задать для управления доставкой операций в ClearCase и возможностью завершения операции в ClearQuest.

Эта памятка по инструменту применима к Microsoft Windows.

  Дополнительные сведения приведены в руководстве Руководство IBM Rational ClearCase по управлению программными проектами.

Этапы работы с инструментом

Для настройки интеграции UCM-ClearQuest выполните следующие действия:

  1. Включите схему ClearQuest для работы с UCM или воспользуйтесь предустановленной схемой с поддержкой UCM
  2. Создайте или обновите базу данных запросов изменений для применения со схемой
  3. Включите применение проекта с ClearQuest

1. Включите схему ClearQuest для работы с UCM или воспользуйтесь предустановленной схемой с поддержкой UCM

Описанные ниже действия касаются применения предустановленных схем с поддержкой UCM для настройки работы базы данных запросов изменений с UCM. Предустановленные схемы UCM, названные UnifiedChangeManagement и Enterprise, содержат тип записи, поле, форму, состояние и прочие необходимые определения для работы с проектом UCM.

Для настройки работы базы данных запросов на изменение с UCM выполните следующие действия:

  1. Создайте пользовательскую базу данных с одной из предустановленных схем с поддержкой UCM.
  2. В ClearQuest Designer выберите пункт База данных > Создать базу данных для запуска мастера создания базы данных.
  3. Выполните этапы мастера, на этапе 4 вам будет предложено выбрать схему для связывания с новой базой данных. Прокрутите список имен схем и выберите новую схему.
  4. Нажмите Готово.

  Дополнительные сведения о том, как включить схему ClearQuest для работы с интеграцией UCM-ClearQuest приведены в главе "Настройка пользовательской базы данных Rational ClearQuest для UCM" в руководстве Руководство IBM Rational ClearCase по управлению программными проектами.

2. Создайте или обновите базу данных запросов на изменение для применения схемы

Предустановленные схемы UCM позволяют использовать интеграцию UCM-ClearQuest немедленно, но вы можете предпочесть создать пользовательскую схему для отслеживания операций проекта и запросов на изменение. Можно даже использовать различные предустановленные схемы. Существует несколько способов сделать это, в зависимости от потребностей вашего проекта.

Мы рекомендуем вам ознакомиться с главой "Настройка пользовательской базы Rational ClearQuest для UCM" руководства Руководство IBM Rational ClearCase по управлению программными проектами перед продолжением. Также в этом разделе содержатся подробные сведения о процедурах для данной задачи.

   Дополнительная информация об этой процедуре приведена в разделе "Планирование использования интеграции UCM с Rational ClearQuest" в руководстве Руководство IBM Rational ClearCase о управлению программными проектами.

3. Включите применение проекта UCM с ClearQuest

  • В левой панели обозревателя проектов ClearCase Project Explorer щелкните правой кнопкой на проекте для отображения контекстного меню. Выберите Свойства для просмотра страницы свойств.
  • Перейдите на вкладку ClearQuest и выберите переключатель Проект поддерживает ClearQuest. В системах Ссылка на пользовательскую базу данных ClearQuest находится возле имени. Откроется диалоговое окно Хранилище схем Rational ClearQuest с соединением, относящимся к выбранной пользовательской базе данных.
  • Выберите пользовательскую базу данных из списка "Ссылка на эту базу данных запросов на изменение".
  • При первом включении проекта ClearQuest отобразит диалоговое окно Вход в систему. Введите имя пользователя, пароль и имя базы данных, к которой привязывается проект.
  • Нажмите OK.

   Дополнительная функция: можно также задать стратегии разработки для проектов, использующих интеграцию UCM-ClearQuest. См. раздел "Стратегии интеграции UCM с Rational ClearQuest" в руководстве Руководство IBM Rational ClearCase по управлению программными проектами для получения дополнительной информации.
