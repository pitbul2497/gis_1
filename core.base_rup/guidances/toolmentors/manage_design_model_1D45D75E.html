Обзор

Для управления продуктом работы: модель разработки выполните следующие действия:

  1. Создайте модель разработки
  2. Создайте пакеты в модели разработки
  3. Создайте уровни в модели разработки
  4. Опишите структуру модели

1. Создайте модель разработки

Представить продукт работы: модель разработки в Rational Rose можно с помощью пакета в логическом виде с именем "Модель разработки". Модель разработки представляет уровень абстрактности, близкий к исходному коду. Например, классы в модели разработки обычно имеют присвоенные им свойства языка и модели, задающие привязку к коду.

2. Создайте пакеты в модели разработки

Пакет является элементом модели общего назначения, организующий элементы модели в группы. Систему при этом можно рассматривать как единый пакет высокого уровня, содержащий все прочие элементы системы.

Дополнительные сведения о пакетах приведены в разделе Пакет электронной справки Rational Rose.

3. Создайте уровни в модели разработки

Модель может содержать несколько уровней, визуализируемых в виде пакетов, по одному на уровень. Уровни могут быть вложены. Для определения пакетов в качестве уровней, необходимо добавить стереотип "уровень" к каждому пакету. Кроме того для описания пакета можно использовать поле Описание в спецификации пакета.

4. Опишите структуру модели

Каждой модели необходима диаграмма класса, показывающая структуру уровней и подсистем модели. Она является главной диаграммой. Рекомендуется переименовать ее в "Структуру модели". Переместите пакеты, представляющие уровни или подсистемы, на диаграмму.
