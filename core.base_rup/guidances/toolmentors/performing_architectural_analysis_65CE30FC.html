Обзор

Эта памятка по инструменту поддерживает выполнение архитектурного анализа с помощью Rational XDE Developer - Java Platform Edition.   

В этой памятке предполагается, что набор структурных моделей уже создан заранее, согласно Рекомендациям по структуре модели XDE.

  • Шаги, описанные в этой памятке по инструменту, соответствуют этапам выполнения определенных задач.
  • Ссылки на разделы электронной справки XDE помечены .
  • В этой памятке по инструменту выполняются следующие шаги:
    • Разработка обзора архитектуры
    • Обзор доступных ресурсов
    • Определение организации подсистем на высоком уровне
    • Идентификация абстракций ключей
    • Идентификация стереотипных взаимодействий
    • Разработка обзора развертывания
    • Идентификация механизмов анализа
    • Просмотрите результаты

Разработка обзора архитектуры 

Rational XDE можно использовать для создания неформальных диаграмм, описывающих обзор архитектуры.

Обзор доступных ресурсов

Архитектор должен рассмотреть возможность многократного использования имеющихся ресурсов, включая модели Rational XDE.

Определение организации подсистем на высоком уровне

Добавьте пакеты в модель проекта для отражения выбранной стратегии уровней. Согласно примеру уровней, приведенному в Рекомендациях по структуре модели Rational XDE, необходимо выполнить следующие действия:

  1. Откройте модель проекта.
  2. Перейдите к пакету или пакетам, содержащим подпакеты уровней.
  3. Создайте пакеты для каждого уровня и присвойте им стереотип <<уровень>>. Справочник: Группировка элементов модели в пакетах , Работа со стереотипами .

Рекомендации по структурированию моделей приведены в следующих документах:

  • Рекомендации по структуре модели Rational XDE
  • Стратегии создания уровней

Идентификация абстракций ключей 

Захватите ключевые абстракции в диаграммах классов с кратким описанием для каждого класса. Выполните следующие действия:

  1. Откройте модель проекта.
  2. Перейдите к пакету, содержащему ключевые абстракции. См. Рекомендации по структуре модели Rational XDE.
  3. Добавьте диаграмму классов. См. Добавление диаграмм .
  4. Добавьте к диаграмме классы со стереотипом <<сущность>>. Справочник: Добавление форм к диаграммам , Присвоение стереотипов .
  5. Добавьте описание к классам. См. Документирование элементов модели .
  6. При необходимости сопоставьте документ с классом. См. Привязка внешних файлов к элементам модели .
  7. Задайте взаимосвязи, существующие между классами.
    • Добавьте взаимосвязи ассоциаций. См. Добавление взаимосвязей ассоциаций .
    • Задайте типы взаимосвязей ассоциаций. См. Определение типов взаимосвязей ассоциаций .
    • Добавьте взаимосвязи обобщения. См. Добавление взаимосвязей обобщений .

Справочник: Работа с диаграммами классов .

Идентификация стереотипных взаимодействий

Это действие выполняется только при запуске данной задачи во время ознакомления.

Целью выполнения данного действия является идентификация данных взаимодействий между ключевыми абстракциями в системе, характеризующими или представляющими значимые типы операций в системе. Эти взаимодействия захватываются в качестве реализаций прецедентов.

Руководства по созданию реализаций прецедентов в XDE приведены в Памятке по инструменту: выполнение анализа прецедентов с помощью XDE.

Разработка обзора развертывания

  1. Добавьте диаграмму развертывания к модели развертывания. См. Добавление диаграмм .
  2. Добавьте к диаграмме узлы. См. Добавление форм к диаграммам .
  3. Добавьте ассоциации между узлами. См. Добавление взаимосвязей ассоциаций .

Справочник: Работа с диаграммами развертывания .
