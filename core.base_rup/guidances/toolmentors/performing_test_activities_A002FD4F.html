Руководство по инструменту: Выполнение тестовых операций с помощью Rational TestManager
В этой памятке по инструменту описывается применение Rational TestManager для выполнения пяти операций тестирования.
Инструмент: Rational TestManager
Взаимосвязи
Связанные элементы
  • Определение конфигурации среды тестирования
  • Определение мотивационных факторов тестирования
  • Определение подхода к тестированию
  • Определение потребностей в оценке и прослеживаемости
  • Определение цели теста
  • Создание реализация теста
Основное описание

Обзор

Rational TestManager является инструментом по управлению всеми операциями тестирования - планированием, проектированием, реализацией, выполнением и анализом. TestManager связывает воедино тестирование с остальным процессом разработки, объединяя ресурсы тестирования и инструменты для предоставления единой точки для понимания точного состояния проекта.

Эта памятка по инструменту применима для систем с Windows 98/2000/NT 4.0.

Этапы работы с инструментами

Для применения Rational TestManager:

  1. Спланируйте тесты
  2. Спроектируйте тесты
  3. Реализуйте тесты
  4. Выполните тесты
  5. Оцените тесты

1.   Спланируйте тесты

При планировании тестов основной задачей является ответ на вопрос "Что мне нужно протестировать?". Завершив планирование теста в вашем распоряжении будет план теста, определяющий тестируемые элементы.

В Rational TestManager план теста может обладать многими свойствами. Добавить свойства можно при первом создании плана теста или добавить их потом.

Некоторые свойства:

  • Описание плана тестирования
  • Владелец плана тестирования
  • Итерации и конфигурации, связанные с планом тестирования
  • Внешние документы, связанные с планом тестирования

В Rational TestManager план тестирования может содержать список тестовых сценариев. Тестовые сценарии можно организовать по папкам.

Спланировав тесты можно приступать к их проектированию.

   Обратитесь к разделу Планирование тестов электронной справки Rational TestManager.

2.   Спроектируйте тесты

При проектировании тестов необходимо ответить на вопрос "Как тест будет выполняться?". Завершив проектирование теста вы создадите проект теста, с помощью которого будет проще понять процесс выполнения тесового сценария и вы сможете переходить к планированию его реализации.

В Rational TestManager можно разработать тестовые сценарии, указав фактические действия, которые следует выполнить в этом тесте. Также можно указать входные и выходные условия и критерии приемлемости.

Спроектировав тесты можно переходить к их реализации.

   Обратитесь к разделу Проектирование тестов электронной справки Rational TestManager.

3.   Реализуйте тесты

Операция реализации тестов в основном создает доступные для многократного использования тестовые сценарии.

В Rational TestManager реализовать тесты можно создав сценарии ручного выполнения. Сценарий ручного выполнения является набором инструкций тестирования для выполнения тестером. Также можно реализовать автоматические тесты с помощью Rational Robot.

Можно расширить Rational TestManager через API, чтобы иметь доступ к вашим собственным инструментам реализации через TestManager. Благодаря этой расширяемости вы можете реализовать тесты, создав сценарии с помощью инструментов, наиболее подходящих для вашей ситуации или организации. Например, можно реализовать сценарии Visual Test, пакетные файлы или сценарии Perl.

Реализовав сценарии вы можете воспользоваться Rational TestManager чтобы связать их с прочими продуктами работы теста в TestManager. Например, можно связать сценарий, созданный в другом инструменте, с тестовым сценарием, созданном в TestManager.

Реализовав тесты можно выполнить их в Rational TestManager.

   Обратитесь к разделу Реализация тестов электронной справки Rational TestManager.

4.   Выполните тесты

Операция выполнения тестов состоит из выполнения тестовых сценариев и обеспечении верного функционирования системы.

В Rational TestManager выполнить тесты можно несколькими способами:

  • Выполните отдельный тестовый сценарий, запускающий одиночную реализацию.
  • Выполните один или несколько тестовых сценариев, выполняющих реализации тестовых сценариев.
  • Выполните комплект, запускающий тестовые сценарии и их реализации среди нескольких компьютеров или пользователей.

Выполнив тесты можно оценить их результаты.

   Обратитесь к разделу Выполнение тестов электронной справки Rational TestManager.

5.   оцените тесты

При оценке тестов определяется качество тестируемой системы.

В Rational TestManager оценить тесты можно изучив результаты выполнения тестов в протоколе теста и выполнив различные отчеты.

Тестовый протокол указывает успешно ли был выполнен сценарий и предоставляет вам возможность проинспектировать информацию для оценки результатов. Из протокола теста можно определить и записать запросы изменений.

В Rational TestManager существует три основных типа отчетов:

  • Отчеты распределения и тенденций тестового сценария: помогает отслеживать состояние планирования тестового сценария, реализации и выполнения.
  • Отчеты тестирования производительности: помогают оценивать относительную эффективность выполнения приложением ключевых задач в заданных условиях.
  • Отчеты со списками: отображают списки различных тестовых ресурсов, хранящихся в проекте Rational.

   Обратитесь к разделу Оценка тестов электронной справки Rational TestManager.
