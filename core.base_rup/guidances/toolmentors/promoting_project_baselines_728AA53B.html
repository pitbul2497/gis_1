Руководство по инструменту: Продвижение контрольной версии проекта с помощью Rational ClearCase
В этой памятке по инструменту описывается присвоение уровней продвижения контрольным версиям с помощью унифицированного управления изменениями (UCM) Rational ClearCase.
Инструмент: Rational ClearCase
Взаимосвязи
Связанные элементы
  • Продвижение контрольных версий
Основное описание

Обзор

В ClearCase контрольные версии UCM обычно представляют собой стабильные конфигурации для компонента.

Контрольным версиям присваиваются уровни продвижения, которые можно использовать для определения качества или степени выполнения работы, представленной контрольной версией. Обычно уровень продвижения указывает уровень тестирования. UCM предоставляет упорядоченный набор уровней продвижения по умолчанию и также поддерживает пользовательские уровни продвижения. Изменение уровня продвижения контрольной версии называется продвижением или понижением контрольной версии.

Рекомендованная контрольная версия - это контрольная версия, утвержденная для общего использования. Рекомендованные контрольные версии используются для операций перебазирования. При перебазировании потока его конфигурация изменяется исходя из рекомендованной контрольной версии целевого потока. Одним из способов применения уровней продвижения является присвоение набору контрольных версий одного уровня продвижения статуса рекомендованных контрольных версий для определенного потока.

Эта памятка по инструменту применима к Microsoft Windows.

Этапы работы с инструментами

Эти процедуры описывают способ изменения уровня продвижения контрольной версии и процедуру присвоения контрольным версиям с определенным уровнем продвижения состояния рекомендованных контрольных версий для потока:

  1. Изменение уровня продвижения контрольной версии
  2. Исходя из уровня продвижения, задайте рекомендованную контрольную версию для потока

Дополнительная информация приведена в руководстве IBM Rational ClearCase - руководство по управлению программными проектами.

1. Измените уровень продвижения контрольной версии

  1. Выберите поток в обозревателе проектов ClearCase. Выберите Файл > Свойства для отображения окна свойств системы.
  2. Перейдите на вкладку Контрольные версии.
  3. В поле Компоненты выберите компоненты, контрольные версии которых требуется продвинуть. Контрольные версии компонентов отображаются в поле Контрольные версии. Выберите контрольную версию и щелкните на пункте Свойства для отображения окна свойств контрольной версии.
  4. На вкладке Общие щелкните на стрелке вниз в списке Уровень продвижения для отображения всех доступных уровней продвижения. Выберите новый уровень продвижения. Нажмите кнопку OK.

2. Исходя из уровня продвижения задайте рекомендованную контрольную версию для потока

  • Выберите Пуск > Программы > Rational Software > Rational ClearCase > Обозреватель проектов.
  • Выберите поток.
  • Выберите Инструменты > Рекомендованные контрольные версии для отображения списка текущих рекомендованных контрольных версий проекта.
  • В области Список поиска выберите уровень продвижения из выпадающего списка.
  • Выберите Список поиска чтобы сделать последние контрольные версии совпадающие с или превышающие выбранныйуровеньпродвижения рекомендованными контрольными версиями.   
  • Нажмите OK.
