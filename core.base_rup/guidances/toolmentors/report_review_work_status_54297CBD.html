Руководство по инструменту: Создание отчета о просмотре и состоянии работы с помощью Rational ClearQuest
Целью этой памятки по инструменту является объяснение применения Rational ClearQuest для просмотра состояния работы.
Инструмент: Rational ClearQuest
Взаимосвязи
Связанные элементы
  • Подтверждение повторного или отклоненного запроса изменения
  • Проверка запросов изменений
  • Проверка изменений в компиляции
  • Создание отчета о состоянии конфигурации
Основное описание

Обзор

ClearQuest облегчает пользователям указание в запросе типов записей для выборки из базы данных ClearQuest. Если пользователь выполняет запрос без указания критериев выбора (фильтров), ClearQuest отображает список всех записей из базы данных. Вот некоторые примеры типичных запросов:

  • Реализаторы выполняют запрос неполадок, которые им требуется исправить, и просматривают те из них, которые обладают наивысшим приоритетом, расположенные вверху списка.
  • Менеджеры проекта запрашивают записи оп дате их создания для отслеживания состояния работы тестеров.
  • Технические писатели при создании информации о выпуске запрашивают необработанные записи, связанные с проектом.

ClearQuest выполняет запрос и затем отображает набор результатов. Участник команды изучает таблицу набора результатов и выбирает записи для просмотра, изменения или применения в качестве основы для отчета или диаграммы.

Существует два типа запросов ClearQuest:

  • Администраторы ClearQuest создают общие запросы для всех пользователей
  • Пользователь может создать и сохранить частные запросы

См. электронную справку ClearQuest: Справка > Оглавление и индекс > Работа с запросами.
