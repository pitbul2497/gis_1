Руководство по инструменту: Просмотр требований с помощью Rational RequisitePro
В этой памятке описывается применение Rational RequisitePro® для облегчения обзора требований.
Инструмент: Rational RequisitePro

Связанные элементы

Основное описание

Обзор

RequisitePro облегчает просмотр требований в среде коллективной разработки проекта. Можно упорядочить всю документацию проекта и обращаться к ней из одного расположения. Затем участники группы разработчиков могут обмениваться комментариями по определенным требованиям или более широким аспектам проекта с помощью электронных обсуждений. Все элементы обсуждений хранятся в базе данных проекта и доступны для последующего просмотра. Для обзора документов требований автор может временно защитить документ в ходе просмотра или ревизии. Ревизии в последующем объединяются в проекте и делаются доступными для всех участников коллектива.

Этапы работы с инструментами

Ниже более подробно описаны следующие концепции и процедуры просмотра требований.

  1. Доступ к документации требований из единого расположения
  2. Запись изменений требований в документах
  3. Разработка диалога коллектива разработчиков проекта
  4. Защита документов требований для просмотра и ревизии

1. Доступ к документам требований

При создании или импорте документов требований в RequisitePro они интегрируются в проект управления требованиями. Все документы связаны с базой данных, что позволяет предоставить всем пользователям быстрый и централизованный доступ. Все документы требований в проекте доступны для открытия из одного диалогового окна. Можно также добавить не связанные с требованиями документы, например, глоссарий, к проекту RequisitePro для быстрого доступа командой разработчиков проекта. Кроме того сведения о требованиях доступны через различные интерфейсы, включая Microsoft® Word, представления и RequisiteWeb.

Для создания документа требований:

  1. Выберите в обозревателе пакет, в который следует поместить новый документ. Затем выберите Файл > Создать > Документ. Откроется диалоговое окно Свойства документа.
  2. На вкладке Общие введите имя и описание документа. В списке Тип документа выберите тип документа для нового документа.
  3. Нажмите кнопку OK.

Вы можете открыть один или несколько документов требований из проводника моделей, выполнив одно из следующих действий:

  • Дважды щелкните на документе.
  • Щелкните правой кнопкой на документе и выберите Открыть.

Для импорта документа Word, содержащего текст требований:

  1. Выберите Файл > Импорт.Откроется мастер импорта.
  2. Выберите документ Microsoft Word. Затем укажите путь и имя документа Word, который следует импортировать, или нажмите кнопку Обзор для перехода к нужному документу.
  3. Нажмите кнопку Далее для перехода к следующей странице мастера. При выборе опции Требования и документ RequisitePro обработает требования автоматически и пометит их в импортируемом документе. Можно обработать требования исходя из ключевых слов, разделителей текста или стилей абзацев Word.
  4. Откроется диалоговое окно Свойства документа. Введите имя и описание документа и выберите тип документа из списка. Затем нажмите кнопку OK.

Дополнительная информация

Обратитесь к следующим разделам электронной справки RequisitePro:

  • Создание документов требований (Индекс: документы > создание)
  • Открытие документов (Индекс: документы > открытие)
  • Импорт требований из документа Word (Индекс: импорт > требований из документа Word)

2. Запишите изменения требований в документах

При изменении текста требований в документе необходимо описать причину изменений. RequisitePro записывает и отслеживает сведения о ревизиях, предоставляя таким образом запись хронологии изменений требований.

Для записи изменения требования в документе:

  1. Измените текст требований.
  2. Щелкните на тексте требования и выберите RequisitePro > Требование > Изменить аннотацию. Откроется окно Описание изменения.
  3. В поле Описание изменения введите причину изменения.
  4. Нажмите кнопку OK чтобы закрыть диалоговое окно.

Информация по изменению требования доступна для всех участников проекта. Выберите RequisitePro > Свойства требования и перейдите на вкладку Ревизия. RequisitePro создает номер ревизии для требования, указывает дату и время внесения изменения, показывает автора ревизии и предоставляет описание изменения.

Дополнительная информация

Обратитесь к следующим разделам электронной справки RequisitePro:

  • Обзор ревизии (Индекс: ревизии > обзор)
  • Создание и изменение сведений о ревизии требования (Индекс: ревизии > требования)

3. Разработка диалога команды проекта

RequisitePro облегчает взаимное общение коллектива разработчиков и просмотр информации с помощью конференций, позволяя пользователям обсуждать свои требования. С помощью конференций пользователи могут быстро создать и распространить темы обсуждения - комментарии, вопросы, проблемы и уведомления об изменениях, касающиеся требований или любых аспектов проекта. Распределение этих тем может затрагивать всю команду или только определенную группу пользователей. Если для проекта включена поддержка электронной почты, отклики можно отправлять через RequisitePro или RequisiteWeb, либо с помощью любого почтового клиента SMTP. RequisitePro извещает пользователей о новых сообщениях и сохраняет нити конференций вместе со связанными требованиями для всей команды.

Конференции помогают команде рационально принимать решения и предлагать изменения. Конференции позволяют пользователям выполнять следующие действия:

  • Создавать обсуждения и связывать их с требованиями, наборами требований или всем проектом.
  • Автоматически распространять сообщения электронной почты (при поддержке этой функции) между аудиторией конференции.
  • Просматривать графическое представление нитей конференций в формате иерархической структуры с комментариями и ответами.
  • Выполнять запросы для обсуждений.
  • Изменять атрибуты обсуждений (приоритет, состояние).
  • Распечатывать обсуждения.

Для создания обсуждения:

  1. Откройте окно Обсуждение выполнив одно из следующих действий:
    • В обозревателе выберите требование и щелкните на пункте Требование > Обсуждения.
    • В представлении выберите одно или несколько требований и щелкните на пункте Требование > Обсуждения.
    • В документе Word щелкните в любом месте на требовании и выберите RequisitePro > Требование > Обсуждения.
  2. Нажмите Создать.
  3. В окне Свойства обсуждения выберите вкладки Общие, Атрибуты, Участники и Требования чтобы задать обсуждение и добавить участников. Вы можете связать обсуждение с требованиями.
  4. Нажмите OK, затем выберите Закрыть.

Для просмотра и отправки ответа в обсуждение:

  1. Выберите Проект > Просмотр обсуждений или щелкните на выделенном значке обсуждения на панели инструментов (указывающем на создание нового сообщения обсуждения).
  2. В списке обсуждений щелкните на нужном обсуждении. Слева от обсуждения с ответами отображается значок развертывания/свертывания.
  3. Щелкните на элементе чтобы прочитать его.
  4. Для ответа на выбранный элемент обсуждения нажмите кнопку Ответить.
  5. В диалоговом окне Ответ на обсуждение введите текст отклика.
  6. Нажмите кнопку OK.

Дополнительная информация

Обратитесь к следующим разделам электронной справки RequisitePro:

  • Создание обсуждений (Индекс: обсуждения > создание)
  • Чтение обсуждений (Индекс: обсуждения > просмотр)
  • Ответ на обсуждения (Индекс: обсуждения > ответ)
  • Настройка электронной почты для обсуждения (Индекс: электронная почта > обсуждения, настройка для проекта)

4. Защита документов требований для просмотра и ревизии

Функция автономного создания в RequisitePro расширяет процесс просмотра, позволяя авторам изымать документ из проекта и изучать его в Microsoft Word. Доступная только для чтения копия при этом остается в RequisitePro; она защищена от прочих участников команды до завершения просмотра. Автономную копию документа, являющуюся обычным документом Word, можно распространять между соавторами для просмотра и изменения. Получившийся в результате этого документ можно опубликовать обратно и цикл просмотра будет завершен.

Пока документ является автономным, пользователь, сделавший его таким, может использовать Word чтобы отметить (создать), снять отметку и удалить требования в документе. Пользователь может направить документ прочим участникам команды для просмотра. (Рекомендуется иметь только одну копию автономного документа и передавать ее всем членам команды по очереди). Другие члены команды могут просматривать документ в RequisitePro, но не редактировать его.

При возврате документа RequisitePro выполняет следующие действия:

  • Создает требования из помеченного текста.
  • Обновляет базу данных, добавляя изменения, внесенные в существующие требования. К ним могут относиться удаления, изменения и добавления. При удалении требований владелец автономного сеанса получит запрос на подтверждение удаления.
  • Все изменения требований, для внесения которых у пользователя отсутствуют права доступа, будут проигнорированы. В таком случае весь документ будет возвращен к исходному состоянию.

Чтобы сделать документ требований автономным:

  1. Выберите Инструменты > Автономные документы.
  2. Щелкните на документе, который требуется сделать автономным. (Можно выбрать сразу несколько документов).
  3. Щелкните на Сделать автономным. Появится окно Сведения о сохранении автономного документа.
  4. Введите причину создания автономного документа и путь к каталогу, в котором требуется его сохранить (или нажмите кнопку Обзор чтобы указать каталог).
  5. При сохранении нескольких автономных документов выберите переключатель Применить ко всем чтобы применить записи Причина и Расположение ко всем документам. Отключите этот переключатель чтобы задать параметры для каждого документа по отдельности.
  6. Нажмите кнопку OK.

Чтобы опубликовать документы обратно:

  1. Выберите Инструменты > Автономные документы.
  2. В списке Автономные документы выберите документ, который требуется опубликовать обратно.
  3. Выберите опцию Опубликовать.
  4. В поле Описание изменений введите описание изменений, внесенных в выбранный документ, пока он был автономным. При публикации сразу нескольких документов и применении изменений ко всем последующим документам выберите переключатель Применить ко всем.
  5. Нажмите кнопку OK. Нажмите Да для принятия обновленного документа.

Дополнительная информация

Обратитесь к следующим разделам электронной справки RequisitePro:

  • Создание автономных документов (Индекс: автономное изменение > создание автономных документов)
  • Публикация автономных документов (Индекс: автономное изменение > публикация автономных изменений)
