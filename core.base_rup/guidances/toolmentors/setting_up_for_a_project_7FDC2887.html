Основное описание

Обзор

Настройка XDE для проекта означает закладку фундамента, позволяющего участникам команда разработчиков одновременно трудиться над одной моделью. Для разработки сложных систем группам аналитиков, архитекторов и разработчиков необходимо иметь возможность просматривать и оценивать ход разработки глобально, при этом одновременно работая над своими фрагментами проекта. Для успешного управления средой, в которой несколько участников команды имеют различные права доступа к одной модели, требуется:

  • Формулирование рабочей стратегии для управления деятельностью команды.
  • Наличие инструментов, поддерживающих эту стратегию.

Следующие действия позволяют задать эту основу:

  • Настройте шаблоны и значения по умолчанию XDE (необязательно)
  • Создайте проекты и модели XDE
  • Разбейте модели XDE на управляемые блоки

Настройте шаблоны и значения по умолчанию XDE (необязательно)

В последующих шагах будут созданы модели XDE с помощью стандартных шаблонов, содержащих базовые структуры пакетов, отвечающие Рекомендациям по структуре модели Rational XDE.

При необходимости можно создать собственные специализированные шаблоны. Rational XDE позволяет задать свойства и опции, такие как шрифты, цвета, стили линий, опции хранения файлов, способы представления стереотипов и прочих сведений, а также применяемые профайлы.

Создайте проекты XDE и модели XDE

Термин "проект" в основном используется в RUP для ссылки на весь процесс разработки программного обеспечения.

В Microsoft .NET процесс разработки ПО называется "решением". Решение состоит из одного или нескольких проектов XDE (а точнее из проектов .NET), содержащих модели XDE, исходный код и прочие поля.

В Java Platform Edition весь процесс разработки ПО называется "приложением". Приложение J2EE состоит из одного или нескольких проектов XDE, содержащих модели XDE, исходный код и прочие файлы.

Точно так же, продукты работы "модели" в RUP являются концептуальными продуктами работы, которые могут состоять из нескольких моделей XDE, связывающих несколько проектов XDE.

Создайте проекты XDE, указанные в Рекомендациях по структуре модели Rational XDE, выполнив действия, описанные в документе Создание проектов.

Добавьте дополнительные модели к проекту при необходимости, выполнив действия, описанные в документе Создание моделей  , используя в качестве инструкции Рекомендации по структуре модели Rational XDE.

Справочник: Настройка и управление средой моделирования.

Разбиение моделей XDE на управляемые блоки

XDE поддерживает разбиение модели XDE на управляемые блоки, позволяя разделить модель XDE на отдельные файлы, называемые управляемые блоки.
