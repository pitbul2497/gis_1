Основное описание

Обзор

Стратегии проекта позволяют внедрить желаемую практику выполнения разработки среди участников команды. Задав стратегии вы сможете, к примеру, минимизировать неполадки, которые могут возникнуть при интеграции работы, задав стратегию, требующую от разработчиков обновлять рабочие области с рекомендованными контрольными версиями проекта перед до ставкой работы в общий поток. Это снижает возможность возникновения ошибок в коде.

Стратегии проекта задаются при создании проекта UCM. Их можно изменить в любое время.

Эта памятка по инструменту применима к Microsoft Windows.

Этапы работы с инструментами

Для настройки стратегий для проектов UCM существует две процедуры.

  • Задать стратегии для новых проектов
  • Задать или изменить стратегии для существующих проектов

1. Задать стратегии для новых проектов

  • На панели инструментов Windows выберите Пуск > Программы > Rational Software > Rational ClearCase > Project Explorer.
  • В меню Файл выберите Создать > Проект для запуска Мастера создания проекта. При появлении приглашения, выберите стратегии для внедрения в данный проект.

Дополнительные сведения приведены в разделе Управление программными проектами электронной справки.

2. Задайте или измените стратегии для существующих проектов

  1. В обозревателе проектов выберите изменяемый проект.
  2. Щелкните правой кнопкой мыши на проекте и выберите Стратегии.
  3. На других вкладках можно просмотреть и выбрать стратегии проекта.

Описание стратегий проекта приведено в разделах контекстной справки окна Стратегии.

Дополнительные сведения о стратегия, доступных для создания в проектах UCM, приведены в главе "Настройка стратегий" руководства Руководство IBM Rational ClearCase по управлению программными проектами.
