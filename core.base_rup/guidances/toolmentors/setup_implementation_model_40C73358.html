Руководство по инструменту: Настройка модели реализации с помощью Rational ClearCase
В этой памятке по инструменту описывается настройка каталога модели реализации с помощью Rational ClearCase.
Инструмент: Rational ClearCase
Взаимосвязи
Связанные элементы
  • Написание плана управления конфигурацией (CM)
  • Настройка среды управления конфигурацией (CM)
  • Создание структуры модели реализации
Основное описание

Терминология

Между терминологией RUP  и ClearCase имеются некоторые различия. Следующие определения связанных с инструментами терминов помогут вам разобраться в различиях.

  • Управление конфигурацией: в контексте ClearCase использование термина Управление конфигурацией означает управление версиями и компоновками. В RUP используются определения IEEE и ISO для управления конфигурацией (CM), также содержащей управление изменениями как часть CM.
  • Управление конфигурацией и изменениями: в ClearCase и Rational ClearQuest управление конфигурацией и изменениями является синонимом определения управления конфигурацией в RUP. В RUP эти термины используются как синонимы.
  • Проект: проект ClearCase является аналогом Хранилища проекта в RUP.

Обзор

В этой памятке по инструменту описывается создание структуры в среде разработки для упорядочивания и хранения продуктов работы элемента конфигурации, связанного с реализацией и доставкой программного продукта. Физическое представление подсистем в среде разработки задает "структуру каталога продукта" и, соответственно, модель реализации программного обеспечения.

Перед тем, как начать пользоваться ClearCase, важно понимать, будете ли вы использовать ClearCase Unified Change Management (UCM), отдельную модель применения или базовый ClearCase, предоставляющий набор инструментов. используемых для создания прочих моделей применения.

В этой памятке по инструменту рассматриваются задачи как для ClearCase UCM, так и для базовой версии ClearCase, тог да как все прочие памятки по ClearCase, представленные в RUP, используют модель UCM.

Эта памятка по инструменту применима к Microsoft Windows.

Этапы работы с инструментом

Чтобы задать модель реализации в Rational ClearCase:

  1. Создайте хранилища управления конфигурацией (CM) (ClearCase VOB).
  2. Импортируйте существующие файлы и каталоги в эти VOB.
  3. (Необязательно) Создайте административный VOB для хранения определений объектов глобальных типов.

1. Создайте хранилища управления конфигурацией (CM) (ClearCase VOB)

Базы объектов с версиями (VOB) используются для хранения версий файлов, каталогов и прочих объектов. Они служат хранилищами для информации об управлении конфигурацией. Обычно VOB создаются для представления подсистем, заданных архитектурой.

Для запуска мастера создания VOB (для каждого этапа мастера доступна контекстная справка):

  1. На панели задач Windows выберите Пуск > Программы > Rational Software > Rational ClearCase > Администрирование > Создать VOB.
  2. Откроется Мастер создания VOB. Следуйте инструкциям для создания ClearCase VOB. Прежде чем начать полезно узнать о соглашениях об именах VOB, используемых в вашей системе или проекте, и уточнит ь место расположения для хранения VOB.
  3. В оставшихся шагах мастера выполните следующие действия:
    • Задайте имя VOB и укажите, должен ли он быть компонентом UCM или VOB проекта (PVOB).
    • Укажите место хранения VOB.
    • Выберите Опции.

  Подробные сведения о создании VOB приведены в электронной справке мастера создания VOB ClearCase.

  Обзор настройки VOB приведен в главе "Настройка VOB ClearCase" в руководстве ClearCase Руководство администратора.

2. Импортируйте существующие файлы и каталоги в VOB

Работая в представлении ClearCase, созданном для проекта, создайте нужную структуру каталогов верхнего уровня в VOB и скопируйте существующие файлы и каталоги во все VOB для создания исходного набора элементов конфигурации разработки  (элементов Rational ClearCase).

  1. В обозревателе ClearCase Explorer выберите каталоги и файлы для импорта.  
  2. Щелкните правой кнопкой мыши для вызова меню и выберите в нем Добавить к управлению исходным кодом. Будут созданы элементы ClearCase для всех выделенных объектов.

  Сведения об импорте больших объемов файлов приведены в описании команды clearimport в руководстве ClearCase Справочник команд.

3. (Необязательно) Создайте административный VOB для хранения определений объектов глобального типа

При необходимости можно создать административный VOB для хранения определений объектов глобального типа. Создайте административный VOB для хранения определений объектов глобального типа, используемых в определении контрольных версий подсистем.

Для создания административного VOB запустите мастер создания VOB, описанный ранее в Шаге 1 и следуйте инструкциям.

   Дополнительные сведения о работе с административными VOB приведены в разделе "Применение административных VOB и глобальных типов" в руководстве ClearCase Руководство администратора.
