Основное описание

Обзор

Для создания нового проекта в RequisitePro используется шаблон проекта. Все шаблоны проекта, поставляемые с RequisitePro (за исключением пустого шаблона) содержат предустановленные типы документов, типы требований и атрибуты, используемые создаваемым проектом. С помощью шаблона прецедента можно создать проект, с помощью этого шаблона можно использовать все функции RUP. В новом проекте можно немедленно начать создавать новые документы требований в Microsoft® Word или отдельные требования в представлениях. Обратитесь к ознакомительным разделам в начале справочного приложения Let's Go RequisitePro для получения дополнительных сведений о начале работы в проекте RequisitePro.

При наличии лицензионных Rational Suite или прочих продуктов Rational можно интегрировать проект RequisitePro с работой в этих продуктах.

  • Информация об интеграции RequisitePro с Rational Rose® приведена в Памятке по инструменту: управление вариантами в Rational Rose и Rational RequisitePro.
  • Информация о добавлении проекта RequisitePro в проект Rational приведена в документе Rational Suite - руководство администратора на компакт-диске с документацией Rational Solutions for Windows. Необходимо добавить проект RequisitePro в проект Rational для интеграции RequisitePro с Rational ClearQuest®.
  • Сведения о настройке интеграции между RequisitePro и ClearQuest приведены в документе Rational Suite - руководство администратора на компакт-диске с документацией Rational Solutions for Windows. Пошаговые инструкции по связыванию записей ClearQuest с требованиями RequisitePro обратитесь к Памятке по инструменту: управление запросами заинтересованных лиц с помощью Rational ClearQuest и Rational RequisitePro.

О шаблоне проекта прецедента RequisitePro

Шаблон прецедента предоставляет следующие типы требований RequisitePro, связанные с наиболее важными элементами трассируемости, согласно Концепции: трассируемость. Шаблон также предоставляет дополнительные типы требований, чаще всего используемые в проектах.

Элемент трассируемости RUP Тип требований RequisitePro
Потребности пользователей/заинтересованных лиц (из Видения) STRQ: тип требований запроса заинтересованных лиц.  Потребность определяется путем присвоение приоритета. ("Высокий" приоритет указывает на "потребность").
Компонент продута (из Видения) FEAT: тип требования компонента
Дополнительное требование (из Дополнительных спецификаций)  SUPL: тип дополнительного требования
 Вариант UC: тип требований прецедента
Раздел варианта (разделы подробного Варианта) UC: тип требования варианта - как дочернее требований родительского варианта (см. Обзор иерархических взаимоотношений в электронной справке RequisitePro).
Элемент проектирования (из Модели проектирования) Управляется в Rose. См. Интегрированное управление прецедентом в электронной справке RequisitePro.

Вы можете добавить отдельный тип требований для раздела варианта вместо использования иерархических требований вариантов. Это удобно если требуется задать различные атрибуты для разделов варианта проекта, присвоить права доступа или правила трассируемости (например, убедиться, что тестовые варианты трассируются к разделам вариантов, а не к вариантам).

Этапы работы с инструментами

Чтобы настроить RequisitePro для проекта выполните следующие действия:

  1. Создайте проект в RequisitePro
  2. Настройте защиту проекта

1. Создайте проект в RequisitePro

  1. Выберите Файл > Создать > Проект. Откроется окно Создать проект.
  2. Выберите значок Шаблон прецедента и нажмите кнопку OK. Откроется окно Свойства проекта RequisitePro.
  3. Введите имя проекта. В поле Каталог будет показан каталог с файлами проекта. Примите каталог по умолчанию или нажмите кнопку Обзор для выбора другой папки.
  4. В списке База данных выберите соответствующую базу данных для применения с проектом. При выборе Oracle или SQL Server выберите Свойства; введите нужную информацию в окне Свойства базы данных и нажмите кнопку OK.
  5. Введите описание нового проекта и нажмите OK чтобы закрыть окно.
  6. При появлении запроса создания нового каталога проекта RequisitePro нажмите Да. RequisitePro создаст новый проект.
  7. Выберите Закрыть чтобы закрыть диалоговое окно Создать проект. В новом проекте будут присутствовать предустановленные типы документов, типы требований и атрибуты. Для их просмотра или изменения выберите проект в обозревателе, выберите в меню Файл > Свойства и перейдите на соответствующие вкладки.

2. Настройка защиты проекта

Так как RequisitePro позволяет нескольким пользователям обращаться одновременно к одним документа проекта и базам данных, защита проекта является важным вопросом. Защиту проекта можно включить и выключить, в зависимости от потребностей группы. При включенной защите пользователи принадлежат группам, а администраторы RequisitePro присваивают группам права доступа. Права доступа определяют тип доступа к проекту, которым обладают пользователи.

Создать новый проект могут все пользователи RequisitePro. При создании нового проекта вы воспринимаетесь как администраторы проекта и автоматически добавляетесь в группу Администраторы. Создав проект вы (и прочие администраторы проектов) могут создавать другие группы и пользователей.

Если защита отключена, открыть проект RequisitePro может любой пользователь. При первом открытии незащищенного проекта RequisitePro позволяет войти под именем пользователя, указанным в поле Вход в проект по умолчанию в окне Опции (выберите Инструменты > Опции), под именем пользователя Windows или под любым указанным именем пользователя. Имя пользователя добавляется в список группы Администраторы проекта.

Все администраторы проекта могут включить защиту, изменить списки Пользователи и Администраторы и присвоить пароли другим пользователям. Если защита включена, окно Вход в проект исчезает при открытии пользователем проекта.

Чтобы задать защиту проекта в RequisitePro:

  1. Выберите Файл > Администрирование проекта > Защита. Откроется окно Защита проекта.
  2. Выберите переключатель Включить защиту для этого проекта. В список Группа будут добавлены три группы по умолчанию: Удаленные пользователи, Администраторы и Пользователи.

Для добавления группы пользователей:

  1. В окне Защита проекта нажмите кнопку Добавить под списком Группы. Откроется окно Права доступа группы.
  2. Введите имя группы.
  3. Выберите права доступа к проекту для группы.
  4. Измените права доступа к документу и типу требований.
  5. Нажмите OK.

Для добавления группы пользователей:

  1. В окне Защита проекта в списке Группы выберите группу, в которую следует добавить пользователя.
  2. Нажмите кнопку Добавить рядом со списком Пользователи группы. Откроется окно Добавить пользователя.
  3. Введите имя пользователя, пароль и подтвердите пароль.
    Примечание: поля Пароль и Подтверждение пароля зашифрованы, в них отображаются особые символы для введенных вами знаков, максимальная длина пароля составляет 14 символов. Помните, имя пользователя и пароль следует вводить с учетом регистра.
  4. При наличии адреса электронной почты у пользователя укажите и его. Он необходим для участия в конференциях RequisitePro через электронную почту.
  5. Нажмите OK. Окно Защита проекта откроется повторно. Пользователь будет добавлен к списку.
  6. Нажмите OK.
