Руководство по инструменту: Настройка тестовой среды в Rational TestFactory
В этой памятке по инструменту описывается выполнение действий по настройке среды Rational TestFactory для реализации созданных тестовых сценариев для тестируемого приложения (AUT).
Инструмент: Rational TestFactory
Взаимосвязи
Связанные элементы
  • Настройка инструментов
  • Разработка справочных материалов
  • Реализация теста
  • Создание реализация теста
  • Установка критериев пригодности к тестированию
Основное описание

Обзор

Перед тем, как использовать Rational TestFactory для автоматического создания тестовых сценариев необходимо настроить конфигурацию тестовой среды. TestFactory создает тестовые сценарии исходя из "схемы приложения", иерархического списка "объектов UI", представляющих окна и управляющие элементы пользовательского интерфейса в тестируемом приложении (AUT).

Эта памятка по инструменту применима для систем с Windows 98/2000/NT 4.0.

Для настройки тестовой среды в Rational TestFactory выполните следующие действия:

  1. Задайте проект в Rational TestFactory и реализуйте AUT
  2. Создайте схему тестируемого приложения
  3. Просмотрите и уточните схему приложения

1.   Задайте проект в Rational TestFactory и реализуйте AUT

При первом открытии проекта в Rational TestFactory необходимо предоставить сведения о проекте и тестируемом приложении (AUT). Основные функциональные области TestFactory остаются недоступными до тех пор, пока вы не зададите сведения о проекте.

Для измерения охвата кода выполняемого тестового сценария AUT должен содержать точки реализации - счетчики, подсчитывающие части кода, охваченные тестовым сценарием. Rational TestFactory использует сведения из реализованных файлов для подсчета охвата данных на базе кода для тестовых сценариев, создаваемых TestFactory, и тестовых сценариев Robot, воспроизводимых из TestFactory.

  Обратитесь к следующим разделами справки Rational TestFactory:

  • Запуск TestFactory
  • Определение сведений для нового проекта
  • Встраивание тестируемого приложения

2.   Схема тестируемого приложения

Хорошо разработанная схема приложения является основой при создании тестовых сценариев в Rational TestFactory. "Создатель схем приложения" - это процесс, создающий схемы приложения, досконально исследующий пользовательский интерфейс тестируемого приложения (AUT). Все окна и управляющие элементы изучаются и сравниваются с известными классами и подклассами, хранящимися в "библиотеке пользовательского интерфейса (UI)". Исходя из результата сравнения, процесс создания схем создает объект UI, являющийся экземпляром совпадающих классов, и помещает его в схему приложения.

Все заданные в библиотеке UI классы имеют определенные наборы "свойств объектов UI", связанных с ними. При создании объекта UI, Создатель схем приложения присваивает объекту набор свойств, связанных с его классом. Эти свойства определяют управляющий элемент в AUT, который представляет объект, и сообщает Rational TestFactory как выполнять управляющий элемент при тестировании и преобразовании.

Если AUT содержит окно входа в систему, требующего ввода, такого как имя пользователя и пароль, необходимо предоставить эти сведения в Создатель схем приложения.

При завершении создания схемы в обзоре процедуры будут указаны окна и управляющие элементы AUT, добавленные Rational TestFactory в схему.

   Обратитесь к следующим разделами справки Rational TestFactory:

  • Создайте схему AUT с помощью мастера создания схем приложений
  • Создайте схему AUT с помощью значка Создать схему!
  • Создание схемы AUT, имеющего окно входа в систему

3.   Просмотрите и уточните схему приложения

Первая версия схемы приложения может оказаться неточным представлением всех управляющих элементов в AUT. Схема приложения может быть неполной или для объектов UI может понадобиться повторная классификация.

Хорошо разработанная схема приложения является критически важной для создания качественных тестовых сценариев. Rational TestFactory может протестировать только управляющие элементы в AUT, представленные объектами UI на схеме приложения. Просмотрев и уточнив схему приложения перед началом тестирования можно повысить область и качество создаваемых тестовых сценариев.

Можно просмотреть схему приложения, сравнив окна и управляющие элементы, показанные в AUT, с объектами UI на схеме приложения. Заметив не представленные на схеме управляющие элементы AUT при необходимости протестировать их нужно определить и устранить причину отсутствия на схеме этих элементов.

   Обратитесь к следующим разделами справки Rational TestFactory:

  • Просмотрите схему приложения
  • Определение необнаруженных управляющих элементов
  • Обработка общих объектов
  • Применение объектов взаимодействия для улучшения схемы приложения
