Обзор

Ниже приведен обзор действий, выполняемых для описания взаимоотношений между субъектами и вариантами:

  1. Обобщение субъектов документа
  2. Взаимоотношения добавления документа между вариантами
  3. Взаимоотношения расширения документа между вариантами
  4. Обобщение прецедентов документа

Подробные сведения о диаграммах прецедентов приведены в:

  • Раздел Диаграммы прецедентов (Обзор) электронной справки Rational Rose.

  • Глава 4, введение в диаграммы и Глава 7, диаграммы прецедентов и спецификации. Справочник: Применение Rational Rose.

1. Обобщения субъектов документа

Можно вставить взаимосвязь обобщения между бизнес-акторами в диаграмме прецедентов с помощью инструмента рисования обобщения, расположенного на панели инструментов диаграммы прецедентов. Создав взаимосвязь вы можете описать ее, добавив текст в поле описания в окне Спецификация обобщения.

2. Взаимосвязи добавления документа между вариантами

Для создания взаимосвязи добавления между двумя вариантами в диаграмме вариантов вначале необходимо создать зависимость между двумя вариантами, затем присвоить ей стереотип добавления.Стереотип добавления используется когда один вариант использует функции другого.Используемый вариант обычно содержит функции, которые могут требоваться многим другим вариантам.

Создав зависимость между вариантами можно описать взаимосвязь, добавив текст в поле Описание в окне Спецификация зависимости.

3. Взаимосвязи расширения документа между вариантами

Для создания взаимосвязи расширения между двумя вариантами в диаграмме вариантов вначале необходимо создать зависимость между двумя вариантами, затем присвоить ей стереотип расширения.Стереотипы расширения применяются для выражения необязательного или условного поведения варианта.

Создав взаимосвязь между вариантами можно описать взаимосвязь, добавив текст в поле Описание в окне Спецификация зависимости.

4. Обобщения прецедентов документа

Можно создать взаимосвязь обобщения между бизнес-вариантами при которой один вариант предоставляет общие функции (например, при наличии абстрактного варианта, предоставляющего общие функции конкретному варианту).

Создав взаимосвязь обобщения между вариантами можно описать взаимосвязь, добавив текст в поле Описание в окне Спецификация обобщения.
