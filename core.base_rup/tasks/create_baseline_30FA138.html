Эта задача определяет, что такое Контрольная версия, и когда Контрольные версии должны создаваться.
Дисциплины: Управление конфигурацией и изменениями

        Основное описание

        Подходящим кандидатом для контрольной версии является набор файлов и каталогов под управлением версии, которые были разработаны, интегрированы и выпущены вместе. Контрольная версия определяет одну и только одну версию элемента. Составные контрольные версии - это группы контрольных версий, доставляемые рабочими группами, которые работают над разными подсистемами общего проекта.

        Часть итерационной методологии разработки состоит в постоянной компоновке исполняемого продукта. Контрольную версию надо рассматривать как передний фронт разработки, который содержит необходимые рабочие продукты для последующей разработки. Таким образом, в конце каждой итерации должны непременно создаваться контрольные версии.

        Создание контрольных версий проекта согласуется с идентификацией конфигурации. Шаг создания контрольных версий и архивирования определен в Задача: Установить стратегии CM.
