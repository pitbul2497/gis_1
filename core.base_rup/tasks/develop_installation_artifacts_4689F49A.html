Задача: Разработка установочных продуктов
Эта задача описывает, как создать все программное обеспечение, которое требуется для того, чтобы быстро, легко и безопасно установить или удалить из системы продукт без влияния на другие приложения и параметры системы.

        Основное описание

        При разработке рабочих продуктов установки Конструктор должен решить, как продукт должен быть упакован, настроен и установлен.

        Обычный набор рабочих продуктов установки включает в себя:

        • Сценарии установки
        • Файлы установки
        • Инструкции установки

        Рабочие продукты установки должны разрабатываться вслед за разработкой других рабочих продуктов. Обычно требуется иметь отдельный набор рабочих продуктов установки для каждой платформы, на которую будет устанавливаться продукт.

        В самом простом сценарии установки отдельный пользователь устанавливает программное обеспечение на отдельный персональный компьютер (узел). Однако, в больших организациях обычно существует специальная роль (сетевой администратор) для задач управления программным обеспечением на уровне всей организации. В этом случае, важно иметь возможность из одного расположения устанавливать и обновлять программное обеспечение на всех рабочих станциях быстро и легко. За это удобство приходится расплачиваться сложностью программного обеспечения установки, требуемого для распределения программного обеспечения и настройки рабочих станций предприятия с одного сервера.  

        Разработчик может применить некоторые коммерческие пакеты, предоставляющие шаблоны и заготовки объектов интерфейса для легкой подготовки программного обеспечения к установке. Эти пакеты можно настроить, чтобы они показывали рисунки продукта и корпоративные логотипы.
