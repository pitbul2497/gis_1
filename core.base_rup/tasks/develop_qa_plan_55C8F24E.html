        Основное описание

        План контроля качества - это составной документ, который содержит все необходимые данные для выполнения задач по контролю качества проекта. Хотя некоторая информация из Плана контроля качества дублируется в Плане разработки программного обеспечения, необходимо все же разработать оба этих плана, потому что у каждого из них свое назначение.

        План контроля качества содержит график проведения проверок и процедур контроля за правильностью выполнения проекта. Критерии правильности установлены в различных сопутствующих планах, на которые опирается план контроля качества. Другими словами, план контроля качества содержит обзор качества выполнения планов проекта, тогда как план разработки программного обеспечения содержит обзор планируемых результатов.

        При выполнении этой задачи руководитель проекта разрабатывает и/или редактирует график контроля качества, определяет его целесообразность и приемлемость и согласовывает с составителями сопутствующих планов.

        Шаги
        Сформулировать характеристики качества проекта

        Руководитель проекта может не формулировать характеристики качества будущего продукта, но он должен убедиться, что они сформулированы и согласованы заказчиком и внесены в спецификации требований к программному обеспечению. У компании-разработчика в заявлении о политике обеспечения качества может быть указан стандартный набор задач по обеспечению качества. Руководитель проекта может использовать их в качестве основы для формулировки своих задач.

        По возможности, характеристики качества лучше указывать в измеряемых величинах. Например:

        • "Известно ноль неисправностей с уровнем серьезности 1" (...и добавить определение уровня серьезности 1)
        • "Максимальное время ответа - 3 секунды"
        • "Пользователь может установить программное обеспечение и начать ввод информации профайла в течение одного часа."
        Определить роли и наборы обязанностей по контролю качества

        На этом этапе определите принципы работы, а также роли и наборы обязанностей, которые нужно будет распределить для выполнения этих задач. В частности, определите иерархию отчетности о результатах проведения процедур контроля качества. Во многих случаях лицо, ответственное за выполнение задачи по контролю качества, должно направлять отчеты напрямую в контрольный совет проекта. В Rational Unified Process рекомендуется, чтобы за качество продукта, выполнение проверок и процедур контроля, утверждение планов и проведение проверок, указанных в разделе Проверки и процедуры контроля Плана контроля качества, отвечал Комитет по разработке программного обеспечения.

        Согласовать план контроля качества с составителями сопутствующих планов

        План контроля качества опирается на другие планы, в которых описаны стандарты проекта и принципы выполнения сопутствующих процессов (например, управление конфигурацией). На основе этих сведений определяются типы будущих проверок качества и частота их проведения. К сопутствующим планам обычно относятся следующие:

        • План ведения документации
        • План измерения показателей
        • План управления рисками
        • План устранения неполадок
        • План управления конфигурацией
        • План разработки программного обеспечения
        • План тестирования
        • План управления подрядчиками
        Разработать список задач по контролю качества и график их выполнения

        Разработайте список задач по контролю качества. Обычно к ним относятся следующие типы проверок:

        • Контроль/проверка планов проекта для определения, выполняются ли они по установленному графику.
        • Контроль/проверка проекта, чтобы определить, что работа ведется согласно разработанным планам проекта.
        • Утверждение допустимых отклонений от стандартных процедур.
        • Оценка развития процесса.

        Руководитель и контрольный совет проекта вместе разрабатывают график проведения проверок и процедур контроля качества. Этот график заносится в план проекта и план итерации, на которые может ссылаться план контроля качества. В зависимости от условий договора заказчик может запрашивать проведение процедур контроля.
