Задача: Проверка вех жизненного цикла
В этой задаче рассмотрен процесс поверки состояния проекта по завершении этапа и определения возможности перехода к следующему этапу.

        Основное описание

        По завершении конечной итерации каждого этапа выполняется проверка вехи жизненного цикла, позволяющая определить возможность перехода к следующему этапу проекта. Она представляет собой точку повторной синхронизации ожиданий технических специалистов и руководителей, однако рассматриваемые неполадки должны относиться к области управления проектом (основные технические неполадки должны быть устранены в конечной итерации этапа), а также в последующей процедуре Задача: Подготовка к закрытию этапа.

        Проверка выполняется для каждой главной вехи, в частности:

        • Веха целей жизненного цикла. Выполняется в конце начального этапа.
        • Архитектура жизненного цикла. Выполняется в конце этапа уточнения.
        • Веха начальных рабочих возможностей. Выполняется в конце этапа построения. Ссылка: Этап построения
        • Веха выпуска продукта. Выполняется в конце этапа внедрения. Ссылка: Этап внедрения

        Вопросы для рассмотрения

        Список вопросов для рассмотрения по умолчанию приведен в разделе Оценка состояния, например:

        • выполнены ли основные задачи этапа (запланированные возможности, уровень качества и рабочие продукты)?
        • допустим ли профайл рисков проекта для перехода к следующему этапу?
        • приемлема ли область действия проекта для всех заинтересованных лиц?
        • находятся ли контрольные версии проекта в известном состоянии (в соответствии с контролем конфигурации)?
        • приемлемы ли текущие расходы и расписание проекта?

        Кроме того, можно проверить экономическое обоснование проекта, которое предварительно было обновлено с учетом изменений области и рисков, с целью уточнения предположений и выводов в свете изменений бизнес-контекста и общего системного контекста проекта.

        Если в конце этапа завершается действие контракта, то особое внимание следует уделить финансовым вопросам.

        Шаги
        Планирование собрания, посвященного проверке вехи жизненного цикла

        В собрании проверки вехи жизненного цикла принимают участие представители заказчика, группа менеджеров проекта (руководитель проекта и руководители групп, отвечающих за различные функциональные области), а также контрольный совет проекта.

        После определения участников собрания следует запланировать дату и время его проведения. Важно предоставить участникам достаточное время для проверки материалов, на основе которых будет приниматься решение об утверждении.

        Распределение материалов собрания

        Перед собранием участникам следует передать материалы для проверки. Рекомендуется отправлять материалы заблаговременно, предоставив достаточное время для их проверки.

        Проведение собрания, посвященного проверке вехи жизненного цикла

        В ходе собрания основное внимание уделяется оценке состояния и экономическому обоснованию проекта. См. раздел Вопросы для рассмотрения.

        По окончании собрания участники должны разрешить или запретить переход к следующему этапу. Если оставшиеся неполадки незначительны, заказчик может принять продукт при условии выполнения утвержденных действий по их устранению. В этом случае руководитель проекта может начать новую итерацию для обработки неполадок в соответствии с их значимостью, либо отложить их до конечной итерации (различие заключается в объеме планирования). Если результаты этапа неприемлемы, руководителю проекта может потребоваться начать новую итерацию, либо переложить ответственность за устранение неполадки на заказчика или контрольный совет проекта.

        Ниже перечислены возможные результаты собрания, посвященного проверке вехи жизненного цикла:

        Этап принят  Представитель заказчика согласен, что проект соответствует ожиданиям этапа и разрешает переход на следующий этап.  
        Условное принятие  Представитель заказчика разрешает переход на следующий этап при условии выполнения конкретных действий по устранению неполадок.  
        Этап не принят  Проект не соответствует ожиданиям этапа: планируется дополнительная итерация, либо заинтересованные лица пересматривают контракт для изменения области действия или отмены проекта.  
        Запись решения

        По окончании собрания создается протокол собрания, в который включаются важные обсуждения, вопросы и результаты проверки вехи жизненного цикла. Если этап не был принят, предварительно планируется дополнительное собрание (если проект продолжается). Точная дата назначается после планирования дополнительной итерации.
