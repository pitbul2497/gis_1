Задача: Проверка проекта контрольным советом проекта (PRA)
В этой задаче рассмотрен процесс проверки состояния выполнения проекта контрольным советом проекта.
Дисциплины: Управление проектом

      Основное описание

      Проверка проекта контрольным советом проекта - это регулярное плановое собрание, на котором обсуждаются текущее состояние проекта, обнаруженные неполадки и риски. Кроме того, на этом собрании рассматриваются вопросы, выходящие за пределы полномочий руководителя проекта.

      Шаги
      Планирование собрания, посвященного проверке проекта контрольным советом проекта

      В этом собрании принимают участие участники контрольного совета проекта и группа управления проектом (руководитель проекта, а также руководители групп, отвечающих за различные функциональные области). Контрольный совет проекта представляет собой организационный элемент, описанный в плане разработки программного обеспечения.

      После определения участников собрания следует запланировать дату и время его проведения. Важно предоставить участникам достаточное время на ознакомление с материалами собрания.

      Распределение материалов собрания

      Перед собранием участникам следует передать оценку состояния (полученную в результате выполнения задачи Отчет о состоянии). Рекомендуется заблаговременно отправлять эту информацию, оставив достаточное время на знакомство с ней.

      Проведение собрания, посвященного проверке проекта контрольным советом проекта

      На собрании руководитель проекта докладывает контрольному совету проекта оценку состояния выполнения. Вопросы относительно состояния выполнения проекта можно обсудить непосредственно на этом собрании, либо передать группе управления проектом в качестве вопроса, требующего внимания. Участники собрания могут обсудить возникающие неполадки и передать ответственность за них контрольному совету проекта или группе управления проектом. Вопросы, требующие внимания, заносятся в протокол проверки для дальнейшего рассмотрения.

      Доклад руководителя проекта может охватывать следующие области:

      • Достигнутые большие вехи проекта.
      • Отклонения от целей, намеченных в плане разработки программного обеспечения.
      • Отклонения, связанные с расписанием/трудозатратами.
      • Отклонения расходов относительно бюджета.
      • Изменение оценки объема работы.
      • Изменения показателей качества.
      • Состояние рисков проекта:
      • Существующие риски, которые были реализованы.
      • Новые риски.
      • Возникающие неполадки - отдельные неполадки руководитель проекта должен передавать на рассмотрение контрольному совету проекта.
      • Обзор результатов предыдущих проверок - текущее состояние неполадок, обнаруженных в ходе предыдущих проверок
      • Ближайшие вехи проекта.
      Создание протокола

      По окончании собрания создается протокол собрания, в который заносятся важные обсуждениям и вопросы, требующие внимания. Руководитель проекта передает вопросы, требующие внимания, ответственным за планирование и распределение работы, при необходимости создавая запросы изменений и рабочие задания.
