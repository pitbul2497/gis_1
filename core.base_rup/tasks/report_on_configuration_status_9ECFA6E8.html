Задача: Создание отчета о состоянии конфигурации
В этой задаче рассмотрен процесс создания отчетов о состоянии предложенных изменений и их реализации.
Дисциплины: Управление конфигурацией и изменениями
Назначение
  • Поддержка задач учета состояния конфигурации проекта.
  • Повышение эффективности проверки продукта за счет применения задач мониторинга дефектов и создания отчетов.
  • Организация сбора данных и создания отчетов для контроля хода разработки и отслеживания тенденций.
Взаимосвязи
РолиОсновной исполнитель:
  • Администратор конфигурации
Дополнительные исполнители:
ВходыОбязательный:
  • План управления конфигурацией
  • Хранилище проектов
    		Необязательный:
    • Нет
    Выходы
    • Показатели проекта
      Шаги
      Создание отчета о состоянии конфигурации и тенденциях дефектов

      Задачи учета состояния конфигурации основаны на формальной процедуре регистрации и мониторинга состояния предложенных изменений, а также состояния реализации предложенных изменений.

      Отчет о дефектах проекта создается в соответствии с требованиями к отчетам о состоянии конфигурации проекта (см. раздел Задача: Задание стратегий управления конфигурацией).
