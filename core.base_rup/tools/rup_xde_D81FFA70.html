Основное описание

IBM® Rational® XDEtm представляет собой инструмент для графической разработки и моделирования компонентов, который использует стандартный язык Unified Modeling Language (UML).

Следующие памятки по инструментам охватывают как Rational XDE Developer - Java Platform Edition, так и Rational XDE Developer - Microsoft .NET Edition.  Отмечены существующие различия между этими двумя платформами.

Обзор

Rational XDE Developer - Java Platform Edition объединяет проектирование и разработку на Java в тесно интегрированную среду. Эта единая среда позволяет избежать необходимости переключения между инструментами вне этой среды.

Rational XDE имеет ряд способов для усовершенствования методов вашей работы. Например, вы можете:

  • Полностью перейти от анализа и проектирования к написанию исходного кода в той же среде.
  • Задать автоматическую или ручную синхронизацию модели и исходного кода.
  • Определить и шаблоны исходного кода, и шаблоны модели с целью сохранения времени и обеспечения соблюдения стандартов.
  • Создать и модели UML, и модели свободной формы и проверить модели на согласование с UML.
  • Скомпоновать несколько моделей в одном и том же проекте, которые могут ссылаться друг на друга.
  • Многократно использовать определенные шаблоны для совместной работы с исходным кодом в вашем коллективе.

Rational XDE Developer - Java Platform Edition можно использовать со следующими продуктами платформы Websphere Studio:

  • IBM Websphere Studio Workbench (WSW)
  • IBM Websphere Studio Application Developer (WSSAD) - аналогичен WSW, однако включает дополнительные функциональные возможности для Java (TM) 2 Enterprise Edition (J2EE) и разработки Web-служб.

Оба являются средами разработки и интеграции инструментов, составленными из проекций, представлений, редакторов, меню и других строительных блоков для среды Rational XDE. Обычно используется несколько проекций:

  • Проекции Java, содержащие редакторы Java, программы просмотра классов и некоторые другие редакторы.
  • Проекция моделирования, содержащая Структуру модели, Инструменты и другие представления.

Rational XDE выполняет практически одно и то же в обеих средах продукта. Некоторые второстепенные отличия упоминаются в памятках по инструментам.
