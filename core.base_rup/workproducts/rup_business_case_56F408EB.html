Артефакт: Экономическое обоснование проекта
Этот артефакт предоставляет необходимую информацию (с точки зрения бизнеса) для определения того, стоит ли инвестировать в данный проект или нет. Для коммерческого программного продукта Бизнес-прецедент должен включать ряд предположений о проекте и порядок величины возврата инвестиций (ROI), если эти предположения верны. Например, ROI будет равен пяти в случае завершения проекта за один год, двум - при завершении за два года, а по истечении двух лет - иметь отрицательное значение. Эти предположения заново проверяются в конце этапа уточнения, когда рамки проекта и план определены более точно.
Домены: Управление проектом
Типы рабочих продуктов: Оценка
Назначение

Основной целью бизнес-прецедента является разработка экономического плана для реализации проекта Рабочий продукт: Представление. После разработки Бизнес-прецедент применяется для выполнения правильной оценки возврата инвестиций (ROI), который обеспечивается проектом. Он предоставляет обоснование проекта и устанавливает его экономические ограничения. Он предоставляет информацию об экономической ценности проекта для лиц, принимающих экономические решения, и применяется для определения того, следует ли продвигать проект.

В критических вехах Бизнес-прецедент изучается заново для определения того, по-прежнему ли правильны оценки возврата и стоимости, и следует ли продолжать проект.

Доводка
Опции представления

Форма и глубина анализа, необходимого для данного рабочего продукта, зависят от уровня инвестиций, требуемых для проекта. Необходимость высокого уровня инвестиций потребует чрезвычайно формального, хорошо изученного и хорошо обоснованного бизнес-прецедента. Чем больше объем инвестиций, тем больше оценок потребует высшее руководство.

В случае проекта, выполняемого по контракту (например, в результате победы в тендере), Запрос на предложение, ответ и последующий контракт совместно формируют Бизнес-прецедент.
