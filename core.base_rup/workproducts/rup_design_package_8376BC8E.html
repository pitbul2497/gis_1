Артефакт: Проектирование пакета
Этот рабочий продукт представляет собой набор классов, взаимосвязей, реализаций вариантов использования проектирования, диаграмм и других пакетов. Он применяется для выделения структуры модели проектирования путем ее разделения на более мелкие части.
Типы рабочих продуктов: Элементы модели
Назначение

Пакеты проектирования применяются для группировки связанных элементов Модели проектирования для организационных целей, а также часто для управления конфигурацией. В отличие от Рабочего продукта: Подсистема проектирования, пакет проектирования не предлагает формальный интерфейс, хотя может демонстрировать некоторое свое содержимое (помеченное как 'общее'), которое предлагает алгоритм работы. Пакеты проектирования следует в первую очередь использовать как инструмент организации модели для группировки связанных элементов; если требуется поведенческая семантика, используйте подсистемы проектирования.

За пакет проектирования и его содержимое отвечает один Роль: Проектировщик. Элементы в пакете могут зависеть от элементов, содержащихся в других пакетах; это вызывает зависимости между пакетами. Зависимости пакетов можно использовать как инструмент для анализа гибкости модели проектирования: модель с перекрестными зависимостями пакетов является менее гибкой по отношению к изменениям.

Пакеты применяются в моделях для группировки схожих элементов, улучшая организацию модели и облегчая ее понимание. Создание пакетов существенно в больших моделях. Даже в меньших моделях соответствующее создание пакетов может существенно повысить понятность системы. Создание некоторого количества пакетов практически всегда является полезным. Более подробная информация приведена в Рекомендации по рабочему продукту: Пакет проектирования.
