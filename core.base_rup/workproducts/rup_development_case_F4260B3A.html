Артефакт: Прецедент разработки
Этот артефакт описывает процесс разработки, которому будет следовать данный проект.
Типы рабочих продуктов: Процесс
Назначение

Прецедент разработки предназначен для фиксации настроенного процесса для организации или отдельного проекта. Он описывает настройку процесса.

Описание
Основное описание

Прецедент разработки описывает Процесс разработки , который будет применяться для организации или для проекта. Он описывает, как процесс применяется к контексту организации/проекта, и как процесс настроен для организации/проекта.  Прецедент разработки применяется для фиксации относящихся к организации или проекту аспектов процесса.  Он предоставляет обзор процесса, который будет применяться, и моменты, которые должны быть осознаны всеми участниками проекта. 

Краткая схема

Прецедент разработки включает информацию о дисциплинах RUP, которые будут использоваться организацией/проектом.

Он обычно содержит следующую относящуюся к организации/проекту информацию:

  • Информация об использовании рабочих продуктов по каждой дисциплине. Например, ссылки на шаблоны, которые организация/проект будет использовать для описания рабочих продуктов.
  • Ссылки на рекомендации и информацию, которые организация/проект желает использовать в дополнение к информации, содержащейся в RUP.
  • Ссылки на информацию в RUP, вместо повторения этой информации в Прецеденте разработки.

В общем, прецедент разработки должен ссылаться на процесс разработки программного обеспечения и на все относящиеся к организации/проекту ресурсы процесса, такие как рекомендации, шаблоны и так далее, для получения подробной информации, а не повторять эту информацию.

Прецедент разработки также включает информацию о том, что именно и с какой целью было настроено.

Ключевые условия

В зависимости от уровня выполненной подгонки (см. уровни подгонки, определенные в Концепции: Настройка RUP), Прецедент разработки можно использовать как:

  • Средство для подгонки процесса (это его основное назначение и то, для чего его использовали изначально).
  •  Инструмент планирования для подготовки к расширенной настройке с помощью Rational Method Composer.

Прецедент разработки затем можно включить как часть самого Процесса разработки , либо просто сохранить как хронологическую документацию, описывающую, что было добавлено/удалено/изменено по сравнению с исходным процессом.  Прецедент разработки  обычно включается как часть процесса разработки в тех экземплярах, где подогнанный процесс нельзя изменить, либо существует информация о процессе, которую следует зафиксировать, и которую нельзя зафиксировать в самом процессе.
