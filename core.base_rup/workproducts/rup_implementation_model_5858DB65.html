Артефакт: Модель реализации
Модель реализации представляет физическую структуру реализации в терминах подсистем реализации и элементов реализации (каталогов и файлов, включая файлы исходного кода, файлы данных и исполняемые файлы).
Домены: Реализация
Типы рабочих продуктов: Модель
Назначение

Модель реализации идентифицирует физические компоненты реализации с целью облегчения их восприятия и управления ними. Модель реализации определяет основные блоки интеграции, вокруг которых организованы рабочие группы, а также блоки, которым можно по отдельности присваивать версии, отдельно развертывать и заменять.

Более подробная модель реализации может также включать файлы исходного кода низкого уровня и порожденные файлы, а также их взаимосвязь с моделью проектирования. Такая степень подробности рекомендуется только в том случае, если имеется автоматическая синхронизация между моделью и файлами.

Доводка
Опции представления

Представления UML: Модель, имеющая стереотип <<implementation model>>. 

Модель реализации может иметь следующие свойства:

  • Введение: Текстовое описание, которое служит в качестве краткого введения в модель.
  • Подсистемы реализации:Подсистемы в модели, представляющие иерархию.
  • Элементы реализации: Элементы в модели, принадлежащие подсистемам.
  • Взаимосвязи: Взаимосвязи в модели, принадлежащие подсистемам реализации. 
  • Диаграммы: Диаграммы в модели, принадлежащие подсистемам реализации.
  • Представление реализации: Представление реализации модели, которое представляет собой архитектурное представление, показывающее подсистемы и уровни реализации.   

Модель реализации является необязательной. Если вы будете создать модель реализации, то основные решения относятся к взаимосвязи между моделями реализации и проектирования, и выбору для модели достаточно важных элементов реализации. Руководство по принятию таких решений приведено в разделе Рекомендации по рабочему продукту: Модель реализации. Дополнительная информация приведена в разделе Приемы: Отображение проекта на исходный код.
