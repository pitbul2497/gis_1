Артефакт: План итерации
Этот артефакт представляет собой последовательный во времени, содержащий зависимости задач, набор деятельностей и задач с присвоенными ресурсами для итерации; подробный план.
Домены: Управление проектом
Типы рабочих продуктов: План
Назначение

План итерации используется следующими сотрудниками:

  • Руководителем проекта - для планирования задач итерации, требуемых ресурсов и для мониторинга хода итерации по отношению к расписанию.
  • Участниками коллектива проекта - для понимания того, что и когда требуется сделать, и от каких еще деятельностей зависит их работа.

Описание
Основное описание

План итерации должен подробно описывать, что следует выполнить, так чтобы оставалось как можно меньше места неопределенности относительно размещения или ответственностей в любой момент времени. Обычно применяется какой-либо инструмент для планирования проекта. Это подробный план для одной итерации. Часто существует два таких плана: один для текущей итерации и еще один, находящийся в разработке - для следующей итерации.

Для определения содержимого итерации требуется:

  • план проекта
  • текущее состояние проекта (в порядке, запаздывает, большое количество неполадок, смещение требований и так далее)
  • список сценариев или вариантов использования, которые должны быть выполнены к концу итерации
  • список рисков, которые следует изучить до конца итерации
  • список изменений, которые следует внести в проект (исправления ошибок, изменения в требованиях)

  • список основных классов или пакетов, которые должны быть полностью реализованы

Эти списки должны быть упорядочены. Цели итерации должны быть ясными, так чтобы при возникновении трудностей элементы можно было удалять из итераций на основании их ранга.

Критерии оценки

Каждая итерация завершается оценкой. При этом оцениваются результаты итерации относительно критериев оценки, которые были определены для плана итерации.

Критерии оценки устанавливаются перед каждой итерацией и задают цели для набора функциональных возможностей, качества и производительности, которые должны быть достигнуты в итерации. На практике достижение этих целей будет различным. Например, в данной итерации набор функциональных возможностей может оказаться шире, качество - почти достигнуто, а производительность - недостаточной.

Также цели могут выражаться как минимальные или желаемые. Например, может быть требуемый набор функциональных возможностей, а также некоторые желаемые функции, которые будут опробованы в этой итерации, если скорость разработки и уровень персонала сделают это возможным.
