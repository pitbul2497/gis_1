Артефакт: План разработки программного обеспечения
Этот артефакт представляет собой всесторонний, составной артефакт, который собирает всю информацию, необходимую для управления проектом. Он включает в себя ряд артефактов, разработанных на начальном этапе, и поддерживается на протяжении проекта.
Домены: Управление проектом
Типы рабочих продуктов: План
Назначение

Назначение плана разработки программного обеспечения - сбор всей информации, необходимой для управления проектом. Он описывает подход к разработке программного обеспечения и является планом верхнего уровня, создаваемым и применяемым руководителями для направления усилий по разработке продукта.

План разработки программного обеспечения используется следующими сотрудниками:

  • Руководителем проекта - для планирования расписания проекта и требований к ресурсам, а также для мониторинга хода проекта в соответствии с расписанием.
  • Участниками коллектива проекта - для понимания того, что и когда требуется сделать, и от каких еще деятельностей зависит их работа.

Существуют ситуации, когда в договоре указывается стандарт, обусловливающий структуру и содержимое плана разработки программного обеспечения. Дополнительная информация

Хорошие планы разработки программного обеспечения развиваются. Полезный план периодически обновляется (это не застоявшееся полочное ПО), и осознается и используется руководителями и исполнителями.

План разработки программного обеспечения является определяющим документом для процесса проекта. Подготовьте единый план разработки ПО, который:

  • соблюдает организационные стандарты для содержимого
  • соответствует договору (если таковой имеется)
  • обеспечивает трассируемость договора и требований организации
  • обновляется на каждой основной вехе
  • развивается вместе с эскизом и требованиями

Стандартный формат поддерживает:

  • многократное использование процессов, методов, опыта и сотрудников
  • неоспоримость организационных ожиданий
  • однородные цели процесса

Главным отличительным признаком хорошего плана разработки программного обеспечения является его лаконичность, отсутствие пустых рассуждений и сосредоточение на значимых стандартах и процедурах.
