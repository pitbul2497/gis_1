Артефакт: План контроля качества
Этот артефакт предоставляет ясное представление о том, каким образом следует обеспечивать качество продукта, артефакта и процесса. Он содержит план оценки и контроля и ссылается на ряд других артефактов, разработанных на начальном этапе. Он сохраняется на протяжении всего проекта.
Типы рабочих продуктов: План
Назначение

Для обеспечения единой контрольной точки для качества проекта.

Описание
Основное описание

План контроля качества  содержит план оценки и контроля и ссылается на ряд других артефактов, разработанных на начальном этапе. Он сохраняется на протяжении всего проекта.

Это ориентированный на процесс рабочий продукт, выделяющий те элементы RUP, которые вносят вклад в достижение качества. План контроля качества не будет содержать сведения о методиках, критериях, показателях и так далее соответствующих обзоров и оценок, сосредоточенных на продукте. Требуется, для соответствия RUP, чтобы сведения об оценке качества продукта предоставлялись в разделе План оценки в Плане разработки программного обеспечения и в Плане тестирования.

Доводка
Опции представления

План контроля качества содержит информацию, которая в большей или меньшей степени может содержаться и в других планах.  Для управления таким возможным наложением можно использовать следующие подходы:

  • Ссылайтесь на содержимое другого плана.
  • Представьте обзор в другом плане и более подробные сведения в данном плане.  Также могут быть полезны ссылки из других планов на план контроля качества.  Это обычно хорошо подходит для крупных проектов, в которых за контроль качества отвечает отдельная организация.
  • Составьте разделы документов так, чтобы они охватывали только те области, которые более нигде не описывались.

Ниже приведено отображение разделов Плана контроля качества на рабочие продукты, которые могут содержать дополнительную информацию:

Раздел План контроля качества  Дополнительный рабочий продукт 
Определения, акронимы и аббревиатуры  Глоссарий 
Управление  План разработки программного обеспечения 
Документация  Прецедент разработки 
Стандарты и рекомендации  Прецедент разработки 
Показатели  Прецедент разработки, План измерений 
План оценки и контроля  Прецедент разработки, План управления конфигурацией 
Оценка и тестирование  Прецедент разработки, План разработки программного обеспечения (План оценки), План тестирования 
Устранение неполадок и корректирующие действия  План устранения неполадок 
Инструменты, методики и методологии  Прецедент разработки, План разработки программного обеспечения (Методы, инструменты и методики) 
Управление конфигурацией  План управления конфигурацией 
Контроль за поставщиками и субподрядчиками  План разработки программного обеспечения (План управления субподрядчиками, План инфраструктуры) 
Записи о качестве  План управления конфигурацией 
Обучение  План разработки программного обеспечения 
Управление рисками  План управления рисками 

В некоторых проектах может не создаваться план контроля качества, а вся необходимая информация может содержаться в других планах.
