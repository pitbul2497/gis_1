Артефакт: Атрибуты требований
Этот рабочий продукт описывает хранилище требований, атрибутов и зависимостей проекта для содействия управлению изменениями из проекции требований.
Домены: Требования
Типы рабочих продуктов: Данные проекта
Назначение

Рабочий продукт Атрибуты требований предоставляет хранилище текста, атрибутов и трассируемости всех требований. Доступ к нему должны иметь все сотрудники организации-разработчика.

Взаимосвязи
РолиОтветственный:
  • Аналитик
Изменен:
  • Аналитик
  • Разработчик архитектуры
  • Разработчик требований

  • Назначение приоритетов вариантов использования
  • Проверка требований
  • Создание подробного описания требований к программному обеспечению
  • Управление зависимостями
Выход из:

  • Определение требований заинтересованных лиц

  • Разработка дополнительных спецификаций

Описание
Краткая схема

Для просмотра текущего состояния этого рабочего продукта должны быть доступны следующие представления:

1. Матрицы атрибутов требований

1.1 <тип требования>
Для каждого типа требований представлена матрица, в которой по одной оси представлены требования, а по другой - все атрибуты. Для каждого требования отображается состояние соответствующих атрибутов.

2. Матрицы трассируемости требований

2.1 <тип требования>

2.1.1 <тип трассируемого требования>
Для каждого типа требований представлена матрица, в которой по одной оси перечислены требования, а все трассируемые элементы - по другой оси. Для каждой трассировки отображается ее состояние (OK или подозрительная).

3. Дерево трассируемости требований

3.1 <тип требования>

3.1.1 <тип трассируемого требования>
Дерево трассируемости обеспечивает графическое представление взаимосвязей трассируемости к или от требований определенного типа требований (корневой субъект).

Доводка
Опции представления

Примите решение о том, следует ли документировать атрибуты в рабочих продуктах требований,  в плане управления требованиями или управлять ими с помощью инструментов управления требованиями, таких как Rational RequisitePro.  При использовании инструмента управления требованиями можно управлять информацией хранилища непосредственно в сохраненных представлениях в базе данных управления требованиями.
