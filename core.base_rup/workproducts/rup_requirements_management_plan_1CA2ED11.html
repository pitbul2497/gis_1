Артефакт: План управления требованиями
Этот артефакт описывает артефакты требований, типы требований и соответствующие атрибуты требований, указывая, какую информацию следует собрать и какие механизмы контроля использовать для измерений, составления отчетов, и контролируя изменения в требованиях к продукту.
Домены: Требования
Типы рабочих продуктов: План
Назначение

План управления требованиями необходимо разрабатывать для указания информации и механизмов контроля, которые будут собираться и какие использоваться для измерений, составления отчетов и контроля изменений в требованиях к продукту.

Назначение плана управления требованиями - описать настройку и управление рабочих продуктов требований, связанных типов требований (см. Концепции: Типы требований), атрибутов требований (см. Рабочий продукт: Атрибуты требований). Этот план также указывает, как будет выполняться управление трассируемостью (см. Концепции: Трассируемость).

Опции представления

Настройка, как минимум, должна включать определение элементов, ограничений и атрибутов трассируемости, применимых к данному проекту. Также значимыми вопросами трасссируемости являются:

  • взаимосвязь с другими планами
  • замечания по инструментам

Взаимосвязь с другими планами

План управления требованиями содержит информацию, которая в большей или меньшей степени может содержаться и в других планах. Для управления таким возможным наложением можно использовать следующие подходы:

  • Ссылайтесь на содержимое другого плана.
  • Представьте обзор в другом плане и более подробные сведения в данном плане. Могут также быть полезны ссылки из других планов на план управления требованиями. Это обычно хорошо подходит для крупных проектов, в которых за управление требованиями отвечает отдельная организация.
  • Составьте разделы документов так, чтобы они охватывали только те области, которые более нигде не описывались.

Ниже приведено отображение разделов Плана управления требованиями на рабочие продукты, которые могут содержать дополнительную информацию:

Замечания по инструментам

Вместо документирования атрибутов трассируемости и ожидаемых значений отдельно, можно ввести эту информацию непосредственно в инструмент, который используется для управления требованиями. Таким образом, в плане управления требованиями понадобится задокументировать только их применение.

Обратите внимание, что план управления требованиями иногда применяется для документирования не только непосредственных элементов управления требованиями. Например, пользователи Rational RequisitePro часто используют этот документ для фиксации других элементов, управляемых данным инструментом, таких как термины глоссария, элементы действий требований и так далее. Однако, хотя RequisitePro также можно использовать для управления такими элементами, как риски и вопросы, в RUP они считаются отдельными рабочими продуктами, управление которых не охватывается планом управления требованиями.
