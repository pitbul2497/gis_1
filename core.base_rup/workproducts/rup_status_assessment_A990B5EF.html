Описание
Основное описание

Одна из задач процесса - гарантировать, что ожидания всех участвующих сторон синхронизированы и согласованы. Периодическая оценка состояния предоставляет механизм управления ожиданиями на протяжении жизненного цикла проекта.

Постоянная открытая связь с объективными данными, создаваемыми непосредственно в текущих деятельностях, и развитие конфигураций продукта обязательны для любого проекта. Данные моментальные копии проекта обеспечивают основу для управления. В то время как периодичность может изменяться, вынуждающая функция должна фиксировать хронологию проекта.

Обратите внимание, что оценка состояния отличается от проверок на вехах.

Доводка
Опции представления

Оценку состояния можно сочетать с оценкой итерации, если итерации выполняются часть (один раз в месяц). В случае очень длинных итераций могут потребоваться промежуточные оценки состояния.

Дополнительная информация

Большая часть информации в Оценке состояния копируется из других источников с целью предоставления всестороннего источника информации для сотрудников, оценивающих проект. Поскольку оценка состояния не обслуживается, нет необходимости в обеспечении согласования информации с развивающейся системой.
