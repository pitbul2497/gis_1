Артефакт: Стратегия тестирования
Этот артефакт определяет стратегический план выполнения тестирования по отношению к одному или нескольким аспектам целевой системы.
Домены: Тестирование
Типы рабочих продуктов: Концепция
Назначение
  • Для выражения стратегии для внешних заинтересованных лиц с целью получения их согласия с данным подходом.
  • Для выражения стратегии для внутренних участников группы тестирования с целью обеспечения координации тестирования.
  • Для того чтобы убедить руководство и других заинтересованных лиц в том, что данный подход является правильным и доступным.

Описание
Краткая схема

Стратегия тестирования фиксирует следующие информационные элементы:

  • Объяснение общего подхода, который будет применяться. Например, объясните, как основной подход будет использовать проверку программного обеспечения на соответствие требованиям или спецификациям проекта, тестирование программного обеспечения в моделях сбоев, будет подвергнуто воздействию известных атак или какой-либо другой общий подход.
  • Определенные типы, методики, стили тестирования, которые будут применяться как часть стратегии, и для каждого из них:
    • Указание рамок и применимости методики
    • Схема применения методики.
    • Какие инструменты потребуются для поддержки данной методики.
    • Критерии определения успешности и текущая оценка применения методики
    • Указания на слабые места или ограничения методики, а также могут ли другие методики охватить их.

Обратите внимание, что для конкретной системы программного обеспечения в данном контексте (технология, домен и так далее), вероятно, всю стратегию или ее часть может будет использовать повторно в последующих жизненных циклах разработок.

Доводка
Опции представления

В определенных культурах тестирования Стратегия тестирования считается неофициальным нерегулярным рабочим продуктом, а в других - является в значительной степени формализованным и часто требует выхода из системы. По существу, формат и содержимое должны изменяться в соответствии с конкретными потребностями организации или проекта.

В качестве альтернативы официальной документации можно просто записать элементы стратегии тестирования в виде набора неофициальных заметок по планированию, возможно, размещаемых на Web-сайте внутренней сети или на доске, и доступных для группы тестирования.
