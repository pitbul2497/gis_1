Артефакт: Рабочее задание
Этот артефакт представляет собой средства руководителя проекта для сообщения ответственному персоналу, что и когда следует выполнить. Он становится внутренним договором между руководителем проекта и ответственными за выполнение.
Домены: Управление проектом
Типы рабочих продуктов: Данные проекта
Назначение

Назначение рабочего задания - превратить планирование в действия в рамках итераций.

Выход из:

Описание
Основное описание

Рабочее задание является договорное соглашение между руководителем проекта и персоналом о выполнении определенной задачи или ряда задач в рамках определенного расписания и имеющего определенные ограничения по конечным продуктам, затраченным усилиям и ресурсам.

Рабочие задания становятся внутренними договорами между руководителем проекта и ответственными за выполнение.
Краткая схема

1. Идентификация

Уникальным образом идентифицирует проект и рабочее задание.

2. Идентификация структуры задания (WBS)

Идентифицирует деятельность (в плане проекта), связанную с данным рабочим заданием. Усилия, затрачиваемые на это рабочее задание, будут выделены этой деятельности для мониторинга.

3. Ответственность

Должности в организации, ответственные за выполнение этого рабочего задания.

4. Связанные запросы изменений

Ссылки на запросы изменений, связанные с данным рабочим заданием (те, которые будут являться побуждающими сигналами, и те, которые будут зафиксированы случайным образом).

5. Расписание

Расписание, охватывающее предполагаемые даты начала и завершения, а также дата завершения по критическому пути.

6. Затраченные усилия и другие ресурсы

Передает человеко-часы, общее и сверхурочное время, а также бюджеты других ресурсов; например, время в среде разработки, время в среде тестирования.

7. Описание работы и ожидаемых выходных данных

Описывает, что следует выполнить и что следует создать - ссылается на описание в Rational Unified Process задач, которые следует выполнить, и рабочих продуктов, которые следует создать, или на вариант разработки.

8. Указание на соглашение между руководителем проекта и ответственным персоналом

Рабочие задание должно быть подписано и датировано ответственным лицом (обычно руководителем группы) и руководителем проекта.

Доводка
Опции представления

Рабочее задание представляет собой механизм, посредством которого руководитель проекта сообщает планы участникам проекта. В небольших проектах это может быть просто обсуждение плана на доске с подтверждением соглашений по электронной почте. В крупных, имеющих сложную структуру проектах возможно использование некоторой формы автоматизированного управления деятельностью, где руководитель проекта вносит официальные распоряжения, которые появляются в списках задач участников проекта (возможно ведение какого-либо протокола для соглашений).

Еще одной возможностью является использование автоматизированной системы управления запросами изменений, расширенной таким образом, чтобы вся работа по проекту (а не только по изъянам) описывалась в запросах изменений, а распоряжения на выполнение работ реализовывались в виде действий (например, через электронную почту, или посредством интеграции с автоматизированной системой управления деятельностями), которые переключаются изменениями состояний в процессе управления запросами изменений. Пример настройки автоматизированного процесса управления изменениями, переключающего внешние действия, приведен в разделе Руководство по инструменту: Настройка процесса запроса изменений с помощью Rational ClearQuest.
