Артефакт: Модель анализа нагрузки
Этот артефакт представляет собой модель, которая идентифицирует один или несколько профайлов нагрузки, считающихся правильно определяющими интересующее состояние системы, в котором может быть проведена оценка программного обеспечения и/или его операционной среды.
Домены: Тестирование
Типы рабочих продуктов: Модель
Расширяет: Тестовый набор
Назначение
  • Для правильного определения и загрузки условий, в которых должны работать целевые элементы тестирования в своей целевой среде конфигурации.
  • Для определения реалистичной репрезентативной нагрузки, которая позволяет правильно оценить риски для производительности.

Описание
Основное описаниеПрофайлы нагрузки, составляющие модель анализа нагрузки , представляют возможные условия, которые следует имитировать в отношении к целевым элементам тестирования в одной или нескольких конфигурациях среды тестирования.  Они обычно определяются путем анализа ожидаемых или существующих характеристик субъекта, статистики бизнес-процесса пользователя (варианты использования), и так далее.
Краткая схема

1. Введение

Указывает назначение, обоснование и цели тестирования производительности в рамках данного проекта.

2. Системные атрибуты и переменные

Указывает атрибуты и переменные данной системы, которые уникально идентифицируют нагрузку для моделируемой системы.

4. Определения субъектов

Идентифицирует классы внешних клиентов, чьи сценарии вариантов использования необходимо смоделировать для имитации / эмуляции нагрузки на тестируемую систему. Дополнительно, этот раздел идентифицирует вклад в нагрузку для теста производительности со стороны любого субъекта.

7. Атрибуты субъекта

Указывает атрибуты и переменные каждого субъекта, которые уникально идентифицируют различные параметры внешних клиентов системы. Для каждого субъекта указывается такая информация, как является ли субъект человеком, скорость внесения данных, время обдумывания, стиль транзакций, сложность транзакций и шаблоны поведения, характеризующие изменчивость взаимодействия пользователя и системы.

6. Рабочий профайл объекта

Идентифицирует определенные сценарии вариантов использования, выполняемые субъектом, и время в процентном отношении или усилия, затраченные субъектом, выполняющим сценарии вариантов использования, на выполнение всего должностных обязанностей.

3. Профайл нагрузки

Для данного профайла идентифицирует число внешних клиентов, имитируемых / эмулируемых во время теста, включая число, тип и распределение транзакций. Профайл может быть определен в терминах "пиковая нагрузка", "средняя нагрузка" и так далее.

5. Измерения и критерии

Указывает измерения и критерии, которые следует использовать для оценки успешного достижения заданных целей производительности. Измерения обычно включают ограничение по времени ответа или пропускную способность.

8. Требования к эмуляции удаленного терминала

Указывает требования и ограничения, которые необходимые принять во внимание при создании конфигурации среды тестирования, приемлемой для реализации и выполнения тестирования производительности.

Доводка
Опции представления

Может потребоваться изменения модели анализа нагрузки (содержимое и формат) в соответствии с потребностями внутренних или внешних стандартов, рекомендаций и так далее. При необходимости некоторые аспекты модели анализа нагрузки можно включить в План тестирования
